Concept 04 · Compile-Time

The Compile-Time Pipeline

This is where Spark earns its keep. As your module compiles, the DSL passes down an assembly line — and you get to bolt your own stations onto it.

Here's the surprising part: a Spark DSL is just data while it compiles — a map the docs call dsl_state. That means you can write ordinary Elixir to inspect it, rewrite it, and check it, all before the module exists at runtime. No macro gymnastics. The pipeline has five stations:

DSL → parse → schema-validate → transform/persist → verify → generate code.

Transformers — rewrite the DSL

A transformer takes dsl_state and returns a modified one. Want every validator to silently gain an :id field? Add it during compilation:

lib/.../transformers/add_id.exdefmodule MyLibrary.Validator.Transformers.AddId do
  use Spark.Dsl.Transformer

  def transform(dsl_state) do
    {:ok,
     Spark.Dsl.Transformer.add_entity(dsl_state, [:fields],
       %MyLibrary.Validator.Dsl.Field{name: :id, type: :string})}
  end
end

Generating code into the module

Transformers can also inject functions. This is how a DSL grows a real API — here, a validate/1 on the user's module:

generate a functiondef transform(dsl_state) do
  body = quote do
    def validate(data),
      do: MyLibrary.Validator.validate(__MODULE__, data)
  end

  {:ok, Spark.Dsl.Transformer.eval(dsl_state, [], body)}
end

Persisters — cache derived values

A persister is a transformer that only writes to the persisted-data map, and always runs after all transformers. Use it to precompute something once and read it cheaply at runtime:

cache the field namesdef transform(dsl_state) do
  names =
    dsl_state
    |> Spark.Dsl.Transformer.get_entities([:fields])
    |> Enum.map(& &1.name)

  {:ok, Spark.Dsl.Transformer.persist(dsl_state, :field_names, names)}
end

Verifiers — check, don't change

A verifier runs after the module is compiled, is read-only, and returns :ok, {:error, e}, or {:warn, w}. Prefer them over transformers whenever you're only validating:

required ⊆ fieldsdef verify(dsl_state) do
  required = Info.fields_required!(dsl_state)
  fields   = Enum.map(Info.fields(dsl_state), & &1.name)

  if Enum.all?(required, &(&1 in fields)) do
    :ok
  else
    {:error, Spark.Error.DslError.exception(
      message: "required fields must appear in fields",
      path: [:fields, :required],
      module: Spark.Dsl.Verifier.get_persisted(dsl_state, :module))}
  end
end
💥
The classic deadlock

Inside a transformer, never call a function on the module being compiled — it doesn't exist yet, and you'll deadlock the compiler. Work with dsl_state instead. Because verifiers run post-compile, they can safely reference other modules — which is why cross-module checks belong there.

🔑
Rule of thumb

Need to change the DSL? Transformer. Need to cache a value? Persister. Only need to check? Verifier — it's safer and avoids compile-time dependencies.
