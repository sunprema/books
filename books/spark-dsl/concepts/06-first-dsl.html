Build Your First DSL

Time to put the five pieces together into one real, working library: a data validator. We'll go corner to corner — no hand-waving.

We're building MyLibrary.Validator. A user will use it, declare some fields, and get a validate/1 function that type-checks, runs custom checks, and transforms values. Everything we learned — entities, sections, schema, a transformer, a verifier, an info module — shows up exactly once. Follow the four steps.

Step 1 · The extension

Define the entity's target struct, the field entity, and the fields section. Note the __spark_metadata__ field — Spark requires it on entity targets for source-location tracking.

lib/my_library/validator/dsl.exdefmodule MyLibrary.Validator.Dsl do
  alias Spark.Builder.{Entity, Field, Section}

  defmodule Field do
    defstruct [:name, :type, :transform,
               :check, :__spark_metadata__]
  end

  @field Entity.new(:field, Field,
    describe: "A field accepted by the validator",
    args: [:name, :type],
    schema: [
      Field.new(:name, :atom, required: true),
      Field.new(:type, {:one_of, [:integer, :string]},
        required: true),
      Field.new(:check, {:fun, 1}),
      Field.new(:transform, {:fun, 1})
    ]
  ) |> Entity.build!()

  @fields Section.new(:fields,
    describe: "Supported & required fields",
    schema: [Field.new(:required, {:list, :atom})],
    entities: [@field]
  ) |> Section.build!()

  use Spark.Dsl.Extension,
    sections: [@fields],
    verifiers: [MyLibrary.Validator.Verifiers.VerifyRequired]
end

end

Step 3 · The verifier

Every name in required must actually be a declared field:

lib/.../verifiers/verify_required.exdefmodule MyLibrary.Validator.Verifiers.VerifyRequired do
  use Spark.Dsl.Verifier
  alias MyLibrary.Validator.Info

  def verify(dsl) do
    required = Info.fields_required!(dsl)
    fields   = Enum.map(Info.fields(dsl), & &1.name)

    if Enum.all?(required, &(&1 in fields)),
      do: :ok,
      else: {:error, Spark.Error.DslError.exception(
        message: "required fields must be declared",
        path: [:fields, :required],
        module: Spark.Dsl.Verifier.get_persisted(dsl, :module))}
  end
end

Step 4 · The DSL module + logic

lib/my_library/validator.exdefmodule MyLibrary.Validator do
  use Spark.Dsl,
    default_extensions: [extensions: [MyLibrary.Validator.Dsl]]

  def validate(module, data) do
    fields   = MyLibrary.Validator.Info.fields(module)
    required = MyLibrary.Validator.Info.fields_required!(module)

    case Enum.reject(required, &Map.has_key?(data, &1)) do
      [] -> validate_fields(fields, data)
      missing -> {:error, :missing, missing}
    end
  end
  # validate_fields/2 walks each field, runs the
  # type check, the :check fn, then the :transform fn.
end

That's a whole library

Four small files and you have a validator with autocomplete, generated docs, compile-time error checking, and a clean runtime API. Spark wrote the tooling; you wrote the meaning.

Using it

it just worksdefmodule MyApp.PersonValidator do
  use MyLibrary.Validator

  fields do
    required [:name]
    field :name, :string
    field :email, :string do
      check &String.contains?(&1, "@")
      transform &String.trim/1
    end
  end
end

MyApp.PersonValidator.validate(
  %{name: "Zach", email: " foo@example.com "})
#=> {:ok, %{name: "Zach", email: "foo@example.com"}}

And when it's wrong…

Declare a required field you forgot to define, and the verifier fires at compile time with a precise message:

a friendly failure** (Spark.Error.DslError) [MyApp.BadValidator]
fields -> required:
  required fields must be declared

No stack-trace archaeology. The error names the module, the DSL path, and the rule — because you declared that rule in 12 readable lines.

Free tooling to turn on

Run mix spark.formatter to teach the formatter your DSL's locals_without_parens, and add an info-backed elixir_sense plugin for autocomplete. Both ship with Spark — you opt in, you don't build them.
