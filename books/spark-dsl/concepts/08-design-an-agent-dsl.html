Design Your Own Agent DSL

Let's earn the whole book in one example. We'll invent a tiny agent language: a way to declare an LLM agent — its model, its system prompt, and the tools it may call — that the runtime can introspect and hand to a model.

We want users to write the block on the right and get, for free, a validated agent definition plus a function that emits standard tool schemas for any LLM API. Nothing here is exotic — it's Chapters 2–5 aimed at a new domain. That's the point: once you own the pattern, every new DSL is a weekend, not a quarter.

The target syntax

what a user will writedefmodule MyApp.SupportAgent do
  use Agentic

  agent do
    model "claude-opus-4-8"
    system_prompt "You are a calm support agent."
  end

  tools do
    tool :lookup_order do
      description "Find an order by its id"
      param :order_id, :string, required: true
      handler &Orders.lookup/1
    end

    tool :issue_refund do
      description "Refund an order"
      param :order_id, :string, required: true
      destructive? true
      handler &Orders.refund/1
    end
  end
end

The extension

Two entities (param nested in tool), two sections (agent, tools):

lib/agentic/dsl.exdefmodule Agentic.Dsl do
  alias Spark.Builder.{Entity, Field, Section}

  defmodule Param do
    defstruct [:name, :type, :required, :__spark_metadata__]
  end
  defmodule Tool do
    defstruct [:name, :description, :handler,
               destructive?: false, params: [],
               __spark_metadata__: nil]
  end

  @param Entity.new(:param, Param,
    args: [:name, :type],
    schema: [
      Field.new(:name, :atom, required: true),
      Field.new(:type, {:one_of, [:string, :integer, :boolean]},
        required: true),
      Field.new(:required, :boolean, default: false)
    ]) |> Entity.build!()

  @tool Entity.new(:tool, Tool,
    args: [:name],
    entities: [params: [@param]],
    schema: [
      Field.new(:name, :atom, required: true),
      Field.new(:description, :string, required: true),
      Field.new(:destructive?, :boolean, default: false),
      Field.new(:handler, {:fun, 1}, required: true)
    ]) |> Entity.build!()

  @tools Section.new(:tools, entities: [@tool])
         |> Section.build!()

  @agent Section.new(:agent,
    schema: [
      Field.new(:model, :string, required: true),
      Field.new(:system_prompt, :string)
    ]) |> Section.build!()

  use Spark.Dsl.Extension,
    sections: [@agent, @tools],
    verifiers: [Agentic.Verifiers.UniqueTools]
end

Info + a guardrail verifier

lib/agentic/info.exdefmodule Agentic.Info do
  use Spark.InfoGenerator,
    extension: Agentic.Dsl, sections: [:agent, :tools]
end
unique tool names — at compile timedefmodule Agentic.Verifiers.UniqueTools do
  use Spark.Dsl.Verifier

  def verify(dsl) do
    names = Enum.map(Agentic.Info.tools(dsl), & &1.name)

    if names == Enum.uniq(names),
      do: :ok,
      else: {:error, Spark.Error.DslError.exception(
        message: "tool names must be unique",
        path: [:tools],
        module: Spark.Dsl.Verifier.get_persisted(dsl, :module))}
  end
end

Guardrails as code

Want "no agent may declare more than 3 destructive tools"? It's three more lines in a verifier: count &1.destructive? and error past the threshold. Policy that used to live in a wiki now fails the build.

The payoff: introspection → tool schema

Now turn the declared tools into the JSON tool definitions any LLM API expects — purely by reading the DSL back out:

lib/agentic.exdefmodule Agentic do
  use Spark.Dsl,
    default_extensions: [extensions: [Agentic.Dsl]]

  # Build provider-ready tool specs from the DSL.
  def tool_specs(agent) do
    Agentic.Info.tools(agent)
    |> Enum.map(fn tool ->
      %{
        name: tool.name,
        description: tool.description,
        input_schema: %{
          type: "object",
          properties:
            Map.new(tool.params, fn p ->
              {p.name, %{type: Atom.to_string(p.type)}}
            end),
          required:
            for p <- tool.params, p.required, do: p.name
        }
      }
    end)
  end

  # Dispatch a model's chosen call to the right handler.
  def dispatch(agent, name, args) do
    tool = Enum.find(Agentic.Info.tools(agent),
                    &(&1.name == name))
    tool.handler.(args)
  end
end

Look what we built

A human declares an agent in readable Elixir; a verifier enforces safety at compile time; tool_specs/1 emits exactly what an LLM tool-calling API wants; dispatch/3 routes the model's answer back to real code. The DSL is the single source of truth for both the human and the machine. That's the whole promise of this book, standing on its own two feet.
