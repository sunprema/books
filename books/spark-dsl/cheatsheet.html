Spark in 30 seconds

Describe a language as entities in sections, package them in an extension. Spark validates input against your schema, runs transformers/verifiers at compile time, and an Info module reads it all back out at runtime. Because the result is inspectable data, it's a natural fit for LLM tool catalogs.

The four building blocks

PieceRole
Entityone constructor (a word)
Sectiona named block (a clause)
Extensionpackages sections + logic
Spark.Dslthe module users use

Define an entity (builder)

@field Entity.new(:field, Field,
  args: [:name, :type],
  schema: [
    Field.new(:name, :atom,
      required: true)
  ]) |> Entity.build!()

Section + extension

@fields Section.new(:fields,
  entities: [@field])
  |> Section.build!()

use Spark.Dsl.Extension,
  sections: [@fields],
  transformers: [...],
  verifiers: [...]

Struct form (legacy)

@field %Spark.Dsl.Entity{
  name: :field,
  target: Field,
  args: [:name],
  schema: [...]
}

Compile-time pipeline

HookUse when
Transformerchange the DSL / gen code
Persistercache a derived value
Verifiercheck (read-only, post-compile)

Never call the compiling module inside a transformer → deadlock.

Introspection

use Spark.InfoGenerator,
  extension: My.Dsl,
  sections: [:fields]

# generated:
Info.fields(mod)
Info.fields_required(mod) # {:ok,v}|:error
Info.fields_required!(mod) # v|raise

The one thing to remember

A Spark DSL is data that describes capability. Humans write it; verifiers guard it; Info modules read it; and machines — including LLMs — can consume it. Describe, don't do.
