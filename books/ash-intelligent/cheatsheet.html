The AshIntelligence Cheatsheet

🔑
AshIntelligence in 30 seconds

A declarative intelligence layer for Ash. A domain event flows through six engines — Event → Context → Memory → Priority → Reasoning → Execution — and out the far end comes a recommendation you can act on. You don't script that pipeline; you declare it in a Spark intelligence do … end block, the same way you declare resources and policies.

Guiding principle

Business logic should be declarative.
Intelligence should be declarative too.

The six engines

EngineJob
Eventobserve domain events
Contextstructured snapshot, no raw DB
MemoryUser / Team / Org tiers
Priorityscore + reason + confidence
Reasoningrecommendations, briefs, risks
Executionactions, gated by approval

The DSL at a glance

elixir · intelligence blockintelligence do
  provider :anthropic
  memory :persistent

  brief :daily do
    include :priorities
    include :risks
  end

  priorities do
    weight :deadline, 0.4
    weight :customer_risk, 0.3
  end

  connector :github
  agent :reviewer
end

Safety rules

  • Pass actor: / tenant: so authz is inherited.
  • Never give the model raw DB / SQL access.
  • Private attributes stay hidden unless loaded.
  • Gate side-effects behind Awaiting Approval.
