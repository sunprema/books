Chapter 04Scaled Dot-Product Attention

“Everything should be made as simple as possible, but no simpler.” — attributed to Einstein

This is the equation. The authors named their mechanism "Scaled Dot-Product Attention," and every word in that name is a design decision. Let's read it as four small steps, not one scary line.

Attention(Q, K, V) = softmax( QK⊤ / √dk ) V

Recall the setup: we pack all the queries into a matrix Q, all the keys into K, and all the values into V — one row per token. Doing it as matrices is the point: the whole sentence is scored in two matrix multiplies, which a GPU eats for breakfast. Here are the four steps.

Step 1 — Score: QK⊤

The dot product of two vectors is large when they point the same way and near zero when they're unrelated — it's a similarity meter. So QK⊤ is a table of similarities: entry (i, j) is how much query i likes key j. For an n-token sentence this is an n × n grid of raw scores, sometimes called the attention logits.

Step 2 — Scale: ÷ √dk

Here is the "scaled" in the name, and it's the step people skip and regret. Each key/query vector has dk = 64 dimensions. A dot product sums 64 little products; if the components are roughly independent with variance 1, the sum has variance dk — it grows with dimension. Large raw scores push the softmax into a corner where one weight is ~1 and the rest are ~0, and there the gradient nearly vanishes, stalling learning.

Why √dk exactly (the paper's footnote)

If the components of q and k are independent with mean 0 and variance 1, then q·k has mean 0 and variance dk, so its standard deviation is √dk. Dividing by √dk rescales the scores back to variance 1 — keeping the softmax in its sensitive, learnable range. With dk = 64, that divisor is 8.

Step 3 — Normalize: softmax

Softmax turns a row of real-valued scores into a row of positive weights that sum to 1 — a probability distribution over "how much to attend to each word." It exponentiates (so bigger scores dominate, but nobody hits exactly zero) and divides by the total. After this step, each query has a tidy set of attention weights across all keys.

Step 4 — Blend: × V

Finally, multiply the weight rows by V. Each output row is a weighted sum of value vectors — exactly the "blend of book contents" from Chapter 3, now as arithmetic. Out comes one vector per token, the same 512 wide as we put in, but each is now steeped in the context most relevant to it. Same shape in, same shape out — which is why we can stack these layers like bricks.

Additive vs. dot-product: why this choice

An older alternative, additive attention, scores with a tiny neural network instead of a dot product. In theory they're comparable; in practice dot-product attention is "much faster and more space-efficient," because it's just matrix multiplication — the single most optimized operation on a GPU. The authors chose speed, then paid the one tax that speed introduced (exploding scores) with the √dk scaling. That's the entire story of the name.

def attention(Q, K, V):            # Q,K:(n,d_k)  V:(n,d_v)
    scores = Q @ K.T / sqrt(d_k)   # (n,n) similarities, scaled
    weights = softmax(scores, axis=-1)  # each row sums to 1
    return weights @ V               # (n,d_v) context-aware output
The Key Idea

Score with a dot product, tame it with √dk, normalize with softmax, blend the values. Four steps, two matrix multiplies, one differentiable line. That's attention — and it's genuinely all you need.

So what did we actually gain?

A single attention head can now let every word gather context from every other word, fast and end-to-end trainable. But one head has to average everything together — and sometimes a word needs to track several different relationships at once (who is the subject? what's the tense? which noun does this adjective modify?). One blend can't do all three cleanly. The fix is almost embarrassingly simple: run several attention heads in parallel. That's next.
