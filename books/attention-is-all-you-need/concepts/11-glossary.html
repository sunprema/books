Chapter 11The Glossary

Every acronym and term we've used, spelled out in one place. Skim it now, or keep it open as a reference while you re-read. Where a concept has its own chapter, that's noted.

Acronyms, expanded

  • LLM — Large Language Model. A Transformer (usually decoder-only) with billions of parameters, pretrained to predict text. (Ch. 10)
  • Q / K / V — Query, Key, Value. The three roles every token plays in attention: what it seeks, what it advertises, what it offers. (Ch. 3)
  • RNN — Recurrent Neural Network. Processes a sequence one step at a time, carrying a hidden state. The bottleneck the Transformer escaped. (Ch. 1)
  • LSTM — Long Short-Term Memory. A gated RNN designed to remember information over longer spans. (Ch. 1)
  • GRU — Gated Recurrent Unit. A streamlined gated RNN, cousin to the LSTM. (Ch. 1)
  • CNN — Convolutional Neural Network. Uses sliding filters; the basis of ConvS2S and ByteNet, the Transformer's convolutional rivals. (Ch. 1, 8)
  • FFN — Feed-Forward Network. The two-layer, per-position sub-layer after attention. (Ch. 7)
  • ReLU — Rectified Linear Unit. The activation max(0, x) inside the FFN. (Ch. 7)
  • BPE — Byte-Pair Encoding. A subword tokenizer that merges frequent character pairs; used for English–German. (Ch. 2)
  • PE — Positional Encoding. The sinusoidal position signal added to embeddings. (Ch. 6)
  • BLEU — Bilingual Evaluation Understudy. The 0–100 overlap metric for translation quality. (Ch. 9)
  • Adam — Adaptive Moment Estimation. The optimizer used, with per-parameter adaptive learning rates. (Ch. 9)
  • WMT — Workshop on Machine Translation. The source of the benchmark translation datasets. (Ch. 9)
  • GPU — Graphics Processing Unit. Massively parallel hardware; the reason parallelism matters so much. (Ch. 1)
  • FLOP — Floating-Point Operation. The unit used to compare training cost. (Ch. 9)
  • SOTA — State Of The Art. The best published result on a benchmark at a given time.
  • BERT — Bidirectional Encoder Representations from Transformers. The encoder-only family. (Ch. 10)
  • GPT — Generative Pretrained Transformer. The decoder-only, generative family. (Ch. 10)
  • RLHF — Reinforcement Learning from Human Feedback. An alignment stage applied after pretraining. (Ch. 10)
  • RoPE — Rotary Position Embedding. A modern positional scheme that rotates Q and K by position. (Ch. 10)

Terms, defined

  • Attention — A soft, weighted lookup: blend the values by how well the query matches each key. The core operation of the whole model. (Ch. 3–4)
  • Self-attention — Attention where queries, keys, and values all come from the same sequence — every word attending to every word. (Ch. 3)
  • Cross-attention — Encoder– decoder attention: decoder queries against encoder keys/values. (Ch. 7)
  • Multi-head attention — Several attention functions run in parallel in separate subspaces, then concatenated. (Ch. 5)
  • Scaled dot-product — The specific attention: softmax(QKᵀ/√dk)V. (Ch. 4)
  • Softmax — Turns a row of scores into positive weights that sum to 1. (Ch. 4)
  • Embedding — A learned vector representing a token; meaning becomes geometry. (Ch. 2)
  • Token — A discrete unit of text (word or subword) the model actually consumes. (Ch. 2)
  • dmodel — The model width; 512 in the base Transformer. Every sub-layer preserves it. (Ch. 2)
  • dk, dv — Per-head key/value width; 64 in the base model (512 ÷ 8 heads). (Ch. 5)
  • dff — Inner width of the FFN; 2048 in the base model. (Ch. 7)
  • Encoder — The tower that reads the input into context-rich vectors. (Ch. 7)
  • Decoder — The tower that generates output one token at a time. (Ch. 7)
  • Auto-regressive — Generating each token conditioned on the tokens produced so far. (Ch. 7)
  • Masking — Setting future positions' scores to −∞ so the decoder can't peek ahead. (Ch. 7)
  • Residual connection — Adding a sub-layer's input to its output — a gradient highway. (Ch. 7)
  • Layer normalization — Re-centers and rescales a vector to keep activations stable. (Ch. 7)
  • Positional encoding — Sinusoidal vectors added to embeddings to inject word order. (Ch. 6)
  • Path length — How many hops a signal travels between two positions; O(1) for self-attention. (Ch. 8)
  • Warmup — The initial phase where the learning rate ramps up linearly. (Ch. 9)
  • Label smoothing — Training toward slightly-less-than-100% confidence; hurts perplexity, helps BLEU. (Ch. 9)
  • Dropout — Randomly zeroing activations during training to reduce overfitting. (Ch. 9)
  • Perplexity — How "surprised" a model is by the true next token; lower is better. (Ch. 9)
  • Pretraining — Self-supervised training on large unlabeled text before task adaptation. (Ch. 10)
  • Scaling laws — The smooth, predictable improvement in quality as model, data, and compute grow together. (Ch. 10)
  • Transformer — The full attention-based encoder–decoder architecture of the 2017 paper — and the subject of this book. (Ch. 7)
