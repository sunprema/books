Chapter 02Words as Vectors

“You shall know a word by the company it keeps.” — J. R. Firth, 1957

A neural network cannot multiply the word "cat." It can only multiply numbers. So before any attention happens, every word must become a list of numbers — a vector. How we choose those numbers turns out to carry a surprising amount of meaning.

First, chop the text into tokens

Text arrives as a stream of characters; the model wants discrete units. The obvious choice — one unit per word — has a nasty failure mode: any word the model never saw during training becomes a blank unknown token. The Transformer paper sidesteps this with subword tokenization. English–German used byte-pair encoding (BPE) with a shared source–target vocabulary of about 37,000 tokens; English–French used a 32,000 word-piece vocabulary.

The trick: keep common words whole, but split rare ones into reusable fragments. "Attention" might be one token; "tokenization" might split into token + ization. Nothing is ever truly unknown, because in the worst case you fall back to pieces you have seen.

Note

This is why a modern LLM sometimes miscounts the letters in a word or fumbles rhymes: it doesn't see letters, it sees subword tokens. "Strawberry" may reach the model as two or three chunks, not ten characters.

From token to vector: the embedding table

Each token id is looked up in a giant table — the embedding matrix — that stores one learned vector per vocabulary entry. In the base Transformer each vector has 512 numbers. That width has a name the paper uses everywhere: dmodel, the model dimension, equal to 512.

# token ids in, vectors out — just a table lookup
vocab_size = 37000
d_model    = 512
E = Embedding(vocab_size, d_model)   # a 37000 × 512 learned table

ids = [1042, 77, 6, 889]   # "The cat sat down"
X   = E(ids)                  # shape: (4, 512) — one row per token

The numbers start random. During training they drift into positions where meaning becomes geometry: words used in similar contexts end up as nearby vectors, and — famously — directions in the space line up with analogies. This is what Firth's epigraph means, made literal in 512 dimensions.

One matrix, three jobs

The Transformer is thrifty with this table. The same weight matrix is shared in three places: the encoder's input embedding, the decoder's input embedding, and the final linear layer that turns the decoder's output back into scores over the vocabulary (the pre-softmax projection). One learned dictionary, read both forwards (word → vector) and backwards (vector → word). The paper also multiplies the embedding weights by √dmodel — about 22.6 — so the embeddings and the positional signal we add next sit at compatible scales.

The Key Idea

An embedding turns a symbol into a point in space where distance means similarity and direction means relationship. Every layer after this one — all the attention we're about to build — is just a way of moving these points around so each word's vector absorbs the context it needs.

The shape to keep in your head

After this chapter, a sentence is no longer text. It is a matrix: one row per token, 512 columns wide. If the sentence has n tokens, the shape is (n, 512). Hold that picture — the rest of the book is operations on this matrix. But it has a hole in it, and it's a big one: a matrix of rows has no built-in notion of order. Row 1 and row 4 don't know they're first and fourth. We'll patch that hole in Chapter 6; the reason it's even a problem is the very thing we build next.

So what did we actually gain?

We turned language into linear algebra without throwing meaning away. Words became vectors whose geometry the model can sculpt. That's the raw material. Now let's meet the tool that does the sculpting — the single idea this entire paper is named after.
