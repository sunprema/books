The one idea

Write the truth down before you act. Treat the log as reality; rebuild state from it; make every action safe to repeat. The log defeats the crash — everything else is commentary.

The exactly-once formula

at-least-once delivery + idempotent processing = exactly-once effect.

Exactly-once delivery is impossible over an unreliable network. Exactly-once effect is something you construct.

Write-ahead logging (ARIES)

Force the log record to stable storage before the data page. Each record has a monotonic LSN; each page stores its last LSN. Recover in three passes: Analysis → Redo (repeat history) → Undo. Redo guarded by page-LSN is idempotent → recovery is restartable.

Deterministic replay rules

Inside workflow code, never:

# ✗ break replay:
now()      # wall clock
rand()     # randomness
db.read()  # direct external I/O

# ✓ instead:
activity(...)      # side effects → recorded
engine_time()    # engine records it
timer(24*3600) # durable, logged wait

Changing code under live histories? Use versioning / patching so old histories replay the old path.

Idempotency key (Stripe pattern)

if key in receipts:
    return receipts[key]  # replay outcome
else:
    r = do_work()
    receipts[key] = r      # SAME txn
    return r

Key + effect commit atomically, or you just move the double-apply window. Prefer naturally-idempotent ops (set, not add).

Sagas & compensation

Long-lived txn = T₁…Tₙ each with compensation C₁…Cₙ. On failure, run compensations in reverse order. Compensation is semantic undo (a refund, not a rollback) — an acceptable approximation, not the exact prior bytes. Compensations must be idempotent & retryable, or the saga gets stuck half-undone. You trade isolation for the ability to span services (Garcia-Molina & Salem, 1987).

The 30-second summary

Name the failure. Write the intention to a durable log before acting. Replay the log to rebuild a dead program's state — keeping the code deterministic. Make side effects idempotent so at-least-once becomes exactly-once in effect. Make waits durable timers. When you can't go forward, run a saga and compensate in reverse. And always know exactly what is promised — and what is yours to build.
