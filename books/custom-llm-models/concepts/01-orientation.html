From Reader to Practitioner

You asked for a book that takes you from "I want to understand custom LLMs" to "I can train one and run it myself." That is exactly the arc — and it starts with being honest about what you're actually going to own.

Here's the promise, stated plainly. By the end of this book you should be able to take an open-weights model — a Llama or a Qwen you download for free — teach it to do your job in your voice, and serve it from a machine you control, answering requests through the same API shape the big providers use. No magic, no black box you have to trust on faith. Just weights, a GPU, and a handful of ideas that turn out to be far more approachable than the jargon suggests.

The reason this is worth your time: an API you call is a rental. A model you fine-tune and serve is an asset. It runs on your hardware, keeps your data on-premises, costs the same whether you send it ten requests or ten million, and behaves exactly the way you trained it to — not the way a vendor's next release decides it should.

🎯
The four things you'll be able to do

1. Explain end-to-end what happens between a prompt and a generated token. 2. Reason precisely about the two scarce resources — GPU memory and the KV cache. 3. Fine-tune a real model on a single GPU with QLoRA. 4. Serve your custom model behind an OpenAI-compatible endpoint.

The one project we'll carry the whole way

Abstract ideas slide off the brain. So every concept in this book earns its keep against a single concrete project — the kind you might actually be handed at work:

🧪
The Acme Support Copilot

Acme has ten thousand resolved support tickets. We want a model that, given a new ticket, drafts a reply in Acme's calm house voice and tags the ticket (billing, bug, how-to, …). It must run privately — the tickets never leave Acme's servers — and cheaply enough to sit behind the support queue all day.

Watch how every chapter cashes out against that goal:

  • How inference works (2–3)why replies stream
  • KV cache (4)how many tickets at once
  • Quantization (5)fit Qwen on one GPU
  • LoRA / QLoRA (6–7)teach the house voice
  • Serving (8–10)put it behind the queue
🧭
Who this is for

You can read code and you've used an LLM, but the internals are fuzzy. You don't need a maths degree — where a formula matters (there's really only one that does), we'll work a concrete number so you can reproduce it. Turn the page and let's open up the box.
