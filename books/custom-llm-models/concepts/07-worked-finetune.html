Concept 07 · Hands-On

A Worked Fine-Tune

Enough theory. Let's QLoRA-fine-tune Qwen into Acme's Support Copilot, start to finish, on one 24 GB GPU. Every step below is a real API you can run.

Step 1 — Shape the data as chat

SFT data is (prompt, response) pairs. The modern format is a list of chat messages — a system instruction, the user's ticket, and the ideal assistant reply. One training example:

jsonl · one Acme training row{"messages": [
  {"role": "system",
   "content": "You are Acme's support assistant. Reply calmly and
              tag the ticket."},
  {"role": "user",
   "content": "I was charged twice this month!"},
  {"role": "assistant",
   "content": "I'm sorry for the scare — I can see the duplicate
              charge and I've refunded it... [tag: billing]"}
]}
⚠️
Chat templates are not optional

Every model formats conversations with its own special tokens — Qwen uses ChatML (<|im_start|>…<|im_end|>); Llama 3 uses <|start_header_id|>…<|eot_id|>. Train with the wrong template and you get a quietly broken model. The good news: TRL applies the model's template automatically when you hand it conversational data. Mismatched templates are the #1 cause of botched fine-tunes.

Step 2 — Configure the 4-bit base and the adapters

Two config objects carry everything from Chapter 6: BitsAndBytesConfig puts the base in 4-bit NF4, and LoraConfig defines the adapters.

python · QLoRA configsimport torch
from transformers import BitsAndBytesConfig
from peft import LoraConfig

# frozen base weights in 4-bit NF4 (QLoRA)
bnb = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.bfloat16,
    bnb_4bit_use_double_quant=True,
)

# trainable low-rank adapters (bf16) on top
lora = LoraConfig(
    r=16,
    lora_alpha=32,          # effective scale = alpha / r
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM",
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj",
                    "gate_proj", "up_proj", "down_proj"],
)
🔧
Reading the knobs

r=16 with alpha=32 gives an effective scale of 2 — a common, sane starting point. The target_modules list attaches adapters to both attention and MLP projections, which tends to learn richer behaviour than attention-only.

Step 3 — Train with TRL's SFTTrainer

SFTTrainer wires the model, the dataset, and the LoRA config together. Pass the adapters via peft_config and it wraps the model for you — no manual get_peft_model.

python · the training runfrom datasets import load_dataset
from trl import SFTTrainer, SFTConfig

model_id = "Qwen/Qwen3-8B"
data = load_dataset("json",
    data_files="acme_tickets.jsonl", split="train")

trainer = SFTTrainer(
    model=model_id,
    train_dataset=data,
    peft_config=lora,
    args=SFTConfig(
        output_dir="acme-copilot",
        model_init_kwargs={"quantization_config": bnb,
                           "dtype": torch.bfloat16},
        per_device_train_batch_size=2,
        gradient_accumulation_steps=8,
        learning_rate=1e-4,   # adapters like a higher LR
        num_train_epochs=1,
        max_length=2048,
        packing=True,
        # assistant_only_loss=True  # train on replies only
    ),
)
trainer.train()
⚠️
These APIs move fast

This targets the current TRL (note it's max_length, not the old max_seq_length; hyperparameters live on SFTConfig). Pin your versions and re-check field names against the installed library — the shape is stable, the exact names drift.

Step 4 — Merge for serving

Training leaves you with a small adapter. To serve it as a normal model, fold ΔW back into the base with merge_and_unload() and save standard weights:

python · bake the adapter infrom peft import AutoPeftModelForCausalLM

merged = (AutoPeftModelForCausalLM
          .from_pretrained("acme-copilot")
          .merge_and_unload())
merged.save_pretrained("acme-copilot-merged")
# now a plain HF model — feed it to vLLM, TGI, or GGUF
🔑
Merge, or don't — your call

Merge → one self-contained model, simplest to serve and quantize. Don't merge → keep the tiny adapter and load it at serve time (vLLM's --enable-lora, Ch. 8), which lets you hot-swap many adapters on one base. Acme has one behaviour, so we merge.

⚡
Don't want to hand-roll it?

Unsloth wraps this exact stack with hand-tuned kernels — up to ~2× faster training and big VRAM savings, with ready Colab notebooks. Axolotl drives the whole run from a YAML config, no Python. Both sit on top of TRL/PEFT/bitsandbytes; learn the primitives here, reach for these for speed.

Acme now has a model

acme-copilot-merged/ is a real Qwen that drafts in Acme's voice and tags tickets. It exists as files on disk. The last job: serve it — fast, for everyone at once.
