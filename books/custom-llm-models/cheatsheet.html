Commands you'll actually type

Serve (merged model)

vllm serve ./my-model \
  --gpu-memory-utilization 0.9 \
  --max-model-len 8192

PagedAttention is always on (no flag). Default mem-util 0.9.

Serve (LoRA adapter)

vllm serve BASE \
  --enable-lora \
  --lora-modules name=/path

Adapter name → the model field in requests.

🎯
Decide first: fine-tune or RAG?

Fine-tune = change behaviour/style/format (closed-book). RAG = inject fresh facts (open-book). Mature systems do both.

🧭
Pick a base

Qwen3 — Apache-2.0 across sizes, fine ladder from 0.6B. Llama 3.x — biggest ecosystem, Community License (check the MAU clause). Same QLoRA recipe; only model_id + chat template differ.

⚠️
Top foot-guns

Wrong chat template → broken fine-tune. Over-large --max-model-len → few concurrent seats. Skipping load-tests at peak → surprise TTFT spikes. "Open weights" ≠ "open source" — read the license.

Grounded in the LoRA, QLoRA, GQA, AWQ, and PagedAttention papers and the current TRL/PEFT/vLLM docs. Pin your library versions — APIs drift, the concepts don't.
