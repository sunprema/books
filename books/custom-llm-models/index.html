Before you turn the page

You already use language models. This book is about the layer underneath — the one where you take an open-weights model, bend it to your problem, and run it yourself, on your own hardware, answering to no one's rate limit.

By the last page you will be able to do four concrete things: (1) explain, without hand-waving, what happens between a prompt going in and a token coming out; (2) reason about the two resources that dominate everything — GPU memory and the KV cache; (3) fine-tune a model like Qwen or Llama on a single GPU with QLoRA; and (4) serve that custom model behind an OpenAI-compatible endpoint with vLLM.

🧭
How this book is built

Chapters 2–5 are the mental model: how a model runs, why generation is the way it is, and the two ideas — the KV cache and quantization — that govern what fits on your card. Chapters 6–7 are custom training: LoRA/QLoRA in theory, then a full worked fine-tune. Chapters 8–10 are serving: inference servers, production trade-offs, and the end-to-end ship-it pipeline.

About this book

Researched from primary sources: the LoRA and QLoRA papers, the GQA and AWQ papers, the vLLM / PagedAttention work, and the current HuggingFace TRL / PEFT docs, plus the Llama and Qwen model cards. Verified against the model generations current in 2026 — Llama 3.1/3.2/3.3 and Qwen2.5 / Qwen3. Code targets the transformers + TRL + PEFT + bitsandbytes stack; pin your versions, because these APIs move fast.

🧪
The running example

We'll carry one concrete project through the whole book: the "Acme Support Copilot" — take an open model, teach it to draft support replies in Acme's house voice and tag each ticket, then serve it privately behind an API. Every abstract idea (KV cache, quantization, LoRA, batching) will cash out against that one goal: a custom model you actually run.
