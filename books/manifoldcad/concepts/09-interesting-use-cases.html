Chapter Nine

Interesting Use Cases

Let's close with the question we opened with, now that we can answer it properly: why reach for a code-driven CAD tool at all? Because some jobs are miserable with a mouse and delightful with a for loop. Here are the places where ManifoldCAD — and the Manifold engine underneath it — genuinely earns its keep.

1 · Parametric printable parts

This is the killer app, so let's start here. Wrap a design in a function and every dimension becomes a knob. One definition yields a whole family of parts — resize it for a different phone, a thicker wall, more vents — and each variant is still a guaranteed-watertight, printable solid. We've been building toward this the whole book; here it pays off.

function ventedBox(w, d, h, wall = 2, slots = 6) {
  const outer = Manifold.cube([w, d, h], true);
  const inner = Manifold.cube([w - 2*wall, d - 2*wall, h], true).translate([0, 0, wall]);
  let box = outer.subtract(inner);
  for (let i = 0; i < slots; i++) {
    const x = -w/2 + wall + (i + 0.5) * (w - 2*wall) / slots;
    box = box.subtract(Manifold.cube([1.5, d + 2, h*0.5]).translate([x, 0, h*0.3]));
  }
  return box;
}
export default ventedBox(60, 40, 25);   // change 4 numbers -> a new enclosure
Version-controlled hardware

Here's a consequence that's easy to underrate: because the part is source code, it lives in Git. We can diff a design change, review it in a pull request, tag a release. Try doing that with a binary CAD file. The whole software workflow we already trust comes along for free.

2 · Generative & mathematical art

Shapes defined by equations are trivial here and painful elsewhere — the exact inversion of what a mouse is good at. The Torus Knot (a warped revolve), the infinite Gyroid minimal surface (a level-set from one formula), and the Menger Sponge fractal (recursion plus coplanar subtractions) all fall out of a few dozen lines. Jewelry, lamps, sculptures, math-viz: same engine, different formula.

3 · Gears, threads & mechanisms

Anything defined by exact geometry benefits from code. The editor ships an Involute Gear Library that computes true involute tooth profiles from module and tooth count — precisely the kind of thing that's error-prone by hand but a straightforward formula in a script. Screw threads, cams, pulleys, lattices: same story, every time. When the math is known, let the math build the part.

4 · An embeddable geometry kernel

Here's the part that surprises people. The editor is only the friendly face — underneath it is manifold-3d, a small, fast WebAssembly library we can drop into our own software. Two consequences worth knowing:

  • In the browser. Because it's WASM with rock-solid booleans, it powers in-page product configurators and web CAD tools — "customise your bracket, see it update live" — with no server round-trip.
  • As a CAD backend. Manifold was adopted as a mesh backend by OpenSCAD, where its robust, fast booleans dramatically sped up rendering — a strong vote of confidence from the programmatic-CAD world.
Same code, two homes

A model we prototype in the ManifoldCAD editor uses the very same API as the npm package. So we prototype in the browser tool, then lift the logic into an app — the cube/subtract/extrude calls don't change. The editor is a REPL for geometry; the library is production.

5 · Mass customisation

When we need a thousand variants — personalised nameplates, made-to-measure enclosures, dental or jewelry blanks — a script loops over a data file and emits one valid, exportable solid per row. Design once, generate forever. This is the point where "a part is a function" stops being a slogan and starts being a factory.

Key idea

Reach for ManifoldCAD when a design is parametric, mathematical, repetitive, or embedded — anywhere "describe it with code" beats "push it with a mouse." That turns out to be a surprisingly large slice of real-world CAD, which is exactly why this small, watertight engine is worth carrying in your toolkit.

So what did we actually gain, over nine chapters? Not just a tool, but a stance: geometry as software. Valid solids, diffable source, an offline workshop, and a straight path from a browser experiment to a shipped product. Now go build something and print it.

Challenges
