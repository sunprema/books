Chapter Nine

Interesting Use Cases

Why reach for a code-driven CAD tool at all? Because some jobs are miserable with a mouse and delightful with a for loop. Here are the places ManifoldCAD — and the Manifold engine underneath it — genuinely shine.

1 · Parametric printable parts

The killer app. Wrap a design in a function and every dimension becomes a knob. One definition yields a whole family of parts — resize it for a different phone, a thicker wall, more vents — and each variant is still a guaranteed-watertight, printable solid.

function ventedBox(w, d, h, wall = 2, slots = 6) {
  const outer = Manifold.cube([w, d, h], true);
  const inner = Manifold.cube([w - 2*wall, d - 2*wall, h], true).translate([0, 0, wall]);
  let box = outer.subtract(inner);
  for (let i = 0; i < slots; i++) {
    const x = -w/2 + wall + (i + 0.5) * (w - 2*wall) / slots;
    box = box.subtract(Manifold.cube([1.5, d + 2, h*0.5]).translate([x, 0, h*0.3]));
  }
  return box;
}
export default ventedBox(60, 40, 25);   // change 4 numbers -> a new enclosure
Version-controlled hardware

Because the part is source code, it lives in Git: diff a design change, review it in a pull request, tag a release. Try doing that with a binary CAD file.

2 · Generative & mathematical art

Shapes defined by equations are trivial here and painful elsewhere: the Torus Knot (a warped revolve), the infinite Gyroid minimal surface (a level-set from one formula), and the Menger Sponge fractal (recursion plus coplanar subtractions). Jewelry, lamps, sculptures, and math-viz all fall out of a few dozen lines.

3 · Gears, threads & mechanisms

Anything defined by exact geometry benefits from code. The editor ships an Involute Gear Library that computes true involute tooth profiles from module and tooth count — the kind of precision that's error-prone by hand but a formula in script. Screw threads, cams, pulleys, and lattices are the same story.

4 · An embeddable geometry kernel

The editor is only the friendly face. Underneath is manifold-3d, a small, fast WebAssembly library you can drop into your own software. Two consequences worth knowing:

  • In the browser. Because it's WASM with rock-solid booleans, it powers in-page product configurators and web CAD tools — "customise your bracket, see it update live" — with no server round-trip.
  • As a CAD backend. Manifold was adopted as a mesh backend by OpenSCAD, where its robust, fast booleans dramatically sped up rendering — a strong vote of confidence from the programmatic-CAD world.
Same code, two homes

A model you prototype in the ManifoldCAD editor uses the very same API as the npm package. Prototype in the browser tool, then lift the logic into an app — the cube/subtract/extrude calls don't change.

5 · Mass customisation

When you need a thousand variants — personalised nameplates, made-to-measure enclosures, dental or jewelry blanks — a script loops over a data file and emits one valid, exportable solid per row. Design once, generate forever.

Key idea

Reach for ManifoldCAD when a design is parametric, mathematical, repetitive, or embedded — anywhere "describe it with code" beats "push it with a mouse." That's a surprisingly large slice of real-world CAD.

Sources: editor "Involute Gear Library", "Gyroid Module", "Torus Knot", "Menger Sponge" examples; Manifold library (elalish/manifold) & its adoption as an OpenSCAD backend.
