Chapter Four

Moving Things Around

A primitive is born at the origin. To build anything real you'll translate it into place, rotate it, and sometimes scale or mirror it. Every transform returns a new solid, so you chain them like a pipeline.

const arm = Manifold.cube([40, 6, 6])
  .translate([0, -3, 0])        // recentre on Y
  .rotate([0, 0, 30])           // spin 30 degrees about Z
  .translate([0, 0, 10]);       // lift 10mm up
Rotations are in DEGREES

Unlike almost every other graphics API, ManifoldCAD's rotate takes degrees, not radians — and applies them as Euler angles in X, then Y, then Z order. Write rotate([0, 0, 90]) for a quarter turn about the vertical axis. No Math.PI needed.

The four movers

MethodEffect
translate([x, y, z])Slide along each axis (mm). Also accepts separate args: translate(x, y, z).
rotate([x, y, z])Rotate by Euler angles in degrees, applied X→Y→Z.
scale(s)Resize. A single number scales uniformly; [x, y, z] scales per axis.
mirror([nx, ny, nz])Reflect across the plane through the origin with that normal — perfect for left/right symmetric parts.

There's also transform(matrix) if you already have a 3×4 affine Mat4, and warp(fn) for bending every vertex by a function — we'll meet warp in the "Smooth & Organic" chapter.

Immutable & chainable

Transforms never mutate the shape you call them on — they hand back a new one. That makes reuse trivial: build a part once, then place copies.

const post = Manifold.cylinder(50, 3);

// Four corner posts from one definition:
const posts = Manifold.union([
  post.translate([  0,   0, 0]),
  post.translate([100,   0, 0]),
  post.translate([  0, 100, 0]),
  post.translate([100, 100, 0]),
]);
Order matters

.rotate(...).translate(...) is not the same as .translate(...).rotate(...). Each transform is relative to the shape's current position, and rotation always pivots about the origin — so rotate first, then translate into place, is usually what you want.

This is exactly how the built-in Rounded Frame example works: it builds a single rounded edge (a cylinder capped with a sphere), then rotates and unions copies of it into a full 12-edge wireframe cube. One part, transformed many ways.

Key idea

Model a piece once at the origin, then use transforms to stamp it across your design. Symmetric and repetitive parts almost write themselves this way.

Sources: translate/rotate/scale/mirror/transform in the Manifold class reference; editor "Rounded Frame" example.
