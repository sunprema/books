Chapter One

What Is ManifoldCAD?

CAD means computer-aided design — drawing the exact 3D shape of a physical object so a machine can make it. ManifoldCAD is CAD where you don't push a mouse around a model; you write a short program, and the program is the model. If you can write a function, you can design a part.

Most CAD tools you may have heard of — Fusion 360, SolidWorks, Blender — are driven by hand: you click, drag, and extrude with a cursor. That's powerful, but it's hard to version, hard to parameterize, and impossible to git diff. ManifoldCAD belongs to the other family — programmatic CAD, alongside OpenSCAD — where a model is described by code. Change a number, press run, and the shape updates.

The one mental model

You start from simple solids (a cube, a sphere, a cylinder), then combine them — glue two together, carve one out of another, keep only where they overlap. That's it. Everything else is variations on "make shapes, then add and subtract them."

What "Manifold" means (and why it matters)

The tool is named after the thing it guarantees. A manifold mesh is a surface that is completely watertight — every edge is shared by exactly two triangles, so the surface cleanly separates a solid inside from the outside, with no holes, cracks, or dangling faces.

This is not academic. A 3D printer, a CNC mill, or a physics simulator all need to know, for every point in space, "is this inside the object or not?" An open or self-intersecting mesh — the bane of hand-modelling — has no answer. ManifoldCAD's whole engine is built so that every operation produces a valid solid, which is exactly why boolean operations (the add/subtract you'll lean on constantly) are so reliable here.

The editor is a local app

Go to manifoldcad.org and you get an editor with a code pane on the left and a live 3D view on the right. A few things worth knowing on day one:

  • It's a PWA (progressive web app). After the first load it works fully offline — there is no server, and your code never leaves your machine.
  • Your script is auto-saved in the browser between sessions.
  • The built-in examples menu (top drop-down) is the fastest way to learn — every shape in this book maps to one.
  • When you're happy, export to .glb (a 3D scene) or .3mf (a print file).

Two rules that will save you an hour

Z is up, and units are millimetres

ManifoldCAD uses a right-handed coordinate system with +Z as up and lengths measured in millimetres. So cube(20) is a 2 cm block, and translating [0, 0, 50] lifts a shape 5 cm off the build plate. (glTF, the export format, treats +Y as up — the exporter handles that conversion for you.)

You export your model — you don't "draw" it

A script's job is to produce one final shape and hand it back with export default. Whatever you export — a Manifold, a scene node, or an array of them — is what gets rendered and saved. You'll see this on the very next page.

That's the whole worldview: solids that stay valid, described by code that stays diffable, in an editor that stays on your machine. Turn the page and let's make something.
