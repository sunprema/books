Chapter Two

Your First Solid

Every good tutorial earns trust by getting something on the screen fast, so let's do that first and ask questions second. Here is a complete, runnable ManifoldCAD script — all four lines of it. Paste it into the editor at manifoldcad.org and you'll see a box appear.

import {Manifold} from 'manifold-3d/manifoldCAD';

const box = Manifold.cube([30, 20, 10], true);

export default box;

Three ideas are doing all the work here, and we'll use every one of them in every script we ever write. Let's name them before we move on:

  1. Import the Manifold class from the built-in module.
  2. Call a constructor (Manifold.cube) to make a solid.
  3. Export that solid as the default — that's the one thing that gets rendered.
A small ergonomics win: destructure the constructors

Typing Manifold. over and over gets old fast. Most examples pull the ones they need into locals: const {cube, sphere, cylinder} = Manifold; — then we just call cube(...). It's a tiny thing, but readable code is code we'll actually maintain.

The four primitives

Every model we build starts from these blocks, so it's worth learning their signatures precisely — the defaults are where the surprises hide.

ConstructorMakes
cube(size?, center?)A box. size is a number (a cube) or [x, y, z]. By default it sits in the first octant (a corner at the origin); pass center=true to centre it.
sphere(radius, segments?)A sphere centred at the origin. segments sets the facet count — its smoothness.
cylinder(height, radiusLow, radiusHigh?, segments?, center?)A cylinder — or a cone if the two radii differ. Handy trick: pass radiusHigh = -1 to mean "same as radiusLow".
tetrahedron()A unit tetrahedron — the simplest closed solid, useful for tests.
 
const {cube, sphere, cylinder} = Manifold;

const block = cube(20);                 // 20mm cube, corner at origin
const ball  = sphere(12, 64);           // radius 12, 64 segments -> smooth
const rod   = cylinder(30, 5);          // height 30, radius 5
const cone  = cylinder(30, 10, 0);      // radiusHigh 0 -> a point -> a cone
const funnel = cylinder(20, 6, 12);     // radiusLow 6, radiusHigh 12 -> a frustum

Resolution: how round is round?

Here's a subtlety worth pausing on. A sphere on a computer isn't really round — it's a many-sided polyhedron wearing a disguise. The segments argument controls how many facets it has, and that's a classic trade-off: more segments give a smoother surface but a bigger, slower mesh. When we leave it out, Manifold picks a sensible default from the shape's size — and we can tune that default globally.

// Set the global level-of-detail once, at the top of the script.
setMinCircularAngle(3);         // max 3 degrees between facets
setMinCircularEdgeLength(0.5);  // no facet edge shorter than 0.5mm
// or force an exact count for every curved primitive:
setCircularSegments(64);

const smooth = sphere(20);      // now honours the settings above
Key idea

Set resolution globally at the top, not per-shape. Then a single knob controls the trade-off between a fast, chunky preview and a slow, glass-smooth final export — across the whole model at once. This is the same instinct as a compile-time flag: one place to turn, everything downstream responds.

The edge case that bites everyone once

A script with no export default renders nothing — the number-one "why is my screen blank?" mistake. Whatever we want to see, it has to be the thing we export. The editor is doing exactly what we told it: nothing.

So what did we gain? A vocabulary of four solids and a single global dial for smoothness. That's genuinely enough to make real objects — but a pile of primitives isn't a part yet. Next we do the operation that turns them into one: combining them.
