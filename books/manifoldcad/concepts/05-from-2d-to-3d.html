Chapter Five

From 2D to 3D

Let's notice a pattern in real objects: a surprising number of them are just a flat outline pushed into the third dimension. A gear is a toothed disc, extruded. A bottle is a silhouette, spun. When we spot that, the honest move is to design in 2D first and lift into 3D — it's simpler, and as we'll see, just as robust.

The CrossSection class

Meet CrossSection, the 2D cousin of Manifold: a set of closed polygons with no self-intersections, guaranteed valid. That guarantee is the same invariant we trusted in 3D, now in the plane — so 2D booleans are exactly as reliable as their 3D counterparts. It has its own primitives and its own transforms.

import {CrossSection, Manifold} from 'manifold-3d/manifoldCAD';
const {square, circle} = CrossSection;

const plate = square([40, 40], true)     // 40x40, centred
  .subtract(circle(6).translate([12, 12]))   // punch a hole
  .subtract(circle(6).translate([-12, -12]));
2D constructor / methodMeaning
CrossSection.square(size?, center?)A rectangle; size is a number or [w, h].
CrossSection.circle(radius, segments?)A filled circle.
.offset(delta, joinType?, miterLimit?, segments?)Grow (or shrink, if negative) the outline — how we get rounded corners and even wall thicknesses.
.add / .subtract / .intersectThe same booleans as in 3D, in the plane.

We can also skip the class entirely and hand a raw array of [x, y] points straight to extrude — a "simple polygon." The stretchy-bracelet example builds its profile exactly this way, which is a nice reminder that the fancy class is a convenience, not a requirement.

Extrude: push a profile upward

// extrude(polygons, height, nDivisions?, twistDegrees?, scaleTop?, center?)
const prism = Manifold.extrude(plate, 10);          // 10mm tall

// A twisted, tapering column — nDivisions gives the twist somewhere to bend:
const drill = Manifold.extrude(square([20, 20], true), 60, 40, 180, 0.3);
//                                          height ^   ^div ^twist ^scaleTop

Those extra arguments unlock a lot, so let's read them like the knobs they are: nDivisions slices the height so a twist has somewhere to bend smoothly, twistDegrees spirals the profile as it rises, and scaleTop shrinks (or grows) the top face to make tapers and pyramids. One function, a whole family of shapes.

Revolve: spin a profile around the axis

Place a profile out to one side of the origin and revolve sweeps it around the Y axis — vases, bowls, pulleys, ring gaskets all fall out of this. Give it a partial angle and you get a wedge instead of a full ring.

// revolve(polygons, circularSegments?, revolveDegrees?)
const profile = CrossSection.circle(4).translate([20, 0]);
const torus   = Manifold.revolve(profile, 64);        // full ring
const scoop   = Manifold.revolve(profile, 64, 270);   // 3/4 sweep
Going the other way: 3D → 2D

The bridge runs both directions. Two Manifold methods return a CrossSection: slice(z) cuts a flat section at a given height, and project() flattens the whole solid onto the X-Y plane (its shadow). The auger example extrudes bead.slice(0) — it slices a 3D bead and re-extrudes that outline. Handy when we need a matching base plate or a 2D drawing of something we built in 3D.

Key idea

If a shape has a natural profile — a constant cross-section, or a silhouette spun around an axis — reach for 2D first. Designing in the plane and lifting into 3D is simpler and more robust than wrestling the full solid. It's the geometric version of solving the easy sub-problem and letting a transform do the rest.

So what did we gain? A second modelling style that lives alongside booleans rather than replacing them — most real parts mix both. We now have primitives, booleans, transforms, and 2D-to-3D. That's enough to build almost anything mechanical. Next chapter we step back and map the entire API in one guided read.
