Chapter Three

Boolean Modeling

If Chapter Two gave us the nouns, this chapter gives us the verbs — and honestly, it's the heart of the whole tool. We take two solids and combine them three ways: glue them together, carve one out of the other, or keep only the overlap. Programmers call this CSG — Constructive Solid Geometry — and nearly every ManifoldCAD model is built out of it.

import {Manifold} from 'manifold-3d/manifoldCAD';
const {cube, cylinder} = Manifold;

const block = cube([40, 40, 20], true);
const hole  = cylinder(20, 8, -1, 64, true);

// Carve the cylinder out of the block:
export default block.subtract(hole);

That's a rounded hole drilled clean through a block — the "hello world" of CAD. Three lines of shapes, one boolean, and a part you could print. Notice how little ceremony there is; that economy is the point.

Three operations, two ways to write each

Every boolean comes in two forms, and it's worth knowing why both exist. There's an instance method (nice for chaining off one shape) and a static function (nice for combining a whole list). They compute the same result — the choice is purely about which reads better at the call site.

OperationMethodStaticResult
Uniona.add(b)Manifold.union(a, b)everything in either shape
Differencea.subtract(b)Manifold.difference(a, b)a with b removed
Intersectiona.intersect(b)Manifold.intersection(a, b)only where they overlap

Combining a whole list at once

The static forms accept an array, and this is the clean way to merge or carve many shapes. It reads far better than a chain of .add().add().add() — and it lets us build shapes in a loop, which is where code-driven CAD starts to feel like a superpower.

const pillars = [];
for (let i = 0; i < 4; i++) {
  const x = (i % 2) * 30, y = Math.floor(i / 2) * 30;
  pillars.push(cylinder(40, 4).translate([x, y, 0]));
}
const legs = Manifold.union(pillars);          // one solid from many

// Difference with a list subtracts them all from the first:
const plate = Manifold.difference([slab, boltHole1, boltHole2, boltHole3]);
Coplanar faces just work — and that's not free

Here's a classic CAD headache: subtract a shape whose face lands exactly on another face, and the math goes ambiguous — lesser tools produce cracks. Manifold resolves these ties deterministically with symbolic perturbation, so coplanar cuts (the Menger-sponge example is built entirely from them) come out clean. We know that at every step the result is still a valid solid, so we can stack booleans without fear.

A two-for-one: split

Sometimes we want both the carved result and the piece we carved away — a part and its mating cavity, say. Doing a subtract and then an intersect separately works, but it's wasteful. split returns both halves in a single pass.

// intersection first, then difference — in a single operation
const [inside, outside] = result.split(cube(100, true));
Key idea

Design by addition and subtraction. A bracket is a block minus some holes minus a slot. A knob is a cylinder plus a grip texture minus a shaft. Once we see parts as booleans of primitives, modelling becomes arithmetic — and arithmetic is something we already know how to compose.

So what did we gain? Three verbs and the confidence to chain them arbitrarily deep. Booleans decide which material exists. Next we handle where it sits — moving, spinning, and scaling our shapes into place.
