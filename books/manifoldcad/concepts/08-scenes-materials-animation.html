Chapter Eight

Scenes, Materials & Animation

Up to now every script exported one solid. Real deliverables have several parts in different colours, arranged in a hierarchy, sometimes even animated. That's what GLTFNode is for — a scene node you export instead of a bare Manifold.

Why not just union everything?

If two parts don't actually touch — a lid and a jar, two puzzle pieces — unioning them fuses them into one nameless blob in the exported file. Wrapping each in its own GLTFNode keeps them distinct, individually coloured, and cleanly organised in the output .glb.

import {GLTFNode, Manifold} from 'manifold-3d/manifoldCAD';

const node = new GLTFNode();
node.manifold = Manifold.cube([20, 20, 20], true);
node.material = { baseColorFactor: [0.2, 0.6, 1.0], metallic: 0 };

export default node;
GLTFNode fieldMeaning
manifoldThe solid this node draws.
materialA glTF PBR material — see below.
translation / rotation / scaleThe node's own transform, relative to its parent.
new GLTFNode(parent)Pass a parent node to build a hierarchy.

Building a hierarchy

Pass a parent to the constructor and nodes form a tree; a parent's transform carries its children with it. The Rounded Frame example does exactly this — a root node holding an "inside" and "outside" node, the inside tinted cyan:

const root = new GLTFNode();
root.translation = [0, 0, 60];

const outsideNode = new GLTFNode(root);   // child of root
outsideNode.manifold = outside;

const insideNode = new GLTFNode(root);
insideNode.manifold = inside;
insideNode.material = { baseColorFactor: [0, 1, 1] };

export default getGLTFNodes();            // export the whole tree
Two ways to hand back a scene

You can export default a single node (its descendants come along), an array of nodes, or call getGLTFNodes() to collect every node you've created. Any of them works.

Materials: the glTF metallic-roughness model

Materials follow the standard glTF PBR model, so they render consistently anywhere a .glb opens.

FieldMeaning
baseColorFactor: [r, g, b]Base colour, each channel 0–1.
metallic: 0–10 = plastic/matte, 1 = metal.
roughness: 0–10 = mirror-smooth, 1 = fully diffuse.
attributes: ['COLOR_0']Tell the exporter to read per-vertex colours set via setProperties.

That last one is the bridge to the previous chapter: the Menger Sponge and Scallop examples set a vertex colour per point, then declare attributes: ['COLOR_0'] so the gradient survives into the exported model.

Animation with morphs

Remember warp? If you register the same function as a morph target, the viewer interpolates from the un-warped shape to the warped one over time — an animation, right in the browser.

import {GLTFNode, Manifold, setMorphStart} from 'manifold-3d/manifoldCAD';

const ball = Manifold.sphere(1, 200);
setMorphStart(ball, func);        // animate FROM this warp toward the final shape

const node = new GLTFNode();
node.manifold = ball;

setAnimationDuration(5);          // seconds
setAnimationMode('ping-pong');    // loop back and forth
export default node;

This is the beating (literally) heart of the Heart example. Use setMorphStart/setMorphEnd to pin the warp functions for the first and last frames, and the global setAnimationDuration/setAnimationMode to control playback.

Key idea & export

When your model is more than one solid, think in nodes: each part gets its own colour, transform, and place in the tree. Then export to .glb for a full-colour animated scene, or .3mf when you're sending it to a printer.

Sources: GLTFNode, GLTFMaterial, setMorphStart/End, getGLTFNodes in the user guide; editor "Rounded Frame", "Menger Sponge" & "Heart" examples.
