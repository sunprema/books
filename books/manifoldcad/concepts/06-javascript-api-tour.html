Chapter Six · The Reference

The JavaScript API Tour

We've been learning the API one verb at a time. Now let's pull back and see the whole map at once — because a small API is one you can actually hold in your head, and this one is deliberately small. This is a guided read of the official user guide at manifoldcad.org/docs/jsuser: the module we import from, the two classes, the free-standing helpers, and the query methods that answer "what did I just make?"

The shape of a script

Every ManifoldCAD script follows the same skeleton, and it's worth internalizing so the structure fades into the background. We import from the manifold-3d/manifoldCAD module, build, and export default the result.

import {Manifold, CrossSection, GLTFNode, show, only} from 'manifold-3d/manifoldCAD';

// ... build your model ...

export default result;   // Manifold | GLTFNode | array | function
What we're allowed to export

The default export can be a single Manifold, a GLTFNode (a scene node — next chapter), an array of either, or even a function that returns one of those. That last form is quietly powerful: it lets the editor call our model with different parameters, which is the seed of parametric design.

The two classes

Almost everything hangs off these two, so if you remember nothing else, remember the split. Manifold is a 3D solid; CrossSection is a 2D region. Both are immutable — every method returns a new object, exactly as we saw with transforms.

Free-standing (module) functions

These live on the module, but the editor also exposes them bare, so we can call them without a prefix. Think of them as global settings and debugging aids rather than shape operations.

FunctionUse
setMinCircularAngle(deg)Global smoothness: max angle between facets on a curve.
setMinCircularEdgeLength(mm)Global smoothness: shortest allowed facet edge.
setCircularSegments(n)Force an exact segment count for every curved primitive.
getCircularSegments(radius)Ask what count the current settings imply for a radius.
show(shape)Debug: render shape and its copies in transparent red (like # in OpenSCAD).
only(shape)Debug: show only this, ghosting the rest in grey (like * in OpenSCAD).
setMorphStart / setMorphEndAttach warp functions for animation (next chapter).
Two debug helpers we'll actually reach for

When a subtract "does nothing," the tool is almost always in the wrong place. Wrap the cutting tool in show(tool) and it renders in red even when it isn't touching the result — so we can see the misplacement instead of guessing. only() solves the opposite problem: hunting a shape hidden inside the solid, by ghosting everything else out of the way. These are our console.log for geometry.

Asking the model questions

Because a Manifold is always a valid solid, its measurements are exact — and that's not a small thing. Exact measurements let us assert print constraints or drive generative logic with confidence.

const part = cube([20, 20, 20]).subtract(sphere(12));

console.log('volume mm^3:', part.volume());
console.log('surface mm^2:', part.surfaceArea());
console.log('triangles:', part.numTri());
console.log('holes (genus):', part.genus());   // topological handle count

const bb = part.boundingBox();                  // { min:[x,y,z], max:[x,y,z] }
const height = bb.max[2] - bb.min[2];
Floating point is still floating point

One honest limitation, stated plainly in the docs: geometry that is marginal — a wall a hair thick, faces almost-but-not-quite coplanar — can wobble under floating-point arithmetic. The fix is a design habit, not a flag: give features real thickness, and lean on Manifold's exact booleans rather than nudging shapes to "just barely" touch.

Key idea

The API is small on purpose: two classes, a handful of free functions, and everything returns a valid solid. When we're unsure of a signature, the editor's live docs (type Manifold.) are the source of truth — and they're the same reference this chapter maps.

So what did we gain? A mental index. We can now predict where a method lives before we look it up. With the full surface mapped, the last two chapters are about pushing past mechanical parts — into organic form, and into full-colour, animated scenes.

Challenges

1. Build a cube with a spherical hollow, then use genus() to confirm its topology. What number means "one tunnel through the part"?
2. Deliberately misplace a subtract so nothing is removed. Add show(tool) and fix it using only what the red ghost tells you.
3. Pick any method from the tables above that we haven't used — say hull or minkowskiSum — read its doc entry, and predict its output before running it.
