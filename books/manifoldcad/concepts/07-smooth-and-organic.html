Chapter Seven

Smooth & Organic Shapes

Cubes and cylinders get us mechanical parts. But what about hearts, knots, ripples, lattices — the shapes that look grown rather than machined? Here's the shift in thinking: instead of describing a shape, we describe a deformation, and let the engine keep the result watertight. Three tools make it happen: warp moves every vertex by a function, refine adds vertices so there's something to move, and smooth rounds a coarse mesh into flowing curves.

warp: a function applied to every vertex

Let's start with the most direct one. warp hands our callback each vertex position and lets us mutate it in place — no triangles to manage, no bookkeeping, just math on points. And because the topology doesn't change (we move vertices, we don't add or remove them), the result stays a valid solid. That's the invariant doing quiet work for us again.

// Push a flat sheet into a gentle wave along X.
const sheet = Manifold.cube([80, 80, 2], true).refine(80);

const wavy = sheet.warp((v) => {
  v[2] += 6 * Math.sin(v[0] / 10);   // v is [x, y, z]; edit it directly
});

export default wavy;
Refine before you warp — this is the gotcha

A warp can only move the vertices that already exist. A plain cube has exactly 8 of them, so bending it does nothing visible — a genuinely confusing first result. refine(n) splits every edge into n pieces first, giving the surface enough points to curve smoothly. The trade-off is the familiar one: more refinement means a smoother warp and a bigger mesh.

This is exactly how the built-in Heart example works — it starts from a high-segment sphere and warps each point outward by solving Taubin's heart equation with Newton's method. The Torus Knot revolves a circle into a tube, then warps that tube into a knot. In both, no triangle is ever touched by hand; the shape is a consequence of a formula.

smooth: round off a coarse mesh

Where warp is explicit math, Manifold.smooth takes a different tack: it interpolates a low-poly mesh into a curved one, and — crucially — lets us keep chosen edges sharp. Pair it with refine to actually generate the rounded triangles the curve implies.

// smooth(mesh, sharpenedEdges?) then refine to realise the curve
const blob = Manifold.smooth(mesh, sharpenedEdges).refine(n);

The Scallop example is the one to study here: it builds a fan of triangles by hand, marks each rib's smoothness, then calls smooth().refine(50) to turn that skeleton into a rippled shell. There's also smoothOut(minSharpAngle, minSmoothness) to round a mesh using only its own geometry, and smoothByNormals when we already have normal data.

Colour by curvature (a taste of properties)

setProperties attaches per-vertex data — most commonly vertex colours. Combined with calculateCurvature, we can paint a surface by how sharply it bends, which is both diagnostic and beautiful:

const shell = Manifold.smooth(mesh, edges)
  .refine(50)
  .calculateCurvature(-1, 0)                 // write mean curvature to property 0
  .setProperties(3, (color, pos, oldProp) => {
    const t = Math.max(0, Math.min(1, oldProp[0] / 3 + 0.5));
    color[0] = t; color[1] = 0; color[2] = 1 - t;   // blue → red
  });
Beyond meshes: level sets

For truly implicit shapes — gyroids, metaballs, blends — reach for Manifold.levelSet(sdf, bounds, edgeLength). It meshes the zero-crossing of a signed distance function we write in plain JavaScript. It's how the Gyroid Module example conjures an infinite minimal surface from one short formula: the geometry doesn't exist until the engine walks the field and finds where it crosses zero.

Key idea

Organic modelling in ManifoldCAD is procedural: describe the deformation as a function of position, give the surface enough vertices to express it, and let the engine keep the result watertight. Sculpting becomes programming — and a sculpture becomes something we can re-run, tune, and diff.

So what did we gain? A third modelling style, and with it the realization that "mechanical" and "organic" aren't different tools here — they're the same engine, pointed at different functions. One thing is still missing, though: everything we've made has been a single, single-coloured solid. The last chapter fixes that.

Challenges

1. Warp a refined cylinder so its radius pulses with height — v[0] and v[1] scaled by 1 + 0.2*sin(v[2]). What happens if you forget to refine?
2. Take the wavy sheet and double the refine count. Compare numTri() and the smoothness of the crest. Where would you stop for a printable part?
3. Write a signed distance function for a sphere and feed it to levelSet. Then blend two spheres by taking the min of their distances — you've just built a metaball.
