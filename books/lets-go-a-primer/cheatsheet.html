The Go Cheatsheet

Everything the chapters skipped on purpose: the syntax. Dense by design — this page is for looking things up, not reading start to finish.

Declaring things

var x int                // declared, zero value (0)
var x int = 5           // declared + initialized
var x = 5              // type inferred: int
x := 5                 // short form — only inside a function
const Pi = 3.14159     // compile-time constant
var a, b = 1, "two"    // multiple declarations
x, y = y, x                // swap, no temp variable needed

Control flow — there is only for

if x > 0 {
    // ...
} else if x == 0 {
    // ...
} else {
    // ...
}

if v, err := doThing(); err != nil {   // init statement scoped to the if/else
    return err
}

for i := 0; i < 10; i++ { }        // classic C-style
for condition { }                    // "while" — same keyword, no condition parens
for { }                               // infinite loop; break/return to exit
for i, v := range items { }        // index + value, for slices/arrays/strings
for k, v := range myMap { }        // key + value, for maps
for _, v := range items { }        // blank identifier discards the index

switch day {
case "Sat", "Sun":
    fmt.Println("weekend")      // no fallthrough by default (opposite of C)
default:
    fmt.Println("weekday")
}

Slices, arrays, maps

arr := [3]int{1,2,3}       // array — fixed size, part of the type
s := []int{1,2,3}          // slice literal
s := make([]int, 3)          // len 3, cap 3, zero-valued
s := make([]int, 0, 10)      // len 0, cap 10
s = append(s, 4, 5)          // always reassign the result!
sub := s[1:3]              // shares the same backing array
len(s); cap(s)             // length / capacity
dst := make([]int, len(s))
n := copy(dst, s)          // actually duplicates data, unlike re-slicing

m := map[string]int{"a":1}     // map literal
m := make(map[string]int)    // empty, ready-to-use map (nil map can't be written to)
v, ok := m["a"]            // ok is false if key absent — the standard existence check
delete(m, "a")          // remove a key, no-op if absent

Structs, methods, interfaces

type Point struct { X, Y float64 }

p := Point{X: 1, Y: 2}     // named fields (preferred)
p := Point{1, 2}            // positional — must supply every field, in order

func (p Point) Dist() float64 { ... }   // value receiver — operates on a copy
func (p *Point) Move(dx, dy float64) { p.X += dx; p.Y += dy } // pointer receiver — mutates original

type Shape interface {
    Area() float64     // satisfied implicitly — no "implements" keyword, ever
}
var s Shape = Point{}    // works as long as Point has a matching Area() method

Errors, defer, panic/recover

v, err := doThing()
if err != nil { return fmt.Errorf("context: %w", err) }  // %w wraps

errors.Is(err, os.ErrNotExist)     // true if err (or anything it wraps) equals target
var pe *os.PathError
errors.As(err, &pe)             // pulls out a specific concrete error type

defer f.Close()               // runs at function exit; multiple defers run LIFO

defer func() {
    if r := recover(); r != nil {   // only useful inside a deferred func, mid-panic
        // convert back into a normal error
    }
}()
panic("unrecoverable")      // crashes unless a deferred recover() catches it

Goroutines, channels, select

go doWork()                    // starts, doesn't block the caller

ch := make(chan int)         // unbuffered — send blocks until received
ch := make(chan int, 10)     // buffered — blocks only once full/empty
ch <- 5                     // send
v := <-ch                    // receive
v, ok := <-ch                 // ok is false once the channel is closed and drained
close(ch)                   // sender signals "no more values"; only the sender should close
for v := range ch { }         // receive until closed

select {
case v := <-ch1:
case ch2 <- x:
default:                    // makes the select non-blocking
}

var wg sync.WaitGroup
wg.Add(1); go func(){ defer wg.Done(); ... }(); wg.Wait()
var mu sync.Mutex
mu.Lock(); defer mu.Unlock()

Generics (Go 1.18+)

func Max[T int | float64](a, b T) T {
    if a > b { return a }
    return b
}
Max(3, 7)              // T inferred as int

type Stack[T any] struct { items []T }  // generic type

Common gotchas at a glance

Watch for these Unused imports/variables are compile errors, not warnings.
append's return value must be kept — append(s, x) alone can silently do nothing observable.
Writing to a nil map panics — read it, it's fine; write, use make first.
Only the sender should close a channel — closing a channel you're only receiving from, or closing twice, panics.
Exported vs. unexported is decided by capitalization, not a keyword — package-level, not per-object.
:= creates new variables — if even one name on the left is new, the rest are reused, not necessarily reassigned across scopes the way you might expect.
