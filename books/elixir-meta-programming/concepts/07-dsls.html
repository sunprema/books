DSLs: The Big Payoff

Put quoting, use, attribute accumulation, and code generation together and you get the thing Elixir is quietly famous for: domain-specific languages that read like configuration but compile to real code.

You've used these for years without calling them metaprogramming. Every one of these blocks is a macro collecting declarations and generating functions behind your back:

Ecto — a schema DSLdefmodule User do
  use Ecto.Schema

  schema "users" do
    field :name, :string
    field :age, :integer
    has_many :posts, Post
  end
end

Each field doesn't do anything at runtime — it records a declaration into module attributes, and schema's machinery generates the struct, the __schema__/1 reflection functions, and the changeset plumbing.

Phoenix — a routing DSLscope "/", MyAppWeb do
  get "/", PageController, :home
  resources "/users", UserController
end
🧭
The shared move

All of these declare intent and let a macro turn that declaration into implementation. You read what the system is, not the bookkeeping of how it's wired. That readability is the payoff metaprogramming buys.

Don't hand-roll the big ones

Writing a one-off macro is easy; writing a robust, introspectable, well-validated DSL is a lot of careful work — option schemas, good compile errors, runtime reflection. The modern Elixir answer is to not do it by hand:

ToolWhat it gives you
SparkA framework for building DSLs — entities, sections, validation, and free introspection. It's what powers every Ash DSL.
AshA whole application framework expressed as Spark DSLs: resources, actions, policies.
 
Ash — a resource DSL (built on Spark)defmodule MyApp.Ticket do
  use Ash.Resource

  attributes do
    attribute :subject, :string
    attribute :status, :atom
  end

  actions do
    defaults [:read, :create, :update]
  end
end
🔑
Introspection is the secret weapon

A Spark DSL doesn't just generate code — it lets you read the declarations back out as structured data at runtime. Keep that fact in your pocket: it's exactly what makes these DSLs perfect for LLM agents, which is where we head next.
