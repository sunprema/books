Hygiene & the Caller's Context

A macro injects code into someone else's module. Hygiene is the guarantee that the variables your macro introduces won't silently collide with theirs — and it's on by default.

Imagine a macro that needs a temporary variable. In a naïve template language, naming it result would be a disaster the day a caller also has a result. Elixir prevents that. Variables created inside a quote live in the macro's own context, invisible to the caller:

hygiene in actiondefmodule Hygienic do
  defmacro set_temp do
    quote do
      temp = 42          # this temp belongs to the macro
    end
  end
end

defmodule Caller do
  require Hygienic
  def go do
    temp = "mine"
    Hygienic.set_temp()
    temp                # => "mine"  — untouched!
  end
end

The macro's temp and the caller's temp are different variables that merely share a spelling. Elixir tracks the context each was defined in. This is what makes macros composable: you can use one without auditing its internals for name clashes.

🔑
Why this matters

Hygiene is the difference between macros that compose and macros that booby-trap. C's textual #define has no hygiene, which is why it's notorious. Elixir's quoting is hygienic, so leakage is opt-in, never accidental.

Breaking hygiene on purpose: var!

Occasionally you do want to reach into the caller's scope — that's the whole point of a macro like if's siblings, or a DSL that defines a variable for you to use. Wrap the variable in var!/1 to say "this one belongs to the caller's context":

deliberate, visible leakdefmacro define_name do
  quote do
    var!(name) = "Ada"   # injects `name` INTO the caller
  end
end

# after calling define_name(), the caller can read `name`
⚠️
Use sparingly

var! makes a macro change variables the caller can see — powerful, and a classic source of "where did that come from?" confusion. Document it loudly, or avoid it. A surprising binding is worse than a verbose API.

Knowing who called you: __CALLER__

Inside a macro, __CALLER__ is a Macro.Env struct describing the call site — its module, function, file, and line. Great for compile-time error messages that point at the right place:

better errorsdefmacro must_be_atom(value) do
  unless is_atom(value) do
    raise CompileError,
      description: "expected an atom",
      file: __CALLER__.file,
      line: __CALLER__.line
  end
  quote do: unquote(value)
end
Hygiene keeps the two temps apart unless you use var!.
