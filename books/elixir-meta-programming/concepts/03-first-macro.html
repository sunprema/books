Your First Macro

A macro is a function that runs at compile time, receives its arguments as AST, and returns AST to be spliced in its place. Let's write one and watch it expand.

Here's the smallest macro worth writing: a my_unless that behaves like if turned inside-out. Notice we don't evaluate the condition or the block — we receive them as quoted code and weave them into a new tree.

lib/my_macros.exdefmodule MyMacros do
  defmacro my_unless(condition, do: block) do
    quote do
      if unquote(condition), do: nil, else: unquote(block)
    end
  end
end

To use it, the caller must require the module first. That's the rule: you must require a module before calling its macros, because the compiler needs the macro loaded while it compiles you.

using itdefmodule Demo do
  require MyMacros

  def run do
    MyMacros.my_unless 1 > 2 do
      IO.puts("math still works")
    end
  end
end
🧭
def vs defmacro

A def takes values and runs at runtime. A defmacro takes AST and runs at compile time, returning AST. Same do … end syntax at the call site — completely different moment in the program's life.

5 ·

See it expand

Because a macro is just data-in, data-out, you can run it by hand. Macro.expand_once/2 applies one layer and hands you the AST it produced:

iex -S mixiex> require MyMacros
iex> ast = quote do: MyMacros.my_unless(1 > 2, do: IO.puts("hi"))
iex> ast |> Macro.expand_once(__ENV__) |> Macro.to_string() |> IO.puts()
if(1 > 2) do
  nil
else
  IO.puts("hi")
end

That printout is the code that gets compiled in place of your call. No runtime cost, no wrapper function — the if is simply there.

A macro has already finished its job before the program ever runs.
🔑
Why macros must be defined elsewhere

You can't define a macro and call it in the same module at the same point — it must already be compiled. In practice macros live in their own module (or are brought in via use, Chapter 5), and callers require or use them.

⚠️
First rule of macro club

Don't write a macro when a function will do. my_unless is a teaching toy — Elixir's real unless earns its keep because a function can't take an unevaluated block. If your inputs are plain values, write def. We'll make this judgment call rigorous in Chapter 10.
