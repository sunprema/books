Concept 01 · The Idea

Code Is Data

Before any macro, one idea has to land: an Elixir program is itself an ordinary Elixir data structure. Once you believe that, metaprogramming stops being magic and becomes plain manipulation of tuples and lists.

Let's start where it's concrete. Open iex and ask Elixir to hand you a piece of code without running it. The tool for that is quote:

iex sessioniex> quote do: sum(1, 2, 3)
{:sum, [], [1, 2, 3]}

You wrote a function call; Elixir gave you back a three-element tuple. It didn't call sum/3 — there is no such function — it just described the call as data. That tuple is a node of the Abstract Syntax Tree (AST), the shape every Elixir expression takes inside the compiler.

The shape of every node

Each AST node is the same tuple of three things:

{ form, metadata, arguments } — the universal shape of an Elixir AST node.

Operators are just function calls with funny names, so they look the same — only the metadata fills in:

iex sessioniex> quote do: 1 + 2
{:+, [context: Elixir, import: Kernel], [1, 2]}

And it nests, exactly like the source nests — an argument can itself be another node:

iex sessioniex> quote do: sum(1, 2 + 3, 4)
{:sum, [], [1, {:+, [context: Elixir, import: Kernel], [2, 3]}, 4]}

Five things quote themselves

If everything were a tuple it would get noisy fast. So five literal types are their own AST — quoting them returns them unchanged:

LiteralExample
atoms:sum
numbers1, 1.0
strings"hello"
lists[1, 2, 3]
2-tuples{:ok, 1}
 
iex sessioniex> quote do: 42
42
iex> quote do: {:ok, "done"}
{:ok, "done"}

That's why a keyword list like [do: ...] survives intact inside quoted code — it's just lists and 2-tuples all the way down.

🔑
The key idea

This property has a name: homoiconicity — the program is written in its own primary data types. Lisp made it famous; Elixir inherits it from Erlang. Because code is data, a macro can be an ordinary function over that data.

Reading it back

The bridge runs both ways. Macro.to_string/1 turns an AST back into readable source — handy for debugging anything you generate:

iex sessioniex> ast = quote do: sum(1, 2 + 3)
iex> Macro.to_string(ast)
"sum(1, 2 + 3)"
🧭
Hold onto this

Every macro you write is a function that receives these tuples and returns these tuples. Nothing more exotic is happening. The next chapter gives you the two tools — quote and unquote — for building them comfortably instead of typing tuples by hand.
