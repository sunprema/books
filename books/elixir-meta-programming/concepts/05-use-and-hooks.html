use, __using__ & Compile-Time Hooks

Every time you write use Ecto.Schema or use ExUnit.Case, a macro is injecting code into your module. This is the single most important real-world metaprogramming pattern in Elixir — let's build it.

use SomeModule is almost pure sugar: it requires the module and calls its __using__/1 macro, splicing whatever that returns into your module. So a library author writes __using__ once; every caller gets the injected behaviour.

what `use` expands to# this …
use Greeter, name: "Ada"

# … is roughly this:
require Greeter
Greeter.__using__(name: "Ada")
a minimal __using__defmodule Greeter do
  defmacro __using__(opts) do
    name = Keyword.get(opts, :name, "world")
    quote do
      def hello, do: "Hello, " <> unquote(name)
    end
  end
end

defmodule App do
  use Greeter, name: "Ada"
end

App.hello()   # => "Hello, Ada"
🧭
Mental model

use = "mix this module's behaviour into mine." It's the back door through which Ecto gives you schema, Phoenix gives you get/post, and ExUnit gives you test.

Accumulate, then generate

The real power combines three pieces: a module attribute set to accumulate, a macro that appends to it, and a @before_compile hook that reads the whole collection and emits a function. This is exactly how a router collects routes:

lib/router.exdefmodule Router do
  defmacro __using__(_opts) do
    quote do
      Module.register_attribute(__MODULE__, :routes, accumulate: true)
      import Router, only: [route: 2]
      @before_compile Router
    end
  end

  defmacro route(path, target) do
    quote do
      @routes {unquote(path), unquote(target)}
    end
  end

  defmacro __before_compile__(env) do
    routes = Module.get_attribute(env.module, :routes)
    quote do
      def routes, do: unquote(Macro.escape(routes))
    end
  end
end
the payoff at the call sitedefmodule MyRouter do
  use Router
  route "/", HomeController
  route "/about", AboutController
end

MyRouter.routes()
# => [{"/about", AboutController}, {"/", HomeController}]
🔑
The compile-time callbacks

Three hooks fire in order at the end of compilation: @before_compile (inject more code), @after_compile (inspect the finished module), and @after_verify (validate once all modules exist). Macro.escape/1 is the helper that turns a runtime value back into AST so you can unquote it.

⚠️
Gotcha

Accumulated attributes collect in reverse insertion order — reverse the list if order matters. And @before_compile must reference a different module than the one being compiled.
