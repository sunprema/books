Quote, Unquote & bind_quoted

Two special forms do almost all the work. quote lifts code up into data; unquote pours a value down into that data. Master the pair and you can assemble any program by hand.

You already met quote: it freezes an expression as AST instead of running it. The catch is that everything inside the block is frozen — including variable names. Watch what happens when we want a value, not a name:

the problemiex> number = 13
iex> Macro.to_string(quote do: 11 + number)
"11 + number"   # the name, frozen — not 13

We didn't want the symbol number baked into the tree; we wanted its value, 13. unquote is the escape hatch — it evaluates its argument and splices the result into the surrounding quote:

the fixiex> number = 13
iex> Macro.to_string(quote do: 11 + unquote(number))
"11 + 13"
quote builds the tree; unquote drops a runtime value into a hole in it.

A useful slogan: inside a quote, you're writing code; inside an unquote, you're back to writing values. They alternate, as deep as you like.

Splicing a list with unquote_splicing

Sometimes you have a list of arguments and want them spread as arguments, not nested as one list. That's unquote_splicing/1:

iex sessioniex> args = [1, 2, 3]
iex> Macro.to_string(quote do: sum(unquote_splicing(args)))
"sum(1, 2, 3)"   # not sum([1, 2, 3])

The double-evaluation trap

Here is the gotcha that bites everyone once. If you unquote the same expression twice, it runs twice:

danger: side effects run twicedefmacro double_log(expr) do
  quote do
    IO.puts(unquote(expr))
    IO.puts(unquote(expr))   # expr is evaluated AGAIN
  end
end

# double_log(roll_dice()) rolls the dice twice. Surprise.

The clean fix is bind_quoted: pass a keyword list and each value is evaluated once, bound to a hygienic variable you then reference freely:

safe: evaluated oncedefmacro double_log(expr) do
  quote bind_quoted: [val: expr] do
    IO.puts(val)
    IO.puts(val)            # same value, no re-eval
  end
end
⚠️
Rule of thumb

If a macro uses an argument more than once — or you just want fewer surprises — reach for bind_quoted by default. It evaluates each input exactly once and keeps the resulting variables hygienic (more on hygiene in Chapter 4).

🔎
Inspect what you built

When a generated expression misbehaves, wrap it: ast |> Macro.to_string() |> IO.puts(), or use Macro.expand_once/2 to see one layer of expansion. You're debugging data, so print the data.
