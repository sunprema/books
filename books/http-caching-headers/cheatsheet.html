Quick ReferenceThe Caching Cheatsheet

The 30-second version

Cache-Control sets freshness — how long a response can be reused with zero requests. ETag + If-None-Match power revalidation — a cheap 304 instead of a full re-download once it's stale. Pick one of two families per endpoint: expire it (max-age/immutable) or validate it (no-cache + a validator).

Cache-Control directives

max-age=Nfresh for N seconds
s-maxage=Nmax-age, shared caches only
no-cachestore, but revalidate every time
no-storenever store, anywhere
privatebrowser cache only
publicshared caches allowed too
immutablenever revalidate while fresh
must-revalidateno serving stale, ever, once expired
stale-while-revalidate=Nserve stale, refresh in background
stale-if-error=Nserve stale if origin 5xx's

no-cache vs. no-store

no-cache = store it, but ask first.
no-store = don't keep a copy at all.
These are opposites in strength, not synonyms — mixing them up is the single most common Cache-Control mistake.

Conditional GET

ETag: "abc123"          ← on the 200
If-None-Match: "abc123"  ← on the next GET
match  → 304 Not Modified, no body
differ → 200 OK, new body + new ETag

Last-Modified / If-Modified-Since is the same idea at 1-second resolution; If-None-Match wins if both are sent.

Strong vs. weak ETags

ETag: "x" = strong, byte-identical — safe for range requests.
ETag: W/"x" = weak, semantically equivalent — fine for ordinary revalidation, which always compares weakly anyway.

Recipes

Fingerprinted assetpublic, max-age=31536000, immutable
HTML shell / API JSONno-cache + ETag
Personalized dataprivate, no-store
CDN edge, shorter TTLadd s-maxage=N

Common traps

  • Forgetting private on personalized responses → a shared cache can leak them across users.
  • ETags derived from per-node state (inode, local clock) → every request 200s instead of 304ing behind a load balancer.
  • immutable without a fingerprinted URL → clients keep stale bytes for the full max-age, no way to force a refresh.
  • Reaching for no-cache when you meant no-store (or vice versa).
