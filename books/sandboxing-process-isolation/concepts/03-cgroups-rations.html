Chapter 03
The rations

Cgroups: Rationing Food, Water, and Air

Walls stop an inmate from seeing the rest of the facility. They do nothing to stop one inmate from eating all the food, running the taps dry, and using up the air everyone else needs. That's a different threat — not escape, but a riot: denial of service. The tool for it is the control group, or cgroup. Cgroups are the quartermaster: they meter how much CPU, memory, disk I/O, and how many processes each cell may consume.

One hierarchy, one clipboard (cgroups v2)

Modern Linux uses cgroups v2, a single unified hierarchy mounted at /sys/fs/cgroup/. Every process belongs to exactly one cgroup, and all the enabled controllers apply to it together. (The old v1 had a separate tangled hierarchy per controller — one clipboard for CPU, another for memory, none of them agreeing. v2 is one clipboard, and we'll see in Chapter 8 that v1's extra machinery was also an escape hatch.)

The controllers worth knowing are the four basic rations:

ControllerKey fileRations
cpucpu.maxCPU time — a quota/period hard cap, plus cpu.weight for proportional share
memorymemory.maxA hard memory ceiling; memory.high throttles before the kill
ioio.maxPer-device disk bandwidth and IOPS caps
pidspids.maxMaximum number of processes — the fork-bomb ceiling

Try this

Rationing is just writing numbers into files. Cap a cell at half a CPU, 256 MiB of memory, and 64 processes:

# mkdir /sys/fs/cgroup/jail
# echo '50000 100000' > /sys/fs/cgroup/jail/cpu.max   # 50000us per 100000us = 50%
# echo 268435456    > /sys/fs/cgroup/jail/memory.max # 256 MiB, hard
# echo 64           > /sys/fs/cgroup/jail/pids.max   # no fork bombs
# echo $$           > /sys/fs/cgroup/jail/cgroup.procs # move this shell in

Now a runaway loop in this shell can spin as hard as it likes — it gets 50% of one core and no more. Allocate past 256 MiB and the kernel's OOM killer visits this cell, not the neighbors.

People forget pids.max and it costs them. A fork bomb doesn't try to escape — it just breeds until the process table is full and the whole machine, guards included, grinds to a halt. One line rationing the process count is the difference between "an inmate misbehaved" and "the facility went dark." Ration the air.

Top-down delegation

Controllers are switched on from the top down through cgroup.subtree_control, and a child cgroup can only use a controller its parent has already enabled. The quartermaster hands rationing authority downward in a chain; a cell can't grant itself a bigger ration than the block it lives in was given. This is what makes nested resource limits — a container inside a pod inside a node — behave predictably.

Key idea Cgroups bound consumption, not privilege. They stop a riot — a hog, a fork bomb, a memory balloon — but they are not a wall against escape. A perfectly rationed inmate can still walk out the front door if the other locks are open. Ration for availability; use the next chapters for containment.
