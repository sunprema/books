The warden's clipboard

Cheatsheet

The whole facility on one clipboard: every primitive, what it stops, and the command to check it. Isolation is least privilege plus defense in depth — everything below serves those two rules.

Namespaces — the walls (8)

Partition what a process can see. Flags: CLONE_NEW*.

mnt mounts · pid process IDs · net network · ipc SysV/POSIX IPC · uts hostname · user UID map + caps · cgroup cgroup root · time clocks.

API: clone(2), unshare(2), setns(2). First was CLONE_NEWNS (2.4.19); newest time (5.6).

User namespace — rootless

Root inside (full caps), unprivileged outside via UID mapping. Foundation of rootless containers. The one namespace an unprivileged user can create to gain (inner) capabilities.

Cgroups v2 — the rations

Bound consumption (DoS), not privilege. Unified hierarchy at /sys/fs/cgroup.

# half a CPU, 256 MiB, 64 procs
echo '50000 100000' > cpu.max
echo 268435456      > memory.max
echo 64             > pids.max

pids.max is your anti-fork-bomb line. Enable top-down via cgroup.subtree_control.

Capabilities — the keys (41)

Root split into units 0–40 (CAP_LAST_CAP=40). Never grant CAP_SYS_ADMIN.

Contraband: CAP_SYS_ADMIN (the new root), CAP_SYS_PTRACE, CAP_NET_ADMIN, CAP_DAC_OVERRIDE, CAP_SYS_CHROOT.

Sets: permitted · effective · inheritable · bounding (the real ceiling) · ambient. Drop from bounding to truly confiscate.

grep Cap /proc/self/status
capsh --print

Seccomp-BPF — the visiting rules

Filter syscalls by number + args (not pointer contents).

Modes: STRICT (read/write/_exit/sigreturn) · FILTER (classic BPF).

Actions: KILL · TRAP · ERRNO · NOTIFY (supervisor, 5.0+) · ALLOW. Docker's default blocks ~44 syscalls.

Filesystem — chroot vs pivot_root

chroot() is not a boundary: cwd unchanged, fds not closed → fchdir escape. Real containers use pivot_root(2) in a mount namespace, then unmount the old root — no path back.

Container = assembly

No kernel "container" object. runc/OCI order: namespaces → cgroups → rootfs + pivot_root → drop caps + seccomp → execve. Seal the cell before the inmate enters.

Escape files (patch these lessons)

CVE-2019-5736 runc /proc/self/exe overwrite · CVE-2022-0492 cgroup-v1 release_agent · CVE-2016-5195 Dirty COW (shared-kernel LPE) · CVE-2024-21626 runc "Leaky Vessels" (fix: 1.1.12) · CVE-2024-0132 NVIDIA toolkit mount TOCTOU (fix: 1.16.2).

Stronger cells — separate floor

Firecracker: KVM microVM, ~6 devices, <125 ms boot, <5 MiB overhead; backs Lambda/Fargate. gVisor: userspace kernel (Sentry) intercepts syscalls; platforms ptrace/KVM/systrap; Gofer for FS. Escaping needs guest kernel + VMM + hardware.

WASM + capabilities — deny by default

Linear-memory sandbox: no ambient memory/syscalls. WASI = no ambient authority; host grants capabilities (preopened dirs). Object-capability: authority only via unforgeable handles. Preview 2 (0.2) stable; Preview 3 async in progress.

wasmtime run --dir=./work mod.wasm
# no --dir = zero filesystem access

Which cell for which inmate?

Trusted first-party → hardened rootless container. Semi-trusted third-party → tighten caps/seccomp, read-only rootfs. Untrusted / AI-generated needing an OS → gVisor or Firecracker. Untrusted compute-only → WASM + capabilities.

The two rules over the gate

Least privilege: start empty, add only what's proven necessary. Defense in depth: every layer has a blind spot — stack independent boundaries so one bug is never fatal. For each layer, know what it stops and its TCB.
