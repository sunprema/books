06 Graph

Graphs: BFS & DFS

A graph is just things (nodes) connected by relationships (edges). Once you see the world this way, it's everywhere: friends, web links, package dependencies, road maps, state machines. Two traversal algorithms — BFS and DFS — are how you explore any of them.

Both visit every reachable node exactly once, using a "to-do" collection and a "visited" set so you never loop forever. The only difference is which to-do item you pick next:

BFS — breadth first

Use a queue (FIFO). Explore ring by ring: all neighbors, then their neighbors. Finds the fewest-hops path in an unweighted graph.

DFS — depth first

Use a stack (or recursion). Plunge down one path to the end, then backtrack. Great for "does a path exist?", cycles, and ordering.

BFS — shortest hops from a start nodefrom collections import deque

def bfs(graph, start):
    seen = {start}
    q = deque([(start, 0)])   # node, distance
    while q:
        node, dist = q.popleft()  # FIFO → breadth
        for nbr in graph[node]:
            if nbr not in seen:
                seen.add(nbr)
                q.append((nbr, dist + 1))

Swap the queue for a stack (or just recurse) and the very same skeleton becomes DFS. Both run in O(V + E) — vertices plus edges — because each node and each edge is touched once. That's optimal: you can't explore a graph without looking at its connections.

BFS finishes A, then B & C, then D, E, F — distance order. DFS would dive A→B→D before touching C.

The killer app: topological sort

Run DFS on a graph of dependencies (A must come before B) and you can produce a topological order — a valid sequence that respects every "before" rule. If DFS finds a cycle, there's no valid order — a real, catchable error.

◎
Where you'll meet it

Build tools & package managers (npm, Cargo, Make) toposort the dependency graph. Spreadsheets recompute cells in dependency order. Web crawlers are BFS/DFS over links. Social / recommendation: "friends within 2 hops" is BFS. Garbage collectors (see the memory book) trace reachability with exactly this. Mazes, game AI, network discovery, "is the cluster fully connected?"

▣
Pick the right one

BFS when you want the shortest unweighted path or nearest match. DFS when you want to exhaust a branch — cycle detection, toposort, connected components, or generating/searching a tree of possibilities.

BFS finds the fewest hops — but real roads and networks have weights (distance, latency, cost). For the cheapest weighted path, we upgrade to Dijkstra.
