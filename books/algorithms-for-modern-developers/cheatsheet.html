Big-O ladder

O(1)hash lookup
O(log n)binary search, heap op
O(n)a scan, two pointers
O(n log n)a good sort
O(n²)nested loop — usually a bug

Biggest win in real code: O(n²)→O(n log n).

Binary search

Sorted/monotonic data → O(log n). Also "search the answer": smallest X where feasible(X) is true.

Tell: sorted input · "smallest/largest that works" · git bisect · B-tree index.

Hash map / set

O(1) avg lookup. Dedup · count (Counter) · group/index · memoize.

Tell: "have I seen this?" · "count of each" · join two lists · kill an O(n²).

Sorting

Python/Java-objTimsort · stable
Rust sortdriftsort · stable
Go / Rust unstablepdqsort · not
C++ std::sortintrosort · not

Stable = keeps equal-key order → multi-key sorts.

Two pointers / window

Collapse O(n²)→O(n). Pointers converge on sorted data; window grows/shrinks over a contiguous range.

Tell: "pair that sums to…" · "longest substring…" · rate limiting · moving average.

BFS / DFS

O(V+E). BFS=queue→fewest hops. DFS=stack→cycles, toposort, components.

Tell: dependencies · "connected?" · crawler · "shortest unweighted path."

Dijkstra / A*

Cheapest weighted path, O((V+E)log V) via min-heap. No negative edges (use Bellman-Ford). A* = + heuristic.

Tell: maps/routing · network latency · weighted graph.

Dynamic programming

Overlapping subproblems + optimal substructure → cache them. Top-down @cache or bottom-up table.

Tell: edit distance · diff/LCS · "min/max ways to…" · knapsack · autocomplete.

Greedy / intervals

Sort by the right key, one pass. Activity selection = sort by end time. Merge = sort by start.

Tell: scheduling · calendars · merge ranges. ⚠ prove it, or use DP.

Heap / Top-K

Min/max in O(1), push/pop O(log n). Top-K of a stream: size-K min-heap, O(n log k).

Tell: "top N" · leaderboard · scheduler · merge K streams.

Probabilistic

Bloom: membership, "definitely no / probably yes," ~10 bits/elem. HLL: distinct count, ~2% err, KBs.

Tell: "seen before?" at scale · unique visitors · pre-filter before exact check.

Decision shortcuts

  • Need order? → sort
  • Need membership/count? → hash
  • Sorted + "find"? → binary search
  • Network/connections? → graph
  • "Top N"? → heap
  • Huge + approx ok? → probabilistic
★
The meta-skill

You rarely implement these — your standard library has them. The skill is recognition: seeing that "find duplicate uploads" is a hash set, "fewest deploys" is BFS, "top trending" is a heap. Learn the patterns; let the library write the code.
