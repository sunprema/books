Chapter 10
The toolbox

Peeking Inside ELF Files

We've been quoting tool output all through this book. Let's put the whole toolbox on the table at once — these are the instruments a systems engineer reaches for daily, and every one of them is just reading the structures we've already met.

If ELF is a shipping container with a manifest stapled to the door, these tools are the different ways of reading that manifest: file glances at the label from across the room, readelf reads the whole manifest line by line, and objdump actually cracks the crates open and shows you what's inside each one, translated into a language you can read.
ToolWhat it's for
fileOne-line summary: format, bitness, architecture, static/dynamic, stripped?
readelfThe definitive ELF structure reader — headers, segments, sections, symbols, dynamic entries. Doesn't need the BFD library, so it's the most "honest" view.
objdumpDisassembly and structure dumps, via the more general BFD library (also understands non-ELF formats).
nmJust the symbol table — quick answer to "does this binary define/reference symbol X?"
lddWhich shared libraries a dynamic binary needs and where they resolve to.
stringsBlunt but effective: every printable string embedded in the file (great for a first look at an unfamiliar binary).

A guided tour, one command at a time

Try this — the 10-second triage

file is almost always step one — it answers "what am I even looking at?" in one line, without you needing to know any ELF internals at all:

$ file ./hello
./hello: ELF 64-bit LSB pie executable, x86-64, version 1 (SYSV),
dynamically linked, interpreter /lib64/ld-linux-x86-64.so.2,
BuildID[sha1]=..., for GNU/Linux 3.2.0, not stripped

Try this — the symbol table

nm lists every symbol the linker knows about. Uppercase letters mean the symbol is defined here; lowercase means it's a local/weak variant; U means "undefined — please resolve me at link/load time":

$ nm ./hello | grep -i main
0000000000001149 T main
                 U __libc_start_main

T means main is defined in the .text section right here in this file. U next to __libc_start_main means "this symbol lives somewhere else — go ask the dynamic linker."

Try this — disassembly

objdump -d disassembles the machine code in .text back into (approximate) assembly — this is where printf("Hello, kernel!\n") becomes a concrete call instruction:

$ objdump -d ./hello --disassemble=main
0000000000001149 <main>:
    1149: 55                    push   %rbp
    114a: 48 89 e5              mov    %rsp,%rbp
    114d: 48 8d 05 b0 0e 00 00  lea    0xeb0(%rip),%rax
    1154: 48 89 c7              mov    %rax,%rdi
    1157: e8 d4 fe ff ff        call   1030 <printf@plt>
    115c: b8 00 00 00 00        mov    $0x0,%eax
    1161: 5d                    pop    %rbp
    1162: c3                    ret

Look for printf@plt — that @plt suffix ("Procedure Linkage Table") is the compiled-in stub that the dynamic linker patches, at load time, to point at the real printf inside libc.so.

Try this — the dynamic section

readelf -d shows exactly what the dynamic linker will read from PT_DYNAMIC (Chapter 7):

$ readelf -d ./hello
Dynamic section at offset 0x2df0 contains 24 entries:
  Tag        Type            Name/Value
  0x0000000000000001 (NEEDED) Shared library: [libc.so.6]
  0x000000000000000c (INIT)   0x1000
  0x000000000000000d (FINI)   0x1234
  0x0000000000000019 (INIT_ARRAY) 0x3db8
  ...

NEEDED entries are the shared-library shopping list from Chapter 7; INIT_ARRAY is exactly the constructor-function table __libc_start_main walks in Chapter 9.

Key idea Every one of these tools is reading the same bytes we've been describing all book — they're just different lenses on the same ELF header, program headers, and section headers. Once the structures click, the tools stop feeling like separate things to memorize.

Static inspection only gets you so far, though — it tells you what a binary could do. To see what actually happens the moment the kernel takes over, you need to watch it live. That's the next chapter.

Sources: readelf — GNU Binutils documentation, objdump(1) — Linux man-pages, nm(1) — Linux man-pages.
