ELF & execve Cheatsheet

The dense version — for when you already know the story and just need the field name or the flag.

The 30-second flow

  1. shell fork() + execve(path, argv, envp)
  2. kernel checks magic 0x7f 'E' 'L' 'F'
  3. kernel maps each PT_LOAD segment
  4. kernel builds stack: argc/argv/envp/auxv
  5. kernel jumps to interpreter's entry (if PT_INTERP), else your e_entry
  6. ld.so maps needed libs, resolves symbols, jumps to your e_entry
  7. _start → __libc_start_main → constructors → main()
  8. main() returns → exit() → destructors → exit_group syscall

e_type

ET_RELrelocatable (.o, .ko)
ET_EXECfixed-address executable
ET_DYNshared object or PIE
ET_COREcore dump

Elf64_Ehdr fields

e_identmagic, class, endian, ABI
e_typeREL / EXEC / DYN / CORE
e_machinerequired CPU arch
e_entryfirst instruction's address
e_phoff / e_phnumprogram header table location/count
e_shoff / e_shnumsection header table location/count

Elf64_Phdr p_type

PT_LOADa segment to map into memory
PT_INTERPpath to the dynamic linker
PT_DYNAMICdynamic linking info
PT_NOTEbuild-id / vendor metadata
PT_GNU_STACKstack executability flag
PT_GNU_RELROread-only-after-relocation region
PT_PHDRlocation of the program header table itself

p_flags bits

PF_X = 1executable
PF_W = 2writable
PF_R = 4readable

Code segments: R+X, never W. Data segments: R+W, never X (with NX enabled).

objdump -d fdisassembly

strings fprintable strings

strace quick-start

strace -e trace=execve cmdjust the exec calls
strace -f cmdfollow child processes
strace -c cmdsummary by syscall
strace -T cmdshow time per syscall
strace -o out.txt cmdwrite trace to a file

Dynamic linker env vars

LD_LIBRARY_PATHextra search dirs for shared libs
LD_PRELOADforce-load libs before all others
LD_SHOW_AUXV=1dump the auxiliary vector at startup
LD_BIND_NOW=1resolve all symbols eagerly (see RELRO)

Security features

ASLR/proc/sys/kernel/randomize_va_space, kernel-level
PIEe_type = ET_DYN on an executable
NX stackPT_GNU_STACK without the E flag
RELROPT_GNU_RELRO — GOT read-only after relocation
Stack canarycompiler-inserted, seeded via AT_RANDOM

hello.c build variants

$ gcc -O0 -g -o hello hello.c        # dynamic, PIE (default)
$ gcc -O0 -static -o hello.static hello.c
$ gcc -O0 -no-pie -o hello.nopie hello.c
$ strip hello                       # drop symtab/debug, runs the same
