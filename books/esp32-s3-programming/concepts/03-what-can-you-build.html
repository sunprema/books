What Can You Build?

You asked what an ESP32 is good for. The honest answer: anything that needs to sense, decide a little, and talk over a wire or radio — cheaply, by the dozen.

A good way to think about it is to ask, "what is this chip uniquely good at?" It's not raw compute — your laptop crushes it. It's the combination: a sensor-facing computer with a radio, cheap enough to deploy in fleets. So the best projects lean on all three at once. Let's walk from the simplest to the surprisingly clever.

The bread-and-butter: telemetry nodes

The classic ESP32 job is a sensor that phones home. A soil-moisture probe in a greenhouse. A temperature and humidity logger in a server rack. An air-quality monitor in a classroom. The pattern is always the same: read a sensor, wrap the number in a tiny message, publish it over Wi-Fi, sleep, repeat.

🔑
Why this is your sweet spot

You're building an Elixir backend. Elixir on the BEAM is built to hold tens of thousands of live connections calmly. So the use case that fits you best is a fleet of telemetry devices — many cheap S3 nodes, one unflappable server. We'll build exactly that by Chapter 8.

"AI on a $5 chip?"

Yes — within reason. The S3's vector instructions accelerate small neural nets: wake-word detection ("Hey…"), person detection, gesture or vibration classification. Think tiny models (TinyML), not chatbots. Espressif's esp-dl and ESP-WHO exist for exactly this.

What it's not for

Knowing the edges keeps you honest. The ESP32-S3 is the wrong tool when you need:

Reach for a Pi instead

  • A full Linux OS & filesystem
  • Heavy video / a real GPU
  • Gigabytes of RAM, big databases
  • Running Elixir/Nerves itself

The S3 shines for

  • One focused, always-on job
  • Battery / sleep-driven sensing
  • Deploying many cheap units
  • Hard real-time pin wiggling

The boundary is roughly "needs an operating system" vs "is the program." A Raspberry Pi runs Linux and can run Elixir via the Nerves framework. A microcontroller like the S3 runs one program, bare to the metal — and talks to your Elixir server from the outside.
