Chapter 01 · What Pi Actually Is

The Harness You Reshape

Most "AI platforms" hand you a cathedral and rent you a pew. Pi — the open-source toolkit from Mario Zechner's earendil-works — does the opposite. It ships a deliberately tiny agent harness, described by its own author as "a minimal agent harness you reshape around your workflow" that "gets out of your way." The whole thesis of this book is that the shape of that tininess — not any single feature — is where the openings are.

Four tools. That is the whole default toolset.

Out of the box, a Pi agent can do exactly four things: Read, Write, Edit, and Bash. Read a file, write a file, edit a file, run a shell command. That is a coding agent's minimum viable body — and it is intentionally not padded with a hundred bundled integrations. Everything else, you add. That constraint is a gift: a four-tool core is small enough to audit, small enough to trust, and small enough that your domain tools become the interesting part of the surface rather than lost in the noise.

The agent loop, drawn

The runtime that makes those tools go is @earendil-works/pi-agent-core: the agent loop plus tool calling and state management. Conceptually it is a small, honest cycle. The model proposes a tool call; the harness executes it against the real system; the result is fed back; repeat until the task is done. Everything you build in this book is a variation on injecting something — a tool, a memory, a guard, a message — into one of these arrows.

The packages, and why the split matters

Pi is not one blob; it is a small stack you can enter at any level. That layering is exactly what lets a solo domain expert build something a big vendor can't be bothered to.

Four run modes = four business shapes

The same harness reaches you four ways, and each one implies a different product:

IInteractiveA human at a terminal. The consultant's cockpit, the analyst's copilot.
PPrint / JSONHeadless, scriptable, one-shot. Batch jobs, render farms, CI.
RRPCDrive it from another process. Your backend orchestrates the agent.
SSDKEmbed the loop directly in your own app. The vertical-SaaS play.

Extensions: full system access, on purpose

Here's the part that turns a coding CLI into a platform. Extensions are ordinary TypeScript modules with full system access. Because they run with real privilege, they can do real work: add custom tools, register commands and keyboard shortcuts, handle events and inject UI, inject messages before each turn, filter message history, implement RAG or custom retrieval, build long-term memory, spawn sub-agents, add permission gates, do SSH execution, integrate MCP, add custom editors. You bundle these as shareable Pi Packages — four layers (extensions, skills, prompt templates, themes) — distributed over npm or git.

BYOK: the fact that unlocks the regulated markets

Install is one line — npm install -g --ignore-scripts @earendil-works/pi-coding-agent — and auth is bring-your-own-key: export ANTHROPIC_API_KEY / OPENAI_API_KEY / GEMINI_API_KEY, or /login to ride an existing Claude Pro/Max, ChatGPT Plus/Pro, or GitHub Copilot subscription. Nobody at earendil-works sits between you and the model. Your keys, your account, your data path. Hold onto that sentence — it is the reason chapters 3 and 4 (finance, healthcare) are even legal to attempt.

The whole thesis

A big vendor sells you a fixed agent and owns the data path. Pi hands you a reshapeable one and gets out of the way. The opening isn't "AI" — everyone has that. The opening is that you supply the tools, the private data, the memory, and the guardrails, on your infrastructure. That combination is a moat a foundation model can't cross.
