Chapter 06 · Domain Four

Industry & the Edge: SSH, RPC and the Machine Floor

The previous chapters kept the agent in an office. This one puts it on the factory floor, in the field, and on the edge box bolted to a machine. The Lens question that dominates here is where it runs — and Pi's answer is unusually literal. Two capabilities matter: extensions can do SSH execution to reach real systems, and RPC mode lets another process drive the harness. Together they turn the agent from a chat window into an operator with hands.

Bash + SSH is the reach; RPC is the leash

Remember the four-tool core: Bash is already a hand that can run commands. An SSH-execution extension extends that hand to remote machines — a bank of edge devices, a fleet of servers, a controller on a production line. RPC mode inverts the control: instead of a human typing at it, your orchestration process calls the harness, hands it a task, and consumes its structured results. The agent becomes a component in a larger control system, not the top of the stack.

The plays

On-call · SSH execution + memory The runbook operator

Encode your on-call runbooks as skills; give the harness SSH reach into the fleet. When an alert fires, the orchestrator hands the agent the incident over RPC; it executes the documented diagnostic steps, gathers state from the affected boxes, and either remediates the known-safe cases or writes a triaged handoff for the human. Long-term memory means it learns the recurring failure shapes of your systems.

Risk — a wrong command on prod is catastrophic; gate every write/restart action.

Field ops · print/JSON + custom tools The field-service brain on the edge box

A ruggedized edge device with the harness installed and BYOK. A technician (or a sensor threshold) triggers it; custom tools read the machine's telemetry and the maintenance manual RAG; it diagnoses and produces the work order. Because it runs locally, it keeps working when the plant's connectivity doesn't — only the model call needs the network, and you can queue that.

Risk — physical-world actions need interlocks the software can't override.

Embedded · RPC mode + pi-tui The operator console

Embed the harness behind an operator console built on pi-tui (its terminal UI with differential rendering). The operator sees a purpose-built panel; the agent runs underneath in RPC mode, driven by the console. It's the difference between shipping "a chatbot" and shipping "the machine's control surface, made conversational."

Risk — the console is a safety-critical UI; design it like one.

Where you get burned

This is the highest-consequence chapter. An agent with Bash and SSH into production or physical machinery can cause real, un-undoable damage, and Pi has no built-in permission system to stop it (chapter 9). Everything here demands hard interlocks, gated writes, dry-run modes, and a sandbox boundary you built and tested. Read-and-recommend before act-and-remediate, always, until you have earned the trust to loosen it.

The opening

Industrial software is dominated by rigid, expensive incumbents. A harness that can be driven by your existing control systems (RPC), reach your existing machines (SSH), and run at the edge (self-contained, BYOK) is a lightweight operator layer you can wrap around infrastructure that will never be rebuilt. Start with read-only diagnostics and runbook triage — the safe, high-value wedge.
