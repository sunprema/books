The 30-second BookBank

Cheatsheet

Everything in this book, on one page. Copy a command, swap in your topic, go.

Commands

CommandDoes
/write-book <topic>Research & build a new book
/write-bookDo all outstanding queue work
/write-book expand "<c>" in <book>Add one chapter to a book
/create-book-from-issue <n>Build from a filed request issue
/publish-library <id>Publish a ready book (owner)
/plugin install bookbank@kitInstall the tooling (once)

A great request

FieldNote
Topic *A scope, not a word (required)
TitleYour wording, or auto-picked
PersonaVoice — id or free text
ThemeLook — id or free text
Seed conceptsMust-have chapters, one per line
NotesAudience, depth, emphasis
Best habit: one honest sentence in Notes about who the reader is steers research and chapter choice.

Persona / theme cascade

first match wins1  <clone>/personas|themes/<id>.json   # local, never published
2  ~/.claude/bookbank/…/<id>.json     # your set, everywhere
3  bookbank/defaults/…/<id>.json     # ships in the plugin

Tiny schemas

persona{ "id", "name", "tagline", "voice" }  // voice steers writing
theme{ "id", "name", "mood",
  "background", "fonts", "tokens" }        // tokens become CSS vars

Book folder

books/<id>/
  book.json          # manifest: request + index
  index.html         # cover + contents
  concepts/NN-*.html # one page per concept
  cheatsheet.html    # quick reference
  assets/{book.css,book.js,img/}

Request by issue

label book-request → it enters the queuegh issue create --repo sunprema/books \
  --label "book-request" \
  --title "[book] <your title>" \
  --body  "Topic: …
Persona: …  Theme: …
Notes: …"

Lifecycle

request → generate → add art → review PR → publish
   issue    write-book   image slots    merge      live shelf

Live library: sunprema.github.io/books/

Don’t forget

  • Offline & self-contained — no CDNs, no hotlinks, local assets only.
  • Accuracy first — grounded in research; voice never bends the facts.
  • Explanatory → SVG; real artwork → one image slot per idea.
  • Check the phone view — the spread must collapse to one column.
  • Turn every spread — arrows and clicks step through a chapter before the next file.
Now go make one Pick a topic you love, choose a narrator, and run /write-book. That’s the whole trick.
