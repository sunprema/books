Themes — Choosing a Look

If the persona is the voice, the theme is the paper, the ink, and the light. A theme is the book’s look & feel: its palette, its fonts, its background. The book you’re reading uses a theme called Paper Light — off-white paper, one calm teal accent, generous whitespace. Change the theme and the same words feel like a different object in your hands.

A theme is tokens plus a mood

Like a persona, a theme is a small JSON file. It carries a set of tokens (the exact colors), a set of fonts, a background, and a mood — a sentence describing the feeling, which guides the topic-specific flourishes:

codex.json (abridged){
  "id": "codex",
  "name": "Codex",
  "tagline": "Warm aged paper, done well",
  "mood": "A well-set printed book on warm paper: generous
           margins, elegant serif, a restrained oxblood accent.",
  "background": { "css": "radial-gradient(… warm paper …)" },
  "fonts": {
    "--font-display": "'Iowan Old Style', Palatino, serif",
    "--font-body":    "Charter, Georgia, serif"
  },
  "tokens": {
    "--bg": "#f4ecd8", "--ink": "#2b2114",
    "--accent": "#8a2f22", "--rule": "#d8c49c" /* … */
  }
}
Look only A theme governs appearance, never wording. The tokens become CSS variables the stylesheet reads everywhere — so re-skinning a book to a new theme is a clean swap, with the prose untouched.

Several of these are dark; Paper Light (this book’s theme) is a bright, minimal one added the same way you’d add your own — which is exactly what we’ll do next.

How a token becomes a page

The mechanism is refreshingly plain. Each token is written verbatim into the top of the book’s stylesheet as a CSS variable, and the whole design reads from those variables — never from hard-coded colors. That’s the discipline that makes a re-skin trivial:

assets/book.css:root{
  --bg: #fbfbf8; --ink: #1e232b;
  --accent: #0f766e; --rule: #e5e3da;
}
h2{ color: var(--ink); border-bottom: 2px solid var(--rule); }
a { color: var(--accent); }
Same book, new skin Because the look lives entirely in tokens, you can regenerate a book with a different theme and get a genuinely different-feeling book — same chapters, same voice, new palette and fonts.

Defining your own look

Themes resolve through the same three-tier cascade as personas (per-clone → per-user → plugin default; first match wins). So a bespoke theme is one file away — set its tokens and drop it in:

~/.claude/bookbank/themes/paper-light.json{
  "id": "paper-light",
  "name": "Paper Light",
  "mood": "A bright, minimal notebook: soft off-white paper,
           one calm teal accent, generous whitespace.",
  "background": { "css": "radial-gradient(circle at 30% 0%, #fff, #fbfbf8)" },
  "tokens": { "--bg": "#fbfbf8", "--ink": "#1e232b",
              "--accent": "#0f766e" /* … */ }
}
Or just describe it. Like voices, you can ask for a look in plain words — “dark academia, warm lamplight” — and BookBank will build a fitting skin. A saved theme is how you make that look reusable.

Voice × look = your book’s character

Persona and theme are independent knobs, and that’s the point. Mix them freely:

Persona (voice)Theme (look)Feels like…
feynmanblueprinta brilliant lecturer at a drafting table

clear-mentorpaper-lightthis book — calm, bright, practical

The two knobs, together

Persona sets the words; theme sets the world they live in. Both are tiny JSON files, both resolve through the same cascade, and both can be described inline if you don’t want to save a file. Now that you can choose a voice and a look, let’s learn to ask for the content well.
