Chapter 03 · The Verbs

3Actions

If attributes are the nouns, actions are the verbs — the meaningful operations your domain supports, named in your own words.

A plain CRUD app gives you create, read, update, destroy. Ash gives you those too — but it wants you to name the real operations: :open, :close, :assign. Let's add them to the ticket:

inside Helpdesk.Support.Ticketactions do
  defaults [:read]

  create :open do
    accept [:subject]
  end

  update :close do
    accept []
    validate attribute_does_not_equal(:status, :closed) do
      message "Ticket is already closed"
    end
    change set_attribute(:status, :closed)
  end

  update :assign do
    accept [:representative_id]
  end
end

Two ideas do a lot of work here:

  • accept [...] — the exact fields this action takes as input. :open accepts only :subject, so a caller can't sneak in a :status. Safety by declaration.
  • change & validate — the business logic attached to the verb. :close refuses to run on an already-closed ticket, then sets the status. This lives on the action, not in a controller far away.
The mental shift Don't ask “how do I update a row?” Ask “what can happen to a ticket?” Each answer becomes a named action. Your code — and every derived API — then reads like the business, not like the database.

Calling an action

Actions run through a small, consistent pipeline: build an input, then hand it to Ash. Every action type follows the same rhythm.

open a ticketHelpdesk.Support.Ticket
|> Ash.Changeset.for_create(:open, %{subject: "My mouse won't click!"})
|> Ash.create!()

# read the closed ones
require Ash.Query
Helpdesk.Support.Ticket
|> Ash.Query.filter(status == :closed)
|> Ash.read!()

# close one you already loaded
ticket
|> Ash.Changeset.for_update(:close)
|> Ash.update!()

Bang vs. non-bang

Ash generates two flavours of every entry point. Ash.create! raises on failure (great for scripts and “this must work” paths). Ash.create returns {:ok, record} or {:error, changeset} — the version you'll pattern-match in real request handlers.

Beyond CRUD There are also generic actions (an action that isn't tied to a single row — e.g. :calculate_shipping_cost) and manual actions for when you need full control. Reach for them rarely; named create/read/update/destroy actions cover most of a domain.
