Chapter 08 · Derive The Rest

8The Ecosystem

This is the payoff of “model your domain, derive the rest.” Each extension reads your resources and generates a whole capability.

Because your actions are introspectable and typed, an extension can look at a resource and build an entire API, admin screen, or job pipeline from it — usually with a few lines of opt-in. You've already met one extension: AshPostgres, the data layer. Here are the others you'll reach for most:

ExtensionWhat it derives
AshPostgresPostgreSQL persistence + generated migrations.
AshGraphQLA GraphQL API — queries & mutations from your actions.
AshJsonApiA JSON:API-compliant REST API.
AshAuthenticationUser sign-in: passwords, magic links, OAuth.
AshAdminAn auto-generated admin UI over your resources.
AshStateMachineDeclared state transitions for a resource.
AshObanBackground jobs & scheduling wired to actions.
AshAiLLM-powered tools & structured actions.
The compounding win Add a field to your resource and it appears — correctly typed and validated — in Postgres, GraphQL, JSON:API, and the admin panel at once. That's the single-source-of-truth promise paying rent. No four-places-to-update.

Getting started for real

Ash uses Igniter — a code generator that patches your project — so setup is a single command. Install the generator, then scaffold a new app with Ash (and Postgres) already wired in:

terminal# install the project generator once
mix archive.install hex igniter_new

# new app, Ash + Postgres installed and configured
mix igniter.new helpdesk --install ash,ash_postgres
cd helpdesk

# generate a resource from the CLI
mix ash.gen.resource Helpdesk.Support.Ticket

From there, everything in this book applies: add attributes and actions to the generated resource, list it in your domain, and start calling it.

Where to go next This book is the map, not the territory. When you're ready for detail, the official docs are excellent and current:
  • Get Started — build the helpdesk end-to-end.
  • What is Ash? — the philosophy in full.
  • hexdocs.pm/ash — every DSL block, searchable.
  • ash.hexdocs.pm/llms.txt — an index built for AI assistants; paste it into your tools.
The one thing to remember Model your domain, derive the rest. Everything in Ash — resources, actions, domains, policies, extensions — is a way to describe your business precisely enough that the machine can build the rest for you. Get the model right, and the framework does the heavy lifting.
