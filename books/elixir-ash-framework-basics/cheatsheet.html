Cheatsheet

Everything in this book, condensed. Ash v3.29.

The whole idea in one line Model your domain, derive the rest. Declare resources & actions; Ash generates the APIs, persistence, validations, and admin.

Resource — the noun

use Ash.Resource,
  domain: MyApp.Support,
  data_layer: AshPostgres.DataLayer

attributes do
  uuid_primary_key :id
  attribute :subject, :string, public?: true
end

Fields are private by default — add public? true to expose them.

Actions — the verbs

actions do
  defaults [:read, :destroy]
  create :open, do: accept [:subject]
  update :close do
    change set_attribute(:status, :closed)
  end
end

Name real operations, not just CRUD. change / validate attach logic to the verb.

Calling actions

# raising (! ) vs {:ok/:error}
Ticket |> Ash.Changeset.for_create(:open, %{…})
       |> Ash.create!()

require Ash.Query
Ticket |> Ash.Query.filter(status == :open)
       |> Ash.read!()

Domain + code interface

defmodule MyApp.Support do
  use Ash.Domain
  resources do
    resource Ticket do
      define :open_ticket, args: [:subject], action: :open
      define :get_ticket, action: :read, get_by: [:id]
    end
  end
end

MyApp.Support.open_ticket!("Help!")

Relationships & derived data

belongs_to :representative, Representative
has_many   :tickets, Ticket

calculate :is_open, :boolean, expr(status == :open)
count :open_count, :tickets do filter expr(…) end

Load related data explicitly: load: [:tickets] — batched, no N+1.

Policies — who may do what

use Ash.Resource, authorizers: [Ash.Policy.Authorizer]

policies do
  bypass actor_attribute_equals(:admin, true) do
    authorize_if always()
  end
  policy action_type(:read) do
    authorize_if expr(representative_id == ^actor(:id))
  end
end

Pass actor: user. Reads filter; first check to decide wins.
