Concept 01 · The Why

Why Rust, Why Now

For fifty years, systems programmers accepted a devil's bargain. Rust is the first language big enough — and stubborn enough — to walk away from the table.

Let me start with the bargain, because everything about Rust is a reaction to it. If you wanted your program to be fast — to talk to the metal, to not pause for a garbage collector, to fit on a tiny chip — you wrote C or C++. And in exchange, you personally promised to never, ever make a certain kind of mistake. You promised to free every piece of memory exactly once. To never touch a pointer after you'd freed it. To never let two threads scribble on the same value at the same time.

Here's the thing: nobody can keep that promise. Not the smartest person you know, not on a good day, not with coffee. These aren't dumb mistakes — they're the kind a brilliant engineer makes at hour nine, in a function they wrote six months ago, that only crashes on a customer's machine on a Tuesday. Microsoft and Google both went and counted: roughly 70% of their serious security bugs came from exactly this one category — memory being used wrong.

🔑
The whole pitch, one sentence

Rust gives you C-level speed and control, and a compiler that refuses to build the programs where that speed would have blown up in your face.

"Safe" isn't the tax — it's the whole point

Now, when people hear "safe language," they think of a trade: surely you pay for that safety with a garbage collector, or a slower runtime, or a fat virtual machine. That's the intuition from Java, Python, Go — all wonderful, all carrying a little runtime engine that watches your memory for you at some cost.

Rust's trick — and it really is a trick worth admiring — is that it moves the checking to before the program runs. The safety is enforced by the compiler, at build time, and then it evaporates. There's no garbage collector riding along. The finished program is just tight machine code, the same shape C would have produced. You paid for safety with the compiler's time and a little of your own — not with your program's speed.

The corner everyone said was empty — fast and safe.

You do not need to memorize the code below. Just feel the shape of it: an ordinary function, ordinary-looking, that would have been a landmine in C and is simply fine here.

rust · perfectly ordinary, perfectly safefn longest_line(text: &str) -> &str {
    text.lines()
        .max_by_key(|line| line.len())
        .unwrap_or("")
}

fn main() {
    let poem = "roses are red\nviolets are cyan-ish";
    println!("{}", longest_line(poem));
}

In C, that innocent &str going in and coming back out is a pointer into somebody's buffer. Return it after that buffer is gone and you've got a use-after-free — a bug that might work in testing and corrupt memory in production. In Rust, the compiler tracks that the returned slice borrows from text, and it will not let the program compile if that borrow could outlive the data. You didn't write a single line of safety machinery. It came for free with the types.

🤖
Why this matters when an agent writes the code

When you tell Claude Code "parse this file and hand me back the longest line," an LLM can absolutely fumble a lifetime or an ownership detail — the same way a human can. The difference in Rust is that the fumble becomes a compile error, not a silent corruption. The agent's mistakes get caught by a machine, out loud, instantly. That is a fantastic property for a workflow where a very fast, very literal collaborator is doing the typing.

Why now, specifically

Rust isn't a hopeful experiment anymore; the grown-ups moved in. A quick tour of the evidence, because it changes how seriously you should take it:

  • The Linux kernel. After years of "let's try it," the kernel maintainers made it official in 2025: Rust is a permanent language for kernel code. The most conservative codebase on Earth said yes.
  • Your phone. Android already ships production components written in Rust to billions of devices.
  • The cloud. AWS's Firecracker — the microVM engine under Lambda and Fargate — is Rust, running trillions of executions a month.
  • Developers themselves. Rust has been the most-admired language in the giant Stack Overflow survey for nine years running. People who use it want to keep using it.

And the timing rhymes with the other big shift: agents. When a model can generate a working draft in seconds, the bottleneck moves from typing to trusting. Rust's compiler is a trust machine. That combination — a language that verifies, and an assistant that generates — is why this is a genuinely good moment to learn the ideas.

🧭
Where we go next

All of that safety flows from one idea — ownership — enforced by a part of the compiler called the borrow checker. People fear it. We're going to make friends with it, because once it clicks, the rest of Rust feels almost inevitable.
