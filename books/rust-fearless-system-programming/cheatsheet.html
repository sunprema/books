Rust — The 30-Second Cheatsheet

The whole book on one page — the instincts you steer an agent with, not a syntax dump.

0 · The Pitch

C-level speed & control + a compiler that won't build use-after-free, null bugs, or data races. Safety is checked at compile time, then vanishes — no garbage collector, no runtime tax.

1 · Ownership (the one idea)

  • Every value has one owner.
  • Owner goes away → value freed. No GC.
  • Move = hand it over (you lose it).
  • Borrow & = lend it (you keep it).
  • Clone = an explicit, visible copy.

2 · The Borrow Rule

Many readers &T or one writer &mut T — never both at once. This single rule is why data races can't compile.

Steering Q: "who owns this, who may change it?"

3 · No Null

Maybe absentOption<T> → Some/None
Can failResult<T,E> → Ok/Err

Compiler forces you to handle the empty/error case.

4 · Illegal States = Unbuildable

  • enum = "exactly one of these shapes."
  • match is exhaustive — forget a case, it won't compile.
  • Newtype (Seconds(u64)) so units can't mix.

5 · Errors Are Values

let s = fs::read_to_string(p)?;

? = "if Err, return it; else unwrap." Failure is visible in the signature. No hidden exceptions.

6 · Fearless Concurrency

  • Data race = compile error in safe Rust.
  • Share safely with Arc<Mutex<T>>.
  • Send/Sync = auto-tracked trip-wires.
  • Async I/O at scale → tokio.

Caveat: deadlocks are still on you.

7 · Zero-Cost Abstractions

Iterator chains (.filter().map().sum()) and generics compile to hand-tuned machine code. Expressive and fast — no trade. Cost is paid at build time.

8 · Cargo (one tool)

$ cargo new · add · build
$ cargo test · run
$ cargo clippy   # lint + teach
$ cargo fmt      # one true style

Deps from crates.io, version-locked.

9 · Agent Workflow

  1. Describe the shape (types, error cases).
  2. Agent scaffolds with Cargo.
  3. "Loop until build+clippy+test are clean."
  4. You review design, not semicolons.
  5. Ask for tests of the nasty edge cases.

10 · Red Flags 🚩

  • unsafe to silence the borrow checker — interrogate every use.
  • "It compiles" ≠ "it's correct" — logic bugs still need tests + your judgment.
  • Empty error handling that ignores a Result.

11 · Use Rust When…

✓ CLIs · WASM · embedded/systems · high-perf services · data/infra engines · security-critical code.

✗ quick scripts · glue · throwaway prototypes · plain CRUD with no perf needs.

Test: is speed, safety, or tight resources load-bearing?

12 · The Division of Labor

You = architect (data model, error cases, is-Rust-right). Agent = builder (syntax, boilerplate, fixing complaints). Compiler = inspector (memory, types, races). Machine polices machine; you stay on design.
