Chapter 07 · The Payoff

The Elixir Workbench

Everything so far was groundwork. Now we spend it. Here are five hooks that make Claude Code a genuinely better teammate on an Elixir/Mix project — each one small, deterministic, and copy-pasteable. Together they turn “please remember to…” into “it simply happens.”

The BEAM already believes in fast feedback loops — mix test.watch, recompilation on save. Hooks let the agent live inside that same loop, so it can’t hand you code that doesn’t format, compile, or pass.

Recipe 1 · Format every file it touches

The classic. After any edit, run mix format on just that file. We read the path from the PostToolUse payload so we don’t reformat the whole tree on every keystroke.

.claude/hooks/format.sh#!/usr/bin/env bash
file=$(jq -r '.tool_input.file_path // empty' < /dev/stdin)
# Only touch Elixir source/config/scripts.
case "$file" in
  *.ex|*.exs|*.heex) mix format "$file" ;;
esac
exit 0
.claude/settings.json"PostToolUse": [
  { "matcher": "Edit|Write",
    "hooks": [
      { "type": "command",
        "command": "$CLAUDE_PROJECT_DIR/.claude/hooks/format.sh",
        "timeout": 30 }
    ]
  }
]

Your diffs are now formatter-clean by construction. The agent never “forgets” — because it isn’t the one remembering.

Recipe 2 · A compile guard that talks back

Formatting is cosmetic; let’s catch real breakage. After an edit, compile with warnings promoted to errors. If it fails, exit 2 and let stderr carry the compiler’s complaint straight back to Claude — which will read it and fix the code.

.claude/hooks/compile-guard.sh#!/usr/bin/env bash
file=$(jq -r '.tool_input.file_path // empty' < /dev/stdin)
case "$file" in *.ex|*.exs) ;; *) exit 0 ;; esac

# Capture compiler output; --warnings-as-errors makes warnings fatal.
if ! out=$(mix compile --warnings-as-errors 2>&1); then
  echo "Compile failed — fix before continuing:" >&2
  echo "$out" >&2
  exit 2     # block; Claude gets the errors as its next task
fi
exit 0
Why this is the whole idea The hook doesn’t just fail — it hands Claude the error text. Deterministic detection, model-driven repair. You’ve built a tiny closed loop where breakage can’t slip past silently.

Recipe 3 · The test gate — don’t stop on red

The crown jewel. Hook Stop so that when the agent tries to finish, we run the suite. If tests pass, we let it stop. If they fail, we exit 2 — which, on Stop, means keep going — and feed the failures back. The agent literally cannot walk away from a red suite.

.claude/hooks/test-gate.sh#!/usr/bin/env bash
# Run the suite quietly; capture output either way.
if out=$(MIX_ENV=test mix test --warnings-as-errors 2>&1); then
  exit 0     # green: allow the agent to stop
fi
echo "Tests are failing — don't stop yet. Fix these:" >&2
echo "$out" | tail -n 40 >&2
exit 2       # red: on Stop, exit 2 keeps Claude working
Add an escape hatch A test gate that never lets go can trap the agent in a loop on a genuinely hard failure. In practice, cap it — e.g. read the transcript to count consecutive blocks, or let continue:false bail after N tries — so a stuck suite doesn’t spin forever. Determinism is a tool, not a trap.

Recipe 4 · Guard destructive Mix & shell

Some commands should never run unattended. A PreToolUse guard on Bash blocks the ones that drop databases or nuke directories — before they execute.

.claude/hooks/guard-bash.sh#!/usr/bin/env bash
cmd=$(jq -r '.tool_input.command // ""' < /dev/stdin)
deny='rm -rf|mix ecto.drop|mix ecto.reset|DROP TABLE|git push --force'

if grep -qiE "$deny" <<<"$cmd"; then
  echo "Refused: '$cmd' matches a destructive pattern." >&2
  exit 2
fi
exit 0

Recipe 5 · Prime every prompt with project context

A UserPromptSubmit hook can quietly prepend live facts to every message — the git branch, the Elixir/OTP versions, a house rule — using the JSON additionalContext channel from Chapter 05.

.claude/hooks/prime-context.sh#!/usr/bin/env bash
branch=$(git rev-parse --abbrev-ref HEAD 2>/dev/null)
elixir=$(elixir -e 'IO.write System.version()' 2>/dev/null)

jq -nc --arg b "$branch" --arg e "$elixir" '{
  hookSpecificOutput: {
    hookEventName: "UserPromptSubmit",
    additionalContext: ("Branch: \($b) · Elixir \($e). " +
      "Prefer with/for over nested case. Run mix test before finishing.")
  }
}'
exit 0
One script to rule them Because hook_event_name is always on stdin (Ch. 04), you can fold all of these into a single hooks/dispatch.sh that cases on the event and calls the right function. Fewer files, one place to audit.

So what did we actually gain?

A workbench. The agent now formats what it touches, can’t hand you code that won’t compile, won’t stop on a red suite, refuses to drop your database, and starts every turn knowing the branch it’s on. None of it depends on the model remembering — which is the entire point.
