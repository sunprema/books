Chapter 02 · The Map

The Lifecycle Map

A hook is only as useful as the moment it fires. So before we write another line, let’s walk the agent’s lifecycle and name every point you’re allowed to tap. There are many events, but they cluster into a handful of phases — and in practice you’ll live in nine of them.

Think of the agent’s turn as a river. Events are the gauging stations along it. You don’t have to instrument every station — you pick the two or three where measuring (or damming) actually changes the outcome.

The river, drawn

The nine you’ll actually use

These are the workhorses. Learn what each is for — the “can it block?” column is the payoff, and Chapter 05 shows exactly how.

SessionEndthe session terminatescleanup, final loggingno
The two you’ll reach for first PostToolUse (do something after every edit — format, compile) and Stop (don’t let the agent walk away until the suite is green). Nearly every Elixir recipe in Chapter 07 is built from this pair.

The full catalog (for completeness)

The reference documents a much larger set — Claude Code emits an event at nearly every seam of its execution. You rarely need these, but it helps to know the map is bigger than the nine. A few worth knowing exist for tool failures, subagent starts, tasks, worktrees, config changes, and MCP elicitations:

PhaseEvents

MCP & miscNotification · Elicitation · ElicitationResult

Don’t memorize this table. Bookmark it. When you catch yourself wishing “I want to run something exactly when X happens,” come back and check whether X is already an event. Very often it is.

So what did we actually gain?

A map. We know when we can act, and which of those moments let us actually stop the agent versus merely watch. The rest of the mechanics — how to register, what data arrives, how to answer — is the same at every station. Next we wire one up properly.

Challenges

  1. Which single event would you hook to “announce every finished turn in your terminal’s notification area”? (Two answers are defensible — name both.)
  2. You want to run a script when the agent’s working directory changes. Is there an event for that? Find it in the catalog.
