The 30-second model

A hook is a shell command Claude Code runs at a lifecycle event. It gets the event as JSON on stdin; it answers with an exit code (0 = ok, 2 = block, other = warn) and optionally JSON on stdout.

Events you’ll use

Config shape

.claude/settings.json{
  "hooks": {
    "<Event>": [
      { "matcher": "Edit|Write",
        "hooks": [
          { "type": "command",
            "command": "$CLAUDE_PROJECT_DIR/.claude/hooks/x.sh",
            "timeout": 30 }
        ] }
    ]
  },
  "disableAllHooks": false
}

ExitEffect

Elixir recipes (essence)

read the file that changedfile=$(jq -r '.tool_input.file_path // empty' < /dev/stdin)

# PostToolUse Edit|Write — format
case "$file" in *.ex|*.exs|*.heex) mix format "$file" ;; esac

# PostToolUse — compile guard (block on failure)
mix compile --warnings-as-errors 2>&1 || { echo "$out" >&2; exit 2; }

# Stop — test gate (exit 2 = keep working)
MIX_ENV=test mix test && exit 0 || { echo "fix tests" >&2; exit 2; }

# PreToolUse Bash — block destructive
grep -qiE 'rm -rf|mix ecto.drop|git push --force' <<<"$cmd" && exit 2

Debug & safety

  • claude --debug — watch matches, commands, exit codes.
  • /hooks — see the loaded config.
  • Test by hand: echo '{…}' | ./hook.sh; echo $?
  • Read stdin once → input="$(cat)". JSON → stdout, traces → stderr.
  • Quote every "$var". Anchor with $CLAUDE_PROJECT_DIR. Keep hooks fast & idempotent. Only register code you’ve read.
