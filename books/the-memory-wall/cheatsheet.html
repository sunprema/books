Glossary

TermWhat it means
Memory wallThe growing gap between how fast processors compute and how fast memory can feed them data (Wulf & McKee, 1994).
ChipletA small die built to be packaged together with other dies, instead of one giant monolithic die.
3D V-CacheAMD's technique of stacking an extra cache die on top of a compute chiplet via through-silicon vias.
HBM / HBM4High Bandwidth Memory — DRAM stacked and wired with a very wide (2,048-bit) bus for TB/s-class bandwidth, used in GPUs/accelerators.
DDR5Mainstream CPU main memory; cheap and dense but far narrower-bus (64-bit/channel) than HBM.
HBFHigh Bandwidth Flash — HBM-style packaging applied to NAND flash, for cheap high-bandwidth capacity (expected ~2027).
PNMProcessing-Near-Memory — a logic layer placed next to (or under) memory to compute close to the data.
UCIeUniversal Chiplet Interconnect Express — open standard for wiring chiplets from different vendors together on one package.
NVLinkNVIDIA's high-bandwidth, low-latency GPU-to-GPU (and CPU-to-GPU, as NVLink-C2C) interconnect.
PCIeThe general-purpose expansion bus; far lower bandwidth than HBM or NVLink, and the usual CPU–GPU bottleneck.
KV cacheThe saved attention "keys and values" from every prior token, re-read on every step of LLM decode.
MoEMixture-of-Experts — activates only a few sub-networks per token, cutting compute, but not KV-cache size.
Arithmetic intensity / rooflineFLOPs performed per byte moved; the model architects use to tell whether code is compute-bound or memory-bound.
Speculative decodingA small draft model proposes several tokens; the big model verifies them all in one memory-bound pass instead of one-at-a-time.
Dark siliconThe fraction of a chip's transistors that can't be powered on simultaneously within its thermal budget.

The four fixes

1 · High Bandwidth Flash

Repackaged NAND for cheap, HBM-adjacent read bandwidth — for weights, not activations.

2 · Processing-Near-Memory

Logic stacked next to memory; ship results, not raw data. Standard interface, radically shorter wires.

3 · 3D memory-logic stacking

Vertical density for cache and memory; limited to roughly 6–8 tiers by cumulative thermal resistance.

4 · Rethinking autoregression

Speculative and non-autoregressive decoding verify or generate many tokens per memory pass, not one.

Decision checklist — before you blame the algorithm

  • Have you measured bytes moved, not just FLOPs or wall-clock time? (Use a cache-aware roofline / bandwidth profiler.)
  • Is the workload's arithmetic intensity low (memory-bound) or high (compute-bound)? That decides which fix even applies.
  • Is hot, latency-sensitive data sitting on the fastest tier it can afford (GPU HBM), and cold data pushed to a cheaper tier (CPU DRAM / NVMe)?
  • If you're serving LLMs: is your KV-cache strategy (quantization, paging, offload) deliberate, or just whatever the default gave you?
  • Are you paying a round-trip latency tax repeatedly (per-token synchronization) that a shorter interconnect hop (NVLink/UCIe) could remove?
  • Could a change in algorithm (speculative decoding, batching, sparsity) reduce the number of memory round-trips instead of trying to widen the pipe?
The one-sentence summary Compute got fast and (relatively) cheap; moving data did not — so the engineers, and the chips, that win from here understand data movement first and arithmetic second.
