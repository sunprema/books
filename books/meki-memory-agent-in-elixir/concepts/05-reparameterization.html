The Re-Parameterization Trick

MeKi trains with an expensive non-linear projection and infers without it. The bridge is a single observation about what is, and isn't, constant after training.

Recall the expert vector from Chapter 3, the version with both memories fused:

elt = αl · RMSNorm( Ml[xt] + βl · Gl(Eglobal[xt]) )

Now look hard at the dynamic term. The projection Gl is applied only to Eglobal — the word embedding table. After training, every input to Gl is a frozen constant. The scalars αl, βl are frozen. Ml is frozen. So the whole right-hand side, for every one of the |V| possible token ids, can be computed once, offline, and stored.

💡
The one idea

A function of a constant is a constant. The SwiGLU only ever sees fixed embeddings, so its output is a fixed table — there is no reason to recompute it at every token, on every device, forever.

9 ·

So we bake a new table. Define the re-parameterized memory:

M̃l = αl · RMSNorm( Ml + βl · Gl(Eglobal) )  ∈ ℝ|V|×dmem

Computed row by row across the whole vocabulary, once, at export time. At inference the entire expert reduces to a bare lookup — the SwiGLU is gone:

ẽlt = M̃l[xt]

followed by the same lightweight gate from Chapter 4. The inference path, in full:

g = σ(Wgate h),  v = ẽ + g,  y = RMSNorm(Wout v)

This is the move that makes MeKi free. The heavy SwiGLU adds O(d²model) FLOPs per token during training, but contributes zero at inference — it has been absorbed into a table that sits in ROM. Stranger still (Chapter 8): making Gl more expensive during training improves the final model, even though none of those FLOPs survive to deployment.

elixir · fold once at export# All inputs are frozen post-training constants.
defn reparameterize(m_l, e_global, g_params, alpha, beta) do
  m_dyn = MeKi.SwiGLU.forward(e_global, g_params)  # {|V|, d_mem}
  (m_l + beta * m_dyn)
  |> MeKi.Norm.rms_norm()
  |> Nx.multiply(alpha)                       # M̃_l, ready for ROM
end
📦
Two models, one weight set

MeKi is the same network in two costumes: a training form with a learnable projection, and an inference form that is a lookup table plus two thin matrices. Re-parameterization is the wardrobe change — and it changes the cost, not the math.
