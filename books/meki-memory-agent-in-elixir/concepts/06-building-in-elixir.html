Building MeKi in Nx & Axon

Four equations, four functions. We write the training form of the block as a custom Axon layer, then hang it beside the FFN.

Start with the two primitives MeKi leans on. RMSNorm (no separate gain — the scalar α carries the scale):

elixir · lib/meki/nn.exdefmodule MeKi.NN do
  import Nx.Defn

  defn rms_norm(x, opts \\ []) do
    opts = keyword!(opts, eps: 1.0e-6)
    ms = Nx.mean(Nx.pow(x, 2), axes: [-1], keep_axes: true)
    x * Nx.rsqrt(ms + opts[:eps])
  end

And the dynamic projection Gl — a SwiGLU mapping d_model → d_mem with intermediate width d_model/2:

elixir · lib/meki/nn.ex  # w_g, w_u :: {d_model, d_ff}   w_d :: {d_ff, d_mem}
  defn swiglu(x, w_g, w_u, w_d) do
    gate = Axon.Activations.silu(Nx.dot(x, w_g))
    up   = Nx.dot(x, w_u)
    Nx.dot(gate * up, w_d)
  end
end
⚙️
Why defn

Functions in Nx.Defn are traced and JIT-compiled by a backend such as EXLA into one fused kernel. The lookup, the SwiGLU, the gate, and the norm become a single compiled graph — no per-op overhead.

Now the block itself. A custom Axon layer takes a forward defn plus a list of inputs and trainable parameters declared with Axon.param/2. Here is the training-form MeKi layer in full:

elixir · lib/meki/layer.exdefmodule MeKi.Layer do
  import Nx.Defn
  alias MeKi.NN

  # hidden: Axon node {b, s, d_model} from the pre-FFN norm
  # tokens: Axon input  {b, s}  of integer ids
  # e_global: the frozen shared word-embedding {|V|, d_model}
  def meki(hidden, tokens, e_global, opts) do
    v = opts[:vocab]; dm = opts[:d_model]
    dmem = opts[:d_mem]; dff = div(dm, 2)

    m_l    = Axon.param("m_l",    fn _ -> {v, dmem} end)
    w_g    = Axon.param("w_g",    fn _ -> {dm, dff} end)
    w_u    = Axon.param("w_u",    fn _ -> {dm, dff} end)
    w_d    = Axon.param("w_d",    fn _ -> {dff, dmem} end)
    w_gate = Axon.param("w_gate", fn _ -> {dm, dmem} end)
    w_out  = Axon.param("w_out",  fn _ -> {dmem, dm} end)
    alpha  = Axon.param("alpha",  fn _ -> {} end)
    beta   = Axon.param("beta",   fn _ -> {} end)

    Axon.layer(
      &forward/12,   # 11 inputs + the opts Axon appends
      [hidden, tokens, Axon.constant(e_global),
       m_l, w_g, w_u, w_d, w_gate, w_out, alpha, beta],
      op_name: :meki
    )
  end

  defnp forward(h, tok, e_glob, m_l, w_g, w_u, w_d,
                w_gate, w_out, alpha, beta, _opts \\ []) do
    # expert: static lookup + dynamic projection, fused
    m_static = Nx.take(m_l, tok)                 # {b,s,d_mem}
    e_rows   = Nx.take(e_glob, tok)             # {b,s,d_model}
    m_dyn    = MeKi.NN.swiglu(e_rows, w_g, w_u, w_d)
    e = alpha * MeKi.NN.rms_norm(m_static + beta * m_dyn)

    # inject: low-rank additive-sigmoid gate, project back up
    g = Nx.sigmoid(Nx.dot(h, w_gate))   # {b,s,d_mem}
    v = e + g
    MeKi.NN.rms_norm(Nx.dot(v, w_out))    # {b,s,d_model}
  end
end

Hanging it beside the FFN

The block returns an Axon node shaped {b, s, d_model} — exactly like the FFN's output. Chapter 2's equation H′ = F(H) + MeKi(H) + A is then literally an Axon.add:

elixir · one decoder layerdef decoder_layer(x, e_global, opts) do
  # ... attention sub-block produces `a` (the residual A) ...
  h = Axon.layer_norm(a, name: "pre_ffn_norm")

  ffn  = MyModel.FFN.swiglu_ffn(h, opts)
  meki = MeKi.Layer.meki(h, opts[:tokens], e_global, opts)

  Axon.add([ffn, meki, a])     # F(H) + MeKi(H) + A
end

⚠️
Two honest caveats

1. The token ids must reach every layer, not just the embedding — pass them through as a second model input (opts[:tokens]). 2. e_global is the model's own word-embedding table; freeze it inside MeKi (it is wrapped with Axon.constant/1 here) so MeKi reads it without owning it.

🔑
What we just built

A drop-in capacity extender: one Axon.param table plus four small matrices, fused by two Nx.take lookups and a gate, added in parallel to the FFN. Train this exactly like the dense baseline — same optimizer, same data. The interesting part happens after training, next.
