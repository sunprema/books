In one breath

Per layer, per token: read a small knowledge vector from a ROM table by token id, nudge it with a thin context gate, project it up, and add it in parallel to the FFN. Train with a non-linear projector; fold that projector into the table for zero-cost inference.

Training-time expert

e = α·RMSNorm( Ml[x] + β·Gl(E[x]) )
  • Mˡ — static table {|V|,d_mem}
  • Gˡ — SwiGLU projector (train only)
  • α,β — learned scalars

Inject (both phases)

g=σ(Wgh); v=e+g; y=RMSNorm(Wov)
  • Fusion: additive-sigmoid (ablation winner)
  • W_gate {d_mem,d_model}, W_out {d_model,d_mem}

Re-parameterize (export)

M̃l = α·RMSNorm( Ml + β·Gl(E) )

Then inference expert is just ẽ = M̃ˡ[x]. SwiGLU gone → zero extra FLOPs.

Where it sits

H′ = F(H) + MeKi(H) + A

Parallel to the FFN — best of 4 placements. Reads the same pre-FFN norm H.

Elixir: inference block

defn meki_infer(h, tok, m, wg, wo) do
  e = Nx.take(m, tok)
  g = Nx.sigmoid(Nx.dot(h, wg))
  rms_norm(Nx.dot(e + g, wo))
end

Gotchas

Pass token ids to every layer, not just the embedding. Freeze E_global inside MeKi. d_mem ≪ d_model is what keeps it cheap. Re-parameterize once at export, never per token. Numbers above are the paper's, not reproduced here.
