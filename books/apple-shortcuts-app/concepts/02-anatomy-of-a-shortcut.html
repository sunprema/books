Chapter 02 · The Mental Model

Anatomy of a Shortcut: Actions & the Pipeline

Every shortcut, from a two-step toy to a fifty-step monster, is the same machine: a pipeline. Content enters at the top, each action transforms it or acts on it, and the result flows to the next step. Understand this one diagram and the whole app stops being mysterious.

Actions: the vocabulary

An action is one verb your device knows. Apple ships hundreds — the OS 26 release alone added more than 25 new ones — and every app you install can contribute its own (chapter 10). They fall into a few families worth recognizing on sight:

  • Get actions fetch content: Get Current Weather, Get Latest Photos, Get Contents of URL.
  • Transform actions reshape what flows through: Resize Image, Make PDF, Summarize Text.
  • Do actions have effects in the world: Send Message, Add New Reminder, Set Focus.
  • Script actions steer the recipe itself: If, Repeat, Choose from Menu, Wait (chapter 4).
Key Idea

You never have to remember an action's exact name. In the editor, the search box matches loosely ("pdf", "text", "remind"), and actions are grouped by app and by category. Browsing the action list is how you discover what your phone can do.

Content flows, and it's smarter than it looks

The magic that makes blocks snap together is the Content Graph: Shortcuts knows what kind of thing each action produces (an image, a URL, a contact, a date, text) and converts between kinds automatically when it can. Hand a web article to Make PDF and it doesn't complain that it got a URL instead of a document — it fetches the page and renders it. Hand a photo to Send Message and it attaches it.

When an action needs something specific, it asks for it in its slot. Tap any parameter (the blue-tinted words inside an action) and you can type a value, pick from a list, or wire in a variable.

Magic Variables: the output of every step, on tap

Every action's output is automatically available to every later step as a Magic Variable. When you tap into a parameter and choose Select Magic Variable, the editor lights up each prior action and lets you point at the one whose result you want — no naming, no "set variable" bookkeeping required.

  • Shortcut Input — whatever the shortcut was launched with (a photo from the share sheet, text from Siri, a file).
  • Current Date, Device Details, Clipboard, Ask Each Time — built-in variables you can drop anywhere.
  • Any action's result — "Contents of URL", "PDF", "Weather Conditions" — appear as chips you insert into text, messages, filenames, anywhere.
Note

Tapping a Magic Variable chip also lets you drill into properties: from a Weather Conditions variable you can pull just the temperature; from a calendar event, just its start time. This is how you compose sentences like "72° and clear — first meeting at 9:30" out of live data.

Inputs and outputs: shortcuts are functions

A shortcut can receive input — from the share sheet, from Siri, from another shortcut calling it via Run Shortcut — and can return output with Stop and Output. That means bigger systems can be built from small named pieces, exactly like functions in real programming: a "Log to Journal" shortcut that three other shortcuts call, a "Get Work Address" shortcut used by both your commute and your expense recipes.

Habit

Name shortcuts like verbs ("Text My ETA", "File This Receipt") and give each one one job. Ten small shortcuts beat one giant one — easier to debug, easier to reuse, and each gets its own Siri phrase for free.
