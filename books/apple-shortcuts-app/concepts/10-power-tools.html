Chapter 10 · Going Deep

The Power-User Toolkit

Past the built-ins, Shortcuts opens onto three bigger ecosystems: every app on your device, the entire web, and your own machines. This chapter is the map of the deep end — plus how to share what you build and how to stay safe running what others share with you.

Apps as building blocks: App Intents

Developers expose their app's verbs to the system through Apple's App Intents framework — and those verbs automatically become Shortcuts actions. Search an app's name in the action picker and you'll find what its developer shipped: Things can add a to-do with a deadline, Fantastical can parse "lunch Tuesday 1pm", Halide can fire a specific camera preset, Slack can set your status. The same intents power Siri, Spotlight, and widgets, so good apps keep getting more scriptable with each OS release.

Note

When choosing between two apps for a job, open Shortcuts and compare their action lists — it's a surprisingly honest signal of an app's quality and longevity. Deep Shortcuts support means the developer builds for the platform, not just on it.

The web as an action: Get Contents of URL

Get Contents of URL is a full HTTP client: GET or POST, custom headers, JSON bodies. Point it at any API, then use Get Dictionary Value to pull fields from the response. That single pair connects Shortcuts to essentially every service with an API:

  • Home dashboards: query your home server, solar inverter, or router.
  • Webhooks: POST to Zapier, Make, n8n, IFTTT, GitHub Actions, or your own endpoint — the escape hatch to any automation the Apple ecosystem can't do natively.
  • Spreadsheet rows, chat messages, tickets: most SaaS products accept a simple authenticated POST.
The shape of an API call, in Shortcuts terms// Get Contents of URL
URL:     https://api.example.com/log
Method:  POST
Headers: Authorization: Bearer •token•
Body (JSON):
  { "item": •Ask for Input•,
    "when": •Current Date• }

// Get Dictionary Value → key: "status"  → If: equals "ok" → notify ✓

Talking to your computers

  • Run Script Over SSH — a shortcut on your phone can execute a command on any machine you can SSH into: restart a service, kick off a backup, pull a server log. Your homelab just grew a glass button panel.
  • Run Shell Script / Run AppleScript (Mac only) — full zsh, Python, or AppleScript inside a shortcut; Shortcuts becomes the friendly face on top of your scripts.
  • Automator & the command line: old Automator workflows import directly, and shortcuts run "Name" works in Terminal — so shortcuts can be called from scripts, cron jobs, and launchd too. Automation flows both directions.
  • Run JavaScript on Web Page — scrape or manipulate the page currently open in Safari from the share sheet.

Sharing your shortcuts

Any shortcut can be shared as an iCloud link (⋯ → Share). Recipients tap the link, see a preview of every action, and add it to their library. This is how the enormous community works — sites like RoutineHub, the r/shortcuts subreddit, and curated libraries such as Matthew Cassinelli's publish thousands of ready-made recipes. Personalized values (your addresses, tokens) are stripped or prompted for on import via Import Questions.

Safety

Treat downloaded shortcuts like downloaded software. Read the action list before running — the preview shows exactly what it does. Be suspicious of shortcuts that phone home (Get Contents of URL to a domain you don't recognize), ask for contacts or photos without an obvious reason, or arrive obfuscated behind dozens of base64 Text blocks. Untrusted-shortcut warnings exist for a reason; so does the delete button.

Habits of heavy users

  • Compose, don't duplicate: shared logic lives in one helper shortcut, called with Run Shortcut. Fix once, fixed everywhere.
  • Folders + naming: group by context (Home, Work, Car, Utilities); prefix helpers with "·" so they sort out of the way.
  • Fail loudly: end risky recipes with a Show Notification "✓ done" — silence is indistinguishable from failure.
  • Version by duplicate: before major surgery on a 50-action shortcut, duplicate it. There is no undo history across sessions.
  • Steal constantly: the Gallery, RoutineHub, and every shared link are open textbooks — import, dissect, remix.
Key Idea

Shortcuts scales from "one tap, three actions" to a genuine automation platform: app intents below, web APIs beside, SSH and shell above. The ceiling is far higher than the friendly grid of tiles suggests — and every floor of the building uses the same blocks you learned in chapter 2.
