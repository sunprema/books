Automations: When Your Phone Acts First

Everything so far required you to push the button. The Automation tab removes the button. An automation is a shortcut bolted to a trigger — a time, a place, a device event — so the recipe runs the moment life does the triggering. This is where Shortcuts stops being a launcher and starts being a butler.

The trigger catalog

Personal automations offer four families of triggers:

FamilyTriggersExample use
Time Time of day · sunrise/sunset (± offset) · alarm stopped/snoozed · wake-up & bedtime (Sleep schedule) · Apple Watch workout starts/ends 7:00 weekdays → morning brief; workout ends → log it
Travel Arrive at a location · leave a location · before I commute · CarPlay connects/disconnects Leave office → text ETA; CarPlay connects → driving Focus + podcast
Communication Message received (from a person, or containing a phrase) · email received "On my way?" from partner → auto-reply with live ETA
Device state App opened/closed · Wi-Fi network joined · Bluetooth device connects · charger connected/disconnected · battery level · Low Power Mode · Airplane Mode · Focus on/off · Do Not Disturb · NFC tag scanned · Sound Recognition (a doorbell, an alarm) · Stage Manager · CarPlay Battery ≤ 20% → Low Power Mode; open TikTok → 10-min timer
Note

Home automations ("when the front door unlocks", "when the last person leaves home") are the same idea but live in the Home app and run on a home hub — they can still call your shortcuts via "Convert to Shortcut" in the Home editor. And as of macOS Tahoe (the 26 release), the Mac finally has automations too: time of day, email or message received, file added to a folder, disk connected, Focus changes, Bluetooth, and more — folder-watch alone replaces a lot of third-party utilities.

Run immediately vs. ask first

Each automation has a switch that decides its personality:

  • Run Immediately — fully silent, no confirmation. Most triggers support it; this is the "butler" mode. Optionally Notify When Run posts a receipt.
  • Run After Confirmation — the trigger posts a notification; the shortcut runs only when you tap it. Right for anything that sends messages or costs money, and required by a few sensitive triggers.
Gotcha

Location triggers need Location Services for Shortcuts set to While Using/Always, and arrive/leave regions are roughly street-sized — don't expect room-level precision (that's what NFC tags are for). Message triggers can read only what you configure, and automations that would interrupt (playing sound, speaking) respect Focus modes.

Key Idea

Great automations are boring: they encode a decision you already make the same way every time. If you'd ever answer "it depends," keep it a manual shortcut — automate only the things you never want to think about again.
