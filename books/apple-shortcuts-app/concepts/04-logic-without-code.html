Logic Without Code: If, Repeat & Menus

A straight pipeline is a good trick; a pipeline that can decide is a tool. The Scripting category gives you the four moves that turn recipes into programs — conditions, menus, loops, and structured data — all without typing a line of syntax.

If / Otherwise: the fork in the road

The If action compares its input against a condition and runs one of two branches. Conditions adapt to the type: text gets contains / begins with, numbers get greater than, everything gets has any value.

Pattern · Weather-aware morning
☂
Get Current Weather Output: Weather Conditions.
⑂
If — Condition contains "Rain" Then: Show Notification "Take the umbrella." Otherwise: do nothing. End If.
Note

Ifs can nest, but two levels deep is the comfort limit in a touch editor. If you feel like you're writing a decision tree, switch to Choose from Menu — or split the branches into separate shortcuts called with Run Shortcut.

Choose from Menu: one button, many tools

Choose from Menu pops a list of options you define, each with its own branch of actions. It's how power users build a single "Swiss Army" shortcut — one Home Screen icon that opens a menu: Log water · Start timer · Text ETA · Scan document. Each menu item is its own mini-pipeline.

Repeat: doing it fifty times

Two flavors of loop:

  • Repeat — run a block N times.
  • Repeat with Each — run once per item in a list: every photo you selected, every line of text, every file in a folder. Inside the loop, Repeat Item is the current item and Repeat Results collects each pass's output.

This is the engine behind batch work: select 40 photos in the share sheet → Remove Background inside a Repeat with Each → save all results to a folder. Chapter 9 builds it in full.

Variables, when magic isn't enough

Magic Variables (chapter 2) cover most needs, but three explicit actions matter for bigger recipes:

ActionWhat it doesUse when
Set Variable Names the current content You'll need this value many steps later
Add to Variable Appends to a growing list Collecting results across a loop
Ask for Input Prompts you at run time (text, number, date…) The recipe needs a human decision mid-run

Text, lists & dictionaries: the data toolkit

A handful of actions handle 90% of data plumbing:

  • Text — a freeform template; drop Magic Variables into it to compose messages, filenames, notes.
  • List — a fixed set of items to pick from or loop over.
  • Dictionary — key→value pairs, the same shape as the JSON that web APIs speak. Paired with Get Contents of URL and Get Dictionary Value, it lets a shortcut call a real API and pull out fields — no code, just keys (chapter 10).
  • Split Text / Combine Text / Match Text (regex!) — slice and dice any text that flows through.
  • Format Date and Calculate Expression — dates and math; as of the 26 release, Calculate Expression also evaluates units and currency conversions.
Key Idea

If you know a scripting language, this is if/else, switch, for each, and dict[key] wearing friendly clothes. If you don't — congratulations, you're learning the real concepts of programming with none of the punctuation.
