07 Orchestration

Orchestrating: Workflows & Skills

Chapter 6 spawned one subagent. Two tools go further: Workflow coordinates many of them from a script, and Skill runs a packaged playbook. Both ask first.

Workflow: deterministic fan-out

A workflow is a script that orchestrates many subagents in the background and returns one consolidated result. Where a single subagent is model-driven, a workflow gives you deterministic control flow — loops, conditionals, fan-out — around the agents.

the shape of a workflow// fan out, then verify each finding
const found = await parallel(dimensions.map(d =>
  () => agent(d.prompt)));
const checked = await pipeline(found, verify, synth);

The building blocks: agent() spawns one subagent, parallel() runs a batch and waits for all, and pipeline() streams each item through stages without a barrier. Phases group the work so progress is legible.

◆
When it earns its cost

Workflows can spawn dozens of agents and burn a lot of tokens — so they're for work that genuinely needs scale: broad audits, migrations across many files, or "generate N approaches, judge them, synthesize the winner." For a single task, plain Agent is the right tool.

Skill: a reusable playbook

A skill bundles instructions (and optional scripts) for a recurring job — "write a book," "review this PR," "deploy." The Skill tool executes it within the main conversation, so the steps run with your full context, not in a separate window.

▣
Slash commands are skills

When you type /write-book or /code-review, you're invoking a skill through this tool. Restrict which skills Claude may run with a Skill(deploy *)-style rule, and declare a skill's own tool access in its allowed-tools frontmatter.

Three levels of delegation

ToolDelegates…
Agentone task to one fresh context
Skilla packaged procedure, in this context
Workflowmany agents, scripted, in the background
★
Compose, don't conflate

They stack naturally: a skill can decide to launch a workflow, and a workflow's stages are agents. Reach for the smallest one that fits — escalate only when the task's scale demands it.
