11 Extend & Converse

Extending with MCP & Talking to You

The last family does two jobs: it lets Claude pull in tools beyond the built-ins through MCP, and it gives Claude clean ways to ask you things and report back.

Working with MCP servers

ToolSearchno prompt

Searches for and loads deferred tools when tool search is enabled. Servers can expose hundreds of tools; rather than load every schema up front, Claude searches for the ones it needs and loads them on demand.

ListMcpResourcesToolno prompt

Lists the resources exposed by connected MCP servers.

ReadMcpResourceToolno prompt

Reads a specific MCP resource by its URI.

WaitForMcpServersno prompt

Waits for servers still connecting in the background so a request can use them without a restart. Appears only when tool search is off — otherwise ToolSearch handles the wait.

▣
How the toolbox grows

This is the answer to "can Claude do X?" when X isn't built in: connect an MCP server and its tools join the set, discoverable through ToolSearch and governed by the same permission rules as everything else.

Asking, and reporting

AskUserQuestionno prompt

Asks you multiple-choice questions to gather requirements or clear up ambiguity — for the genuine forks only a human can settle, not for things Claude can decide itself.

ReportFindingsno prompt

Reports code-review findings as a structured list — file, summary, failure scenario per finding — so the client renders them instead of printing text. Used when review instructions call for it (v2.1.196+).

Artifactasks first

Publishes an HTML or Markdown file as an artifact: a private, interactive page on claude.ai you can share in your org. Needs a Team/Enterprise plan and /login.

ShareOnboardingGuideasks first

Uploads ONBOARDING.md and returns a share link teammates open in Claude Code. Called from /team-onboarding; Pro/Max/Team/Enterprise.

★
The toolbox, end to end

Eleven chapters, one pattern: Claude reads and searches, edits and runs, watches and delegates, plans, reaches the web, schedules, and extends — and every single call passes through your permission rules first. That's the whole machine.
