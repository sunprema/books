02 Read-only

Reading & Searching the Codebase

Before Claude changes anything, it has to see the code. Four read-only tools cover it: find files by name, search inside them, read one in full, and ask a language server what a symbol means. All four run free.

Globno prompt

Find files by name pattern. **/*.ts at any depth, *.{json,yaml} for sets. Results sort by modification time and cap at 100 — if the cap trips, Claude sees a flag and narrows the pattern.

Grepno prompt

Search file contents, built on ripgrep. Three output modes: files_with_matches (default), content, count. Scope with glob or type; cross lines with multiline.

Readno prompt

Return a file's contents with line numbers (always absolute paths). Handles images as visual content, PDFs by page range, and Jupyter notebooks cell-by-cell. Files only — it lists directories with ls via Bash.

LSPno prompt

Code intelligence from a real language server: jump to definition, find references, type info, list/search symbols, find implementations, trace call hierarchies. Auto-reports type errors after each edit.

Glob vs. Grep: name vs. content

two questions, two tools# "where are the test files?" → Glob (by name)
Glob  pattern="**/*.test.ts"

# "where do we call deprecated()?" → Grep (by content)
Grep  pattern="deprecated\("  type=ts  output=content
⚠
The .gitignore gotcha

They disagree on ignored files. Grep respects .gitignore (skips ignored files; pass a path directly to force it). Glob does not — it finds ignored files too. Flip Glob with CLAUDE_CODE_GLOB_NO_IGNORE=false. That mismatch surprises people hunting for a build artifact.

▣
ripgrep regex, not POSIX

Grep uses ripgrep's syntax. Metacharacters need escaping — finding interface{} in Go takes the pattern interface\{\}. Forgetting this is the most common "why no matches?" moment.

Reading big files

A whole-file Read that exceeds the token limit returns the first page plus a PARTIAL view notice telling Claude how to continue with offset and limit. (A read that already passes an explicit offset/limit and still overflows just errors.) For a 40-page PDF, Claude reads in ranges like "1-5", up to 20 pages a call.

★
LSP needs a plugin

The LSP tool is inactive until you install a code-intelligence plugin for your language; the plugin carries the server config and you install the server binary separately. No plugin, no jump-to-definition.
