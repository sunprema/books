03 Mutating

Changing Code Safely

Three tools write to disk: Edit for surgical changes, Write for whole files, NotebookEdit for Jupyter cells. All three ask first — and all three share one guard rail that prevents Claude from clobbering a file it hasn't actually seen.

Edit: exact string replacement

Edit swaps an old_string for a new_string. No regex, no fuzzy matching — what you write is what it looks for. Three checks must pass, in order:

  1. Read-before-edit — Claude must have read the file this conversation, and it must be unchanged on disk since. This runs first, before any string matching.
  2. Match — old_string must appear exactly, down to a single space of indentation.
  3. Uniqueness — it must appear exactly once. Otherwise add surrounding context to pin one spot, or set replace_all: true.
★
Viewing counts as reading

You don't strictly need the Read tool first. Viewing a file with Bash cat, head, tail, sed -n 'X,Yp', grep, egrep, or fgrep — on a single file, no pipes or redirects — also satisfies read-before-edit. Pipe the output and it no longer counts.

a unique-match editEdit
  old_string: "timeout = 30"
  new_string: "timeout = 60"
  # fails if "timeout = 30" appears twice —
  # add context or replace_all: true

Write: the whole file, or nothing

Write creates a new file or overwrites an existing one with the full content. It never appends or merges. If the path already exists, Claude must have read it at least once first — a Write to an unread existing file errors. New files are exempt. For a small change, the right tool is Edit, not a re-Write.

⚠
Overwrite, not patch

Because Write replaces everything, handing it partial content silently drops the rest of the file. The read-before-overwrite rule exists precisely to stop Claude from blowing away content it never saw.

NotebookEdit: one cell at a time

Jupyter notebooks aren't plain text, so they get their own tool. It targets a cell by cell_id with three modes:

ModeEffect
replaceOverwrite the cell's source (default)
insertAdd a cell after the target (or at the top with no cell_id); needs cell_type
deleteRemove the target cell

Its permission rules use the Edit(...) path format — Edit(notebooks/**) covers NotebookEdit calls there too.
