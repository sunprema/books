10 Time & Reach

Scheduling & Staying in Touch

Some work happens later, or while you're away. This family lets Claude schedule itself, pace its own loops, and reach you when something's worth your attention. Every tool here runs free.

Scheduling within a session

CronCreateno prompt

Schedule a recurring or one-shot prompt within the current session. Tasks are session-scoped and restored on --resume/--continue if unexpired.

CronList · CronDeleteno prompt

List every scheduled task in the session, and cancel one by ID.

ScheduleWakeupno prompt

Reschedules the next iteration of a self-paced /loop. Claude calls it at the end of each pass to choose when the next runs — one minute to one hour out. You never call it directly.

⚠
Session vs. cloud

These crons live in the session — close it and they stop. For durable cloud schedules that outlive the terminal, that's RemoteTrigger (next page). On Bedrock/Vertex/Foundry, ScheduleWakeup is unavailable, so a no-interval /loop falls back to a fixed schedule.

Reaching you

PushNotificationno prompt

Send a desktop notification — and a phone push when Remote Control is connected — so a long task or scheduled run can reach you when you step away.

SendUserFileno prompt

Send a file from the session to your device with an optional caption — a report, diagram, screenshot, or built artifact. As of v2.1.196 a display input picks render (inline), attach (download card), or auto.

RemoteTriggerno prompt

Create, update, run, and list Routines on claude.ai — the durable cloud schedules behind the /schedule command.

SendMessageno prompt

Message an agent-team teammate, or resume a subagent by its agent ID. Stopped subagents auto-resume in the background.

★
Hosted-only, by nature

Push, file delivery, and Routines all run through Anthropic-hosted infrastructure, so they're not available on Amazon Bedrock, Google Vertex AI, or Microsoft Foundry. Routines additionally need a Pro, Max, Team, or Enterprise plan.
