01 Foundation

The Toolbox & the Permission Model

Claude Code can't touch your machine directly. Everything it does — reading a file, running a test, fetching a URL — happens through a tool: one of a fixed set of named capabilities, each call checked against your rules before it runs.

You rarely name tools when you chat; Claude picks them. You name them when you decide what it's allowed to do. The same name shows up in six places:

  • permissions.allow / permissions.deny in settings, and the /permissions UI
  • the --allowedTools / --disallowedTools CLI flags
  • the Agent SDK's allowedTools / disallowedTools
  • a subagent's tools / disallowedTools frontmatter
  • a skill's allowed-tools frontmatter
  • a hook's if condition (matchers use the bare name)
★
The signature distinction

Every tool is one of two kinds: it either runs free or it asks first. Reads, searches, and listing are free; anything that changes your machine or reaches out — Bash, Edit, Write, WebFetch — prompts you the first time. We mark this badge on every tool in this book.

Adding and extending tools

The built-in set is fixed, but you can grow it. Connect an MCP server to add brand-new tools (chapter 11). To package a reusable prompt-driven workflow, write a skill — it runs through the existing Skill tool rather than adding a new entry.

A deny rule always wins; an allow rule skips the prompt; otherwise the tool's default decides.

One rule format: ToolName(specifier)

Most rules share a shape. The specifier depends on the tool, and several tools share a format:

RuleApplies to
Bash(npm run *)Bash, Monitor
Read(~/secrets/**)Read, Grep, Glob, LSP
Edit(/src/**)Edit, Write, NotebookEdit
Skill(deploy *)Skill
Agent(Explore)Agent
WebFetch(domain:x.com)WebFetch
WebSearchno specifier — all or nothing
▣
Two shortcuts worth knowing

An Edit(...) allow rule also grants read on that path — no matching Read(...) needed. And tools with no specifier (like ExitPlanMode) take just the bare name. Hook matcher fields always use bare names, never the parenthesized form.
