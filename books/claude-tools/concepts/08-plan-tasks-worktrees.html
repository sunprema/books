08 Working in the Large

Planning, Tasks & Worktrees

For anything bigger than a quick edit, Claude has tools to think before it acts, track the work, and isolate it from your live tree. Most run free; one asks first.

Plan mode

EnterPlanModeno prompt

Switch into plan mode to design an approach before writing any code — research and reason, no edits.

ExitPlanModeasks first

Present the finished plan for your approval and leave plan mode. The one prompt here is the plan itself — you approve before code happens.

Isolated git worktrees

EnterWorktreeno prompt

Create an isolated git worktree and switch into it, so changes never touch your working tree. Pass a path to re-enter an existing one instead.

ExitWorktreeno prompt

Leave the worktree session and return to the original directory.

▣
Restricted inside a worktree

From within a worktree — or a subagent pinned with isolation: worktree — only the path form of EnterWorktree is allowed, and the target must live under .claude/worktrees/. ExitWorktree isn't offered to a subagent that already runs in its own directory.

The task list

Five tools manage the structured task list that tracks multi-step work. All run free:

ToolDoes
TaskCreateadd a task
TaskGetfull details for one task
TaskListall tasks + status
TaskUpdatestatus, dependencies, details — or delete
TaskStopkill a running background task by ID
⚠
Two deprecations to know

TaskOutput is deprecated — prefer Read on the task's output-file path. And TodoWrite (the older session checklist) is disabled by default as of v2.1.142 in favor of the Task* tools; set CLAUDE_CODE_ENABLE_TASKS=0 to bring it back.

★
Why this trio belongs together

They're the scaffolding for large work: plan the approach, track the steps as tasks, and do the risky edits inside a worktree you can throw away. A migration across fifty files uses all three at once.
