04 Execution

Running Commands

When Claude needs to build, test, or run anything, it reaches for Bash (or PowerShell on Windows). Both ask first. The trick to using them well is knowing exactly what doesn't carry over between calls.

Each command is its own process

Bash runs every command fresh. That has three consequences people trip on:

  • cd carries over — but only inside the project or an added working directory. Wander outside and Claude Code resets you, appending Shell cwd was reset to <dir>.
  • Environment variables do not — an export in one command is gone by the next. Persist them with CLAUDE_ENV_FILE or a SessionStart hook.
  • Aliases and functions do — at startup Claude Code sources your ~/.zshrc/~/.bashrc/~/.profile and applies them to every command.
▣
Pin the directory

Want every command to start in the project root regardless of prior cd? Set CLAUDE_BASH_MAINTAIN_PROJECT_WORKING_DIR=1.

Two limits bound every command

LimitDefaultCeiling
Timeout2 min10 min (timeout param)
Output30,000 chars150,000 (BASH_MAX_OUTPUT_LENGTH)

Past the output limit, Claude Code saves the full text to a file and hands Claude the path plus a short preview — it reads or greps that file for the rest.

a long job, run in the background❯ npm run dev      # run_in_background: true
# Claude keeps working; watch & stop it with /tasks
★
Don't block on a server

For dev servers or watch builds, Claude sets run_in_background: true so the process starts as a background task and the conversation continues. List and stop background tasks with /tasks.

PowerShell: the native Windows path

The PowerShell tool runs commands natively instead of routing through Git Bash. On Windows without Git Bash it's on automatically; with Git Bash it's rolling out; on Linux, macOS, and WSL it's opt-in and needs pwsh 7+ on your PATH.

enable it in settings.json{
  "env": {
    "CLAUDE_CODE_USE_POWERSHELL_TOOL": "1"
  }
}

Three settings steer where it's used: "defaultShell": "powershell" for interactive ! commands, "shell": "powershell" on a hook, and shell: powershell in skill frontmatter. Claude Code spawns PowerShell with -ExecutionPolicy Bypass at process scope only, so enterprise Group Policy still applies.

⚠
Preview caveats

During preview, PowerShell profiles aren't loaded and sandboxing isn't supported on Windows. As of v2.1.196 it matches Bash's handling of search/diff exit codes, so a no-match grep isn't reported as a failure.
