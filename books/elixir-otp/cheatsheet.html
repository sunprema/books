Processes (Ch. 01)

pid = spawn(fn -> ... end)      # start; returns a PID
pid = spawn_link(fn -> ... end)  # start + link (shared fate)
send(pid, msg)                    # drop msg into pid's mailbox
receive do
  pattern -> ...
after 1_000 -> ...             # optional timeout (ms)
end
self()                           # my own PID

Let it crash (Ch. 02)

WireDirectionOn death
spawn_linktwo-waysignal propagates — you die too
Process.monitor(pid)one-way{:DOWN, ref, :process, pid, reason} message
Process.flag(:trap_exit, true)   # turn EXIT signals into messages
# then: {:EXIT, from, reason} arrives in your mailbox

GenServer (Ch. 03)

CallbackHandlesCommon return
init/1startup{:ok, state}
handle_call/3sync (waits){:reply, val, state}
handle_cast/2async{:noreply, state}
handle_info/2raw msgs, timers{:noreply, state}
handle_continue/2post-init work{:noreply, state}
terminate/2cleanup(best effort)
GenServer.start_link(mod, arg, name: __MODULE__)
GenServer.call(server, req, timeout \\ 5000)  # waits for reply
GenServer.cast(server, req)                    # :ok immediately
Rule

One message at a time — keep callbacks fast; prefer call for back-pressure; offload slow work to a Task.

Supervisor (Ch. 04)

children = [{Stack, ["a"]}, MyApp.Cache]
Supervisor.init(children, strategy: :one_for_one)
StrategyOn a child crash…
:one_for_onerestart just that child (default)
:one_for_allrestart every child
:rest_for_onerestart it + all started after it
restart:When restarted
:permanentalways (default)
:transientonly on abnormal exit
:temporarynever

max_restarts: 3, max_seconds: 5 — exceed it and the supervisor gives up and escalates to its supervisor.

DynamicSupervisor (Ch. 05)

{DynamicSupervisor, name: MyApp.Sup, strategy: :one_for_one}
DynamicSupervisor.start_child(MyApp.Sup, {Worker, arg})
DynamicSupervisor.terminate_child(MyApp.Sup, pid)
# init: strategy: :one_for_one, max_children: 10_000

Registry (Ch. 06)

{Registry, keys: :unique, name: MyApp.Registry}
# self-register via a :via tuple as the process name:
name = {:via, Registry, {MyApp.Registry, key}}
Registry.lookup(MyApp.Registry, key)  #=> [{pid, value}] | []

:unique = one PID per key (sessions) · :duplicate = many (pub/sub). Entries auto-clear on death.

Task & Agent (Ch. 07)

# Task — concurrency
t = Task.async(fn -> work() end);  Task.await(t, 10_000)
Task.async_stream(list, &fun/1, max_concurrency: 20)
{Task.Supervisor, name: MyApp.TaskSup}      # supervise fire-&-forget

# Agent — state
Agent.start_link(fn -> 0 end, name: C)
Agent.get(C, & &1);  Agent.update(C, &(&1 + 1))

Application (Ch. 08)

# mix.exs
mod: {MyApp.Application, []}

# lib/my_app/application.ex
use Application
def start(_type, _args) do
  Supervisor.start_link(children, strategy: :one_for_one, name: MyApp.Supervisor)
end
The mental model

Isolated processes that crash cleanly, wrapped as GenServers, arranged into supervision trees (static + dynamic), addressed through a Registry, sized with Task/Agent, all rooted in one Application. Fault tolerance isn't code you write — it's a tree you shape.
