The 30-Second Memory Cheatsheet

Everything in one place — the mental model, the four bugs, and how the five languages each answer "who frees the heap?"

Stack vs Heap

Stackauto, LIFO, ~MBs, dies on return, 1-instruction alloc/free
Heapmanual/GC, any order, GBs, lives until freed, search to alloc

Rule of thumb: known size + dies with the call → stack. Else → heap.

The 4 manual bugs

  • Leak — never freed → memory creeps up
  • Use-after-free — touch a freed shelf
  • Double-free — free twice, corrupt heap
  • Dangling/null — pointer outlives its target

~70% of serious CVEs are memory-safety bugs.

How GC decides

  • Liveness = reachability from roots (stack + globals)
  • Tracing: mark reachable → sweep the rest. Handles cycles.
  • Refcounting: free at count 0. Prompt, but leaks cycles.
  • Generational: most objects die young → collect the young often.

RUST ownership

let s2 = s1; // move; s1 invalid
&T   // many readers
&mut T // one writer
Box Rc Arc Weak

Compiler inserts drop. No GC, no pause, deterministic.

ZIG allocators

const b = try a.alloc(u8,n);
defer a.free(b);

No global allocator — pass it in. defer frees on every exit. Arena = free all at once. DebugAllocator catches leaks.

JAVA the JVM

  • Tracing, generational (Eden→Survivor→Old)
  • G1 default · ZGC <1 ms (generational by default, JDK 25)
  • Concurrent collectors use read/write barriers

PYTHON two collectors

  • Refcounting frees the 99% instantly
  • Generational cycle GC mops up cycles
  • GIL guarded refcounts; free-threaded build (3.13/3.14) makes it optional

GO low-pause GC

  • Concurrent tri-color mark-sweep, hybrid write barrier
  • Sub-ms pauses; tune with GOGC (def 100)
  • Escape analysis keeps non-escaping values on the stack

Who frees the heap?

C / Zigyou (Zig: defer)

Choosing

  • Control / no runtime → C, Zig, Rust
  • Proven safety + speed → Rust
  • Productivity at scale → Java, Go
  • Speed of thought → Python
★
The one-liner

Memory management is lifetime management. The stack frees itself; the heap is freed by you, the compiler, or a collector — pick which bill you'll pay (latency, memory, CPU, or dev time).
