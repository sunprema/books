The three rules people forget

① Only nil and false are falsy — 0 and "" are true. ② Tables index from 1. ③ Undeclared assignments are global; local everything.

Tables & iteration

local t = {10, 20, x = 99}
t[1]        -- 10        (array part)
t.x         -- 99        (hash part)
for i,v in ipairs(t) do end   -- 1..n, stops at nil
for k,v in pairs(t)  do end   -- all keys, any order
table.insert(t, v); table.remove(t)
table.sort(t, function(a,b) return a>b end)

Metatables & objects

M.__index = M
setmetatable(obj, M)          -- missing keys fall through to M
function M:method() return self.x end   -- `:` adds self
obj:method()                  -- == M.method(obj)
-- common metamethods: __index __newindex __call __tostring __add __eq

Coroutines

local co = coroutine.create(fn)
coroutine.resume(co, args)    -- run/continue -> true, yielded...
coroutine.yield(v)            -- pause, hand v back
coroutine.wrap(fn)            -- as a callable generator
coroutine.status(co)          -- suspended | running | dead

Embedding — the lua library

{:lua, "~> 0.4"}                       # mix.exs
Lua.new()                              # sandboxed VM
Lua.eval!(lua, "return 2+2")           # => {[4], lua}
~LUA[return 2+2]c                      # precompiled chunk
Lua.set!(lua, [:a, :b], val)           # put (nested path)
Lua.get!(lua, [:a, :b])                # read back
Lua.set!(lua, [:f], fn args -> [..] end)   # expose a fn
Lua.load_api(lua, MyAPI)               # deflua module
Lua.call_function!(lua, [:string,:upper], ["hi"])
Lua.put_private/get_private!           # host-only secrets
defmodule MyAPI do
  use Lua.API, scope: "kit"
  deflua double(v), do: 2 * v
  deflua peek(), state do Lua.get_private!(state, :key) end
end

Gotcha grab-bag

No ++/+=/continue · 1 == "1" is false · don't store nil inside a sequence · keep arrays dense + objects string-keyed across the bridge · Luerl trails C-Lua on speed & newest 5.5 features — do hot loops in Elixir.

Lua 5.5 (Dec 2025), what's new

Optional global declarations (catch typos at compile time), ~60% smaller arrays, incremental major GC, named vararg tables, read-only for variables. Integers (64-bit) have been standard since 5.3.
