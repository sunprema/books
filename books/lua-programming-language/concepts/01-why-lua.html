Chapter One

Why Lua?

Lua isn't trying to be the language your app is written in. It's the language you hand to your users — a tiny, fast, embeddable interpreter that lets people extend your program without touching its source or your servers' shell.

Born in 1993 at PUC-Rio in Brazil, Lua ("moon" in Portuguese) was designed from day one as an extension language: something you embed inside a bigger host application. That heritage explains everything good about it. The whole interpreter is a few hundred kilobytes. It has eight types, one real data structure, and a syntax you can learn over a coffee.

You've already run Lua without knowing it. It scripts Neovim, Roblox, World of Warcraft, Redis, Nginx (OpenResty), and countless game engines. The pattern is always the same: a performance-critical host written in C, Rust, or — in our case — Elixir, with Lua bolted on as the safe, hot-swappable logic layer.

The host-and-script pattern

Keep this mental model for the whole book. Your application is the host. It owns the database, the network, the secrets. Lua is the guest: it runs logic you (or your users) write, but only through doors the host opens for it.

Key idea

Choose Lua when you need untrusted or frequently-changing logic to run inside a trusted host: business rules, automations, game behavior, config that's really code. If you instead need a general application language, that's what Elixir is for — Lua complements it, it doesn't replace it.

Why it pairs so well with Elixir

Elixir runs on the BEAM, a VM built for millions of isolated, supervised processes. Lua is built to be embedded and sandboxed. Put them together and each Lua script can run inside its own BEAM process — crashable, time-limitable, memory-limitable — giving you safe multi-tenant scripting almost for free. We'll build exactly that by the end of the book.

On the BEAM, Lua is pure Elixir

The Elixir lua library wraps Luerl — a Lua 5.x implementation written in Erlang. There's no C interpreter, no NIF, no port. A Lua VM is just an Erlang term you can copy, store, and pass between processes. That's a superpower we'll lean on hard.

What you'll be able to do by the end

  • Read and write idiomatic Lua 5.5 — tables, metatables, coroutines.
  • Embed a sandboxed Lua VM in Elixir and pass values both ways.
  • Expose your own Elixir functions to Lua with deflua.
  • Let Lua scripts drive your Ash domain — safely, with the actor and tenant enforced.
  • Ship real features: user-defined rules, programmable automations, and config-as-code.

Sources: lua.org (official site & manual), Programming in Lua (Ierusalimschy). Luerl: github.com/rvirding/luerl.
