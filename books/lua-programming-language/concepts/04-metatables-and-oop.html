Chapter Four

Metatables & OOP

A metatable is a table that describes how another table behaves: what happens when you add two of them, index a missing key, or print one. It's Lua's single mechanism for operator overloading, inheritance, and proxies — all from the same small idea.

local Vector = {}
Vector.__index = Vector          -- look up missing keys on Vector itself

function Vector.new(x, y)
  return setmetatable({x = x, y = y}, Vector)
end

function Vector.__add(a, b)      -- overload the + operator
  return Vector.new(a.x + b.x, a.y + b.y)
end

function Vector.__tostring(v)
  return string.format("(%g, %g)", v.x, v.y)
end

local sum = Vector.new(1, 2) + Vector.new(3, 4)
print(sum)                       -- (4, 6)   via __tostring

The metamethods you'll actually use

MetamethodFires when…
__indexreading a key the table doesn't have
__newindexwriting a key the table doesn't have
__callcalling the table like a function
__tostringtostring() / print()
__add __sub __eq __lt __concatthe matching operator

How __index powers everything

When you read t.foo and t has no foo, Lua consults t's metatable. If __index is a table, it looks foo up there; if it's a function, it calls it. That one rule gives you defaults, lazy fields, read-through caches, and class inheritance.

Objects with the colon

Remember s:upper() from Chapter Two? The colon is method sugar: obj:m(a) means obj.m(obj, a). It passes the receiver as a hidden first parameter named self.

local Account = {}
Account.__index = Account

function Account.new(balance)
  return setmetatable({balance = balance or 0}, Account)
end

function Account:deposit(amount)   -- `:` defines self implicitly
  self.balance = self.balance + amount
  return self                      -- enables chaining
end

function Account:withdraw(amount)
  assert(amount <= self.balance, "insufficient funds")
  self.balance = self.balance - amount
  return self
end

local a = Account.new(100)
a:deposit(50):withdraw(30)
print(a.balance)                   -- 120

Inheritance is one more __index

local Savings = setmetatable({}, {__index = Account})
Savings.__index = Savings

function Savings:addInterest(rate)
  return self:deposit(self.balance * rate)   -- reuse the parent method
end

local s = setmetatable({balance = 1000}, Savings)
s:addInterest(0.05)
print(s.balance)                   -- 1050
Use it sparingly

Metatables are powerful enough to reinvent a whole object system — and people have, a dozen incompatible ways. For embedded scripting, prefer plain tables and free functions. Reach for metatables when you genuinely need operators, defaults, or a guarded proxy (a great way to make a "read-only" config table for untrusted scripts).

Source: Lua 5.5 Reference Manual §2.4 (metatables and metamethods).
