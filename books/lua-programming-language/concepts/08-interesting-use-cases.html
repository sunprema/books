Chapter Eight

Interesting Use Cases

Theory is done; now the fun part. Each pattern below is something genuinely hard to build without an embedded scripting layer — and almost easy with Lua sitting inside Elixir & Ash.

1 · User-defined business rules

Every SaaS eventually gets the request: "let us decide when an order is flagged." Hard-coding that means a deploy per customer. Instead, store each tenant's rule as Lua and run it against the record.

-- tenant-authored rule: returns true to flag the order
return order.total > 5000
   and order.country ~= "US"
   and not customer.verified
def flag?(order, customer, rule_src) do
  {[flagged?], _} =
    Lua.new()
    |> Lua.set!([:order], order)
    |> Lua.set!([:customer], customer)
    |> Lua.eval!(rule_src)
  flagged? == true
end
Why Lua and not "config"

Rules engines start as JSON and inevitably grow conditionals, math, and string handling until you've built a bad programming language. Lua is the small, safe programming language — skip the reinvention.

2 · Programmable automations (Zapier-in-a-box)

Give users a script that fires on an event and can act on your domain through ash_lua — with their own actor, so they can only touch their own data.

-- "when a high-value deal closes, create a follow-up task"
local deal = event.record
if deal.amount >= 10000 then
  assert(crm.task.create({
    title = "VIP onboarding: " .. deal.name,
    due   = "in 2 days",
    fields = { "id" }
  }))
end

3 · Game & simulation logic, hot-reloaded

This is Lua's home turf. Run each entity's behavior as a coroutine (Chapter Five), resume it once per tick, and reload the script without restarting the server — designers iterate while the world runs.

function on_tick(npc, world)
  if world.threat > npc.courage then
    return "flee"
  elseif distance(npc, world.player) < 2 then
    return "attack"
  end
  return "patrol"
end

4 · Safe formula & template fields

Spreadsheets-style computed columns, invoice templates, alert message bodies — anywhere a user wants a little expression evaluated against their data. A one-line eval! with the row set as globals does it, and the sandbox guarantees the formula can't escape.

return string.format("Hi %s, you have %d items ($%.2f)",
  customer.name, cart.count, cart.subtotal)

5 · A REPL / scripting console for power users

Expose a curated API and let advanced customers automate bulk work themselves — backfills, one-off migrations, reports — instead of filing support tickets that land on your engineers.

local stale = assert(projects.project.read({
  filter = { archived = false, last_seen_before = "90d" },
  fields = { "id", "name" }
}))
for _, p in ipairs(stale) do
  assert(projects.project.update({ id = p.id, archived = true }))
end
print("archived " .. #stale .. " projects")
The throughline

Every example is the same shape: logic that should change without a deploy, written by someone you don't fully trust, running against data you must protect. Lua-in-Elixir turns that from a security project into a feature you ship on a Tuesday.
