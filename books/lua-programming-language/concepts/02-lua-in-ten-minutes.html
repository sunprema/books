Chapter Two

Lua in Ten Minutes

Let's run something first, then explain it. Here is enough Lua to read most scripts you'll meet — variables, functions, control flow, and the truthiness rules that trip up every newcomer.

-- variables are global by default; ALWAYS prefer `local`
local name = "Ada"
local count = 42          -- integer (Lua 5.3+)
local ratio = 3.14        -- float
local ready = true
local nothing = nil       -- absence of a value

-- string concatenation uses .. (not +)
print("hello, " .. name)  -- hello, Ada
print(#name)              -- 3   (# is "length")
Foot-gun #1: globals

A bare x = 1 creates a global. Forget a local and you've leaked state across your whole script. Type local until it's muscle memory. (Lua 5.5 can finally make this an error — see the cheatsheet.)

Functions are values

Functions are first-class: store them in variables, pass them around, return them. This is the bedrock that makes Lua a good scripting language.

local function add(a, b)
  return a + b
end

-- same thing, written as an assignment
local mul = function(a, b) return a * b end

-- multiple return values are built in
local function minmax(t)
  return math.min(table.unpack(t)), math.max(table.unpack(t))
end

local lo, hi = minmax({4, 9, 1, 7})
print(lo, hi)             -- 1   9

Truthiness: only nil and false are false

This is the single most important rule in the language. Zero is true. The empty string is true. Everything except nil and false passes an if.

if 0 then print("0 is truthy!") end        -- prints
if "" then print("'' is truthy!") end      -- prints
if nil then print("never") end             -- skipped

-- the `and`/`or` idiom: a default value
local port = config.port or 8080           -- if nil/false, use 8080
-- a ternary, Lua-style:
local label = ready and "go" or "wait"
Foot-gun #2: the and/or ternary

cond and A or B works unless A itself is false or nil — then you always get B. Fine for defaults; dangerous for booleans. When in doubt, use a real if.

Control flow

-- numeric for: start, stop (inclusive), optional step
for i = 1, 5 do print(i) end       -- 1 2 3 4 5
for i = 10, 0, -2 do print(i) end  -- 10 8 6 4 2 0

-- while / repeat
local n = 3
while n > 0 do n = n - 1 end
repeat n = n + 1 until n >= 3      -- runs at least once

-- there is no `continue`; use goto or an if
for i = 1, 5 do
  if i % 2 == 0 then goto next end
  print(i)                         -- 1 3 5
  ::next::
end

-- not-equal is ~=, not !=
if name ~= "Bob" then print("not bob") end
Things that surprise newcomers

No ++ or +=. Comments are -- (and --[[ ... ]] for blocks). Arrays start at 1, not 0. Comparing different types with < is an error, but 1 == "1" is simply false (no coercion in equality).

Strings come with a library

local s = "Programming in Lua"
print(s:upper())                   -- method-call syntax on strings
print(s:sub(1, 11))                -- "Programming"
print(string.format("%d%%", 87))   -- "87%"
print(("a,b,c"):gsub(",", ";"))    -- a;b;c   2

That s:upper() colon is the same OOP sugar we'll meet with tables: it passes s as a hidden first argument. Hold that thought — it returns in Chapter Four.

Source: Lua 5.5 Reference Manual, §3 (statements) and §6 (standard libraries).
