Chapter Three

Tables: The One Data Structure

Lua has exactly one container type: the table. Arrays, dictionaries, objects, namespaces, modules — all of them are tables wearing different hats. Learn the table and you've learned Lua's whole data model.

-- a table is an associative array: keys → values
local t = {}
t["name"] = "Lua"
t.version = 5.5            -- t.x is sugar for t["x"]
t[1] = "first"

-- build it all at once with a constructor
local person = {
  name = "Grace",         -- string key
  age = 85,
  hobbies = {"compilers", "sailing"},   -- a nested table
}
print(person.name, #person.hobbies)     -- Grace   2

The same table is an array and a map

Integer keys 1, 2, 3… form the array part (the "sequence"); everything else lives in the hash part. One table holds both at once.

local mixed = {"a", "b", "c", color = "blue"}
print(mixed[1], mixed.color)   -- a   blue
print(#mixed)                  -- 3  (length of the sequence part)
Foot-gun: 1-based, and beware the hole

Sequences start at index 1. The length operator # is only well-defined for sequences with no gaps. Put a nil in the middle (t[2] = nil) and #t may return 1 or 3 — it's allowed to find any "border". Don't store nil inside a sequence.

Iterating: ipairs vs pairs

This distinction matters constantly when you marshal data between Lua and Elixir.

IteratorVisitsOrder
ipairs(t)only 1,2,3… until the first nilguaranteed ascending
pairs(t)every key (array + hash)unspecified
 
local cfg = {"x", "y", debug = true}

for i, v in ipairs(cfg) do print(i, v) end
-- 1  x
-- 2  y        (stops; never sees `debug`)

for k, v in pairs(cfg) do print(k, v) end
-- 1  x  /  2  y  /  debug  true   (some order)
Why this bites you across the bridge

When Lua hands a table to Elixir, the library can't know if you meant a list ["x","y"] or a map %{1 => "x", 2 => "y"}. A clean sequence usually decodes to a list; a mixed or sparse table decodes to a map of {key, value} pairs. Keep your "arrays" dense and your "objects" string-keyed and you'll never be surprised.

The table library

local q = {}
table.insert(q, "a")          -- push
table.insert(q, "b")
table.insert(q, 1, "z")       -- insert at position 1 -> {z, a, b}
local last = table.remove(q)  -- pop -> "b"
print(table.concat({"a","b","c"}, "-"))   -- a-b-c

-- sort in place, optionally with a comparator
local nums = {3, 1, 2}
table.sort(nums)                          -- {1, 2, 3}
table.sort(nums, function(a, b) return a > b end)  -- {3, 2, 1}

Tables as namespaces

A Lua "module" is just a table of functions you return. This is exactly how the standard library (string, math, table) is organized — and how you'll expose your Elixir API to scripts later.

local geometry = {}
function geometry.area(w, h) return w * h end
function geometry.perimeter(w, h) return 2 * (w + h) end
return geometry   -- a script that `require`s this gets the table
Key idea

Everything structural in Lua is a table: data, modules, classes, even the global environment _G itself. Master the table and the rest of the language is small.

Source: Lua 5.5 Reference Manual §3.4.7 (the length operator), §6.6 (table library).
