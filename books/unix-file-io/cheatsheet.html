Quick Reference

The Unix File I/O Cheatsheet

Every syscall, flag, and gotcha from the book on one page. POSIX / Linux, grounded in the man-pages.

📁 The four verbs

int     open(path, flags, mode);
ssize_t read(fd, buf, count);
ssize_t write(fd, buf, count);
int     close(fd);

Everything is a stream of bytes: files, terminals, pipes, sockets — same four verbs for all of them.

🎫 File descriptors

0STDIN_FILENO
1STDOUT_FILENO
2STDERR_FILENO

A small int; index into the process's private descriptor table. Kernel always hands back the lowest free number.

🔑 open() flags

Access mode (pick exactly one — not a bitmask):

  • O_RDONLY / O_WRONLY / O_RDWR

OR in extras:

  • O_CREAT create if missing (needs mode)
  • O_TRUNC · O_APPEND · O_EXCL

final mode = mode & ~umask.

⚠️ errno on failure

ENOENTno such file
EACCESpermission denied
EEXISTO_CREAT|O_EXCL, exists
EMFILEtoo many open fds
EINTRinterrupted — retry
ESPIPEcan't seek this fd

📏 Short reads & writes

  • Return < requested is normal, not an error.
  • read returning 0 = true EOF.
  • Always loop until done or 0/-1.
while (sent < n) {
  w = write(fd, p+sent, n-sent);
  if (w==-1 && errno==EINTR) continue;
  if (w==-1) return -1;
  sent += w;
}

🗂 The three tables

descriptor tableper process
open file descriptionper open · holds offset
inodeper file on disk

2× open() same path → separate offsets. fork/dup → shared offset.

📍 lseek()

off_t lseek(fd, offset, whence);
SEEK_SETfrom byte 0
SEEK_CURfrom current pos
SEEK_ENDfrom end of file

Seeking past EOF + writing later → a hole, reads as \0. Pipes/sockets: ESPIPE.

🧵 stdio buffering

_IOFBFfull — flush when full
_IOLBFline — flush on \n
_IONBFnone — flush every call

stdout: line-buffered on a terminal, fully buffered on a file/pipe. stderr: always unbuffered. Crash ⇒ unflushed bytes lost — use fflush.

🔀 dup2 & redirection

int dup2(oldfd, newfd);
fd = open("out.txt", O_WRONLY|O_CREAT|O_TRUNC, 0644);
dup2(fd, 1); close(fd);

Closes newfd, then aliases it to oldfd's open file description — atomically.

🚿 Pipes

int p[2]; pipe(p);  // p[0]=read p[1]=write
  • No writers left → read returns 0 (EOF).
  • No readers left → write ⇒ SIGPIPE/EPIPE.
  • Writes ≤ PIPE_BUF (4096 on Linux) are atomic.

⏱ 30-second mental model

  • A file = bytes + an offset you can move.
  • fd → open file description (offset) → inode (data).
  • Short reads/writes are normal; loop.
  • stdio buffers in your memory; the kernel already has what you write().
  • dup2 + pipe + fork = every shell redirection and pipeline.
