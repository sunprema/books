Chapter 05 · Templates

Control Flow

You already know if/for/promises from every other framework — Svelte's twist is that these are real template blocks, not JSX expressions, so they read like the language they're branching on.

#if — branching

{#if porridge.temp > 100}
  <p>too hot!</p>
{:else if porridge.temp < 80}
  <p>too cold!</p>
{:else}
  <p>just right!</p>
{/if}

Blocks don't have to wrap whole elements — they can wrap bare text inside one, same as any other template tag.

#each — lists, and the key that matters

{#each todos as todo, i (todo.id)}
  <li>{i + 1}. {todo.text}</li>
{/each}

The (todo.id) is the key — it must uniquely identify each item. Without one, Svelte matches by position and may reuse the wrong DOM node when the array reorders (an input mid-list can end up showing the wrong value). With one, Svelte moves the actual node instead of patching it in place — which is also what makes animate:flip possible in Chapter 08. Destructuring works too: {#each items as { id, name }, i (id)}. An empty array falls through to an optional {:else} clause.

#await — promises, inline

{#await fetchUser(id)}
  <p>loading…</p>
{:then user}
  <p>{user.name}</p>
{:catch err}
  <p>failed: {err.message}</p>
{/await}

Three branches for the three states a Promise can be in. Skip the pending state with the shorthand {#await promise then user}, or handle only the failure case with {#await promise catch err}. If the expression isn't a Promise at all, Svelte just renders the :then branch immediately.

#key — force a fresh instance

{#key user.id}
  <Avatar {user} />
{/key}

Destroys and recreates its contents whenever the key expression changes — the escape hatch for "I want this to fully reset, not just update," e.g. replaying a mount transition per user.

Always key your #each Skipping the key is fine for a static list that never reorders. The moment items can be added, removed, or reordered — and they usually can — key it. It's cheap insurance against subtle state bugs.

Try it

Render a list of names with {#each names as name, i (name)}, each wrapped in an <input value={name}>. Shuffle the array with a button. Then remove the key and shuffle again — watch an input's typed text "stick" to the wrong name.
