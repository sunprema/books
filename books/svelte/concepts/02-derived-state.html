Chapter 02 · Computed Values

$derived

Component bodies run once. Any value you compute from state needs to be told to recompute — that's what $derived is for. Think of it as a formula cell in a spreadsheet: declare the relationship once, and it never goes stale.

let count = $state(0);
let doubled = $derived(count * 2);

That's it — no dependency array to get wrong. Svelte tracks every reactive value read synchronously inside the expression and reruns it exactly when one of them changes. The expression must stay free of side effects: no mutating other state, no fetch calls. If the computation needs more than one line, reach for $derived.by:

let numbers = $state([1, 2, 3]);
let total = $derived.by(() => {
  let sum = 0;
  for (const n of numbers) sum += n;
  return sum;
});

$derived(expr) is literally sugar for $derived.by(() => expr) — reach for whichever reads better.

Recompute, not re-render

Two things make this cheap. First, Svelte uses push-pull reactivity: a write to count immediately marks doubled dirty (push), but the new value isn't actually computed until something reads it (pull) — often just before the DOM updates. Second, if a derivation recomputes to the same value it produced before, Svelte skips updating anything downstream. Referential equality, not just "did the inputs change," is what decides whether the ripple continues.

Not deeply reactive, but destructurable A $derived value doesn't get its own proxy the way $state objects do — but destructuring one (let { a, b } = $derived(obj)) still preserves reactivity for a and b individually.

Don't shadow $derived with $effect

If you catch yourself writing $effect(() => { doubled = count * 2; }), stop — that's a derivation wearing an effect's clothes. It works, but it's an extra tick, an extra dependency to track by hand, and a step toward the "effect chains" that make Svelte 4 apps hard to trace. Reach for $derived whenever the value is purely a function of other state.

Since 5.25, a non-const derived can be temporarily overridden — reassigned directly — which is handy for optimistic UI: show the new value immediately, then let the next real recompute correct it if the server disagrees. Use untrack() from svelte when you need to read state inside a derivation without adding it as a dependency.

Try it

Write let isEven = $derived(count % 2 === 0) next to the counter from Chapter 01. Click the button a few times and watch isEven flip — you never told it when to update.

The tell that you need $derived instead of $effect: ask "does this value describe other state, or does it do something?" Describing is derived. Doing is effect. Ninety percent of bugs I've seen in Svelte 5 code came from answering that question wrong.
