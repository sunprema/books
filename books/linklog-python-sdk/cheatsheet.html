Quick Reference

The LinkLog Cheatsheet

30-second summary

Record everything on every request into an invisible in-memory buffer → discard it on success (no file, ever) → commit a forensic checkpoint on failure. Core path is Linux-only (O_TMPFILE, /dev/shm); a hidden-temp-file fallback runs elsewhere for local dev.

Install & set up

package: llog · python ≥ 3.9 · deps: msgspec, xxhash · Apache-2.0$ pip install ./sdk/python

import llog
llog.init(service="payment-api")   # once, at process start

Before init() runs, every llog call is a silent no-op. Full signature:

Instrument

@llog.intent(name)

Decorator marking the business purpose; root of the graph. Opens the buffer; discards on success, commits on failure. Works on sync and async functions.

with llog.scope(name, replay_boundary=False, **attrs):

A logical step; nests, and becomes the parent for everything inside. replay_boundary=True marks an external-call seam for the replay generator.

llog.snapshot(name, value)

Capture a labelled value under the current parent. Bounded & safe; no-op outside an intent.

llog.commit() -> str | None

Manually commit the current session (a failure that never raised). Returns the artifact path or None.

Detect failures

PRIMARY — web: pure-ASGI middleware, auto-opens an intent "{METHOD} {path}"llog.instrument_fastapi(app, capture_5xx=True)
# or, on any ASGI app:
app.add_middleware(llog.LinkLogMiddleware, capture_5xx=True)

FALLBACK — CLIs / workers
llog.install_excepthook()      # chains to the previous hook; process still exits non-zero
llog.uninstall_excepthook()

Middleware is primary because frameworks swallow exceptions into 500s and sys.excepthook never fires for them. It catches both a raised exception and a swallowed 5xx (when capture_5xx=True).

mode="staging" auto-starts a watchdog that heartbeats /dev/shm/llog/live/{pid}; the sidecar reaps by heartbeat staleness, not pid. Choose staging when OOM-kills are failures you must capture.

Privacy — allowlist capture

llog.init(service="payment-api", capture_mode="allowlist",
          allow_fields={"amount", "currency", "order_id"},
          allow_config="llog.allow.json")
  • Ceiling = allow_fields ∪ allow_config JSON ∪ LLOG_ALLOW_FIELDS env (CSV).
  • Gate leaves, not containers — declaring "user" does not expose user.ssn. Structure is always kept; declare the safe leaf names.
  • Withheld values render as <withheld: str len=16> (type + shape, never the value).
  • Fails closed: a malformed config narrows capture, never widens it, never crashes init.

Storm control

Per-signature token bucket (signature = exception type + top frame + intent name), capped at max_commits_per_min (default 10). Over-cap commits are discarded but counted; the suppressed count rides the next committed ERROR event. Read llog.stats.suppressed. Distinct bugs have distinct signatures, so they never starve each other.

Traps to avoid

Foot-guns
  • Verify on disk. The SDK swallows its own errors — assert the artifact exists in events/, never trust a return value.
  • Linux for real / benchmarks. macOS runs the fallback; numbers and behavior there don't reflect the mechanism.
  • Don't double-wrap. With the middleware owning the request, don't also put @llog.intent on the route handler — use scope()/snapshot() inside instead.
  • snapshot() outside an intent is a counted no-op — there's no trace to attach to.
  • Same tmpfs. Keep the buffer and events/ on one filesystem (never /tmp) — links & renames can't cross it (EXDEV).
  • Declare leaves, not containers in the allowlist.

Inspect an artifact (downstream)

$ llog-ctl inspect events/<uuid>.llog   # human-readable graph, intent, error, frames
$ llog-ctl verify  events/<uuid>.llog   # checksum + classification
$ llog-ctl cat     events/<uuid>.llog   # JSON envelope
